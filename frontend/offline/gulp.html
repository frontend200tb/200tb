Установка gulp

Раньше была одна версия gulp которую нужно было устанавливать глобально и локально. Локальная версия используется в gulpfile.js глобальная для запуска в консоле. Но начиная с 4 версии gulp нужно устанавливать gulp-cli. Разделение произошло из-за проблем совместимости глобальной и локальной версии. gulp-cli (gulp command line interface) - это всего лишь способ запустить gulp-task из консоли.

  1. Если нет node.js и npm, то ставим сначала их nodejs
  2. Установим gulp-cli глобально во всей системеnpm i -g gulp-cli(сокращенно от npm install --global gulp-cli)
  3. Создадим папку для нашего проекта и перейдем в нее 
    mkdir project1
cd project1
d://project1/
  4. Инициализируем проектnpm init -y(то же что и npm init , не отвечая на вопросы)
  5. В итоге в папке с проектом появится файл package.jsond://project1/package.json
  6. После этого можно устанавливать пакеты, которые будут являться зависимостями для нашего приложения. Установим gulp в наш проект и занесем плагин gulp в раздел devDependencies (который нужен только для разработки) файла package.jsonnpm i -D gulp(сокращенно от npm install --save-dev gulp)
  7. В итоге в папке с проектом появится папка node_modules, в которой установлен gulp и все необходимые зависимости для того чтобы работал только gulp. Эту папку трогать не нужноd://project1/node_modules
  8. Создадим файл gulpfile.js в который будем прописывать задачи для gulpd://project1/gulpfile.js

gulpfile.js

Поскольку gulp работает на node.js, то в файле gulpfile.js доступны такие команды языка javascript как require, exports и другие

  1. В файле gulpfile.js создадим переменную в которую поместим плагин gulpconst gulp = require('gulp');или создадим только те переменные, какие мы будем использоватьconst {src, dest, watch, series, parallel} = require('gulp');
  2. Создадим функцию вывода текста в консоль и назовем ее hello 
    function hello () {
  console.log('Привет, я выполнился')
};
exports.hello = hello;
  3. Чтобы запустить эту задачу выполним команду в папке проектаgulp hello

Задачи по умолчанию

  1. В файле gulpfile.js напишем экспорт задачи по умолчаниюexports.default = style;
  2. Эта задача будет выполняться при запуске gulp в папке проектаgulp

Переместить файл

  1. Чтобы переместить файл напишем функцию move(), в ней запишем задачу gulp.src (возьми файл), pipe (перейди в следующую задачу), gulp.dist (сохрани файл) и экспортируем эту функцию в node.js 
    function move() {
  return gulp.src('src/css/main.css')
    .pipe(gulp.dest('dist/css/main.css'));
};
exports.move = move
  2. Запустим таск movegulp move
  3. В итоге появится новый файл d://project1/dist/css/main.css

Плагин gulp-sass

  1. Установим в наш проект плагин gulp-sass для преобразования файлов sass в css и занесем его в раздел зависимости файла package.jsonnpm i -D gulp-sass
  2. В файле gulpfile.js создадим переменную sassconst scss = require('gulp-sass');
  3. Напишем функцию (function) которая вернет (return) выполненную задачу (task). Первая задача берет файл scss и передает второй задаче.Вторая задача преобразует его в css. Третья задача записывает его в папку назначения. Если будут ошибки при компиляции, они запишутся в файл sass.logfile. 
    function style () {
  return gulp.src('src/scss/style.scss')
    .pipe(scss().on('error', scss.logfile))
    .pipe(gulp.dest('dist/css'));
}
  4. В итоге получим 
    const gulp = require('gulp'),
const scss = require('gulp-sass');
function style () {
  return gulp.src('src/scss/main.scss')
    .pipe(scss().on('error', scss.logfile))
    .pipe(gulp.dest('dist/css'));
}
exports.style = style;
  5. В терминале запустим задачу stylegulp style

Плагин gulp-concat

  1. Установим в наш проект плагин gulp-concat (для соединения файлов в один и переименования его) и занесем его в раздел зависимости файла package.jsonnpm i -D gulp-concat
  2. В файле gulpfile.js создадим переменную concatconst concat = require('gulp-concat');
  3. Напишем функцию (function) которая вернет (return) выполненную задачу (task). Первая задача берет исходные файлы и передает второй задаче.Вторая задача соединяет их в один и дает ему имя. Третья задача записывает его в папку назначения. 
    function concat () {
  return gulp.src('src/scss/style.scss')
    .pipe(concat(style.min.css))
    .pipe(gulp.dest('dist/css'));
}

gulp watch

  1. Напишем функцию слежения, которая при изменении файлов запускает другую функцию 
    function watch () {
  gulp.watch('src/sass/main.sass', style);
}
exports.watch = watch;
  2. В терминале запускаем задачу watchgulp watch

Плагин browser-sync

  1. Установим плагин browser-sync для автоматического обновления страницы в браузереnpm i -D browser-sync
  2. В файле gulpfile.js создадим переменную browserSyncconst browserSync = require('browser-sync').create();
  3. Напишем функцию которая запускает сервер 
    function browsersync () {
  browserSync.init({
    server: {
      baseDir: "./dist/"
    }
  });
}
  4. В итоге получим 
    const gulp = require('gulp'),
const scss = require('gulp-sass');
const concat = require('gulp-concat');
const browserSync = require('browser-sync').create();
function server () {
  browserSync.init({
    server: {
      baseDir: "./dist/"
    }
  });
}
function style () {
  return gulp.src('src/scss/main.scss')
    .pipe(scss().on('error', scss.logfile))
    .pipe(concat(style.min.css))
    .pipe(gulp.dest('dist/css'))
    .pipe(browserSync.stream())
}
function watch () {
  gulp.watch('src/sass/main.sass', style);
  gulp.watch('src/**/*.html').on('change', browserSync.reload);
}
exports.style = style;
exports.watch = watch;
exports.browsrsync = browsersync;

exports.default = gulp.parallel(browsersync, watch);
