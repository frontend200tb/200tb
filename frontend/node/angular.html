переключение темы

Приложения Angular строятся на базе паттерна проектирования MVC (модель - представление - контроллер). Это простота расширения, удобство сопровождения, удобство тестирования, стандартизация. Angular является частью стека MEAN (MongoDB, ExpressJS, Angular, NodeJS)

Установка

  1. Для установки Angular требуется установленные node.js и npm. Проверим версии установленных node.js и npm 
    node -v
v16.13.1
npm -v
8.1.2
    (сокращенно от node --version и npm --version)
    если их нет, то поставить с оффициального сайта https://nodejs.org/en/
  2. Установим angular cli глобально во всей системе. Эта же команда обновит уже установленную версию.npm i -g @angular/cli(сокращенно от npm install --global @angular/cli)
    Поскольку Angular CLI устанавливается глобально, то устанавливать его отдельно для каждого проекта не нужно. Все команды Angular CLI начинаются со слова ng. Проверим установленную версию Angular CLIng -v(сокращенно от ng version)
  3. Есть два способа установки Angular. Первый это ручная сборка (создать файл package.json с нужными зависимостями и запустить npm init), мы ей пользоваться не будем. Будем использовать только второй способ это через специальное консольное приложение Angular CLI.
  4. Создадим новый проект на Angular в папке app (если ее нет то она создастся)ng new app --style=scss --skip-tests --skip-install --routingapp имя нового проекта
    --style=scss добавит в проект scss вместо обычного css
    --skip-tests не создавать тесты
    --skip-install не устанавливать npm модули
    --routing добавить код для маршрутизации
    --prefix смена префикса по умолчанию для селекторов
    --skip-git не добавлять проект в репозиторий
    --dry-run не создавать файлы
    Все опции, которые мы прописали при создании проекта попадут в файл настроек .angular-cli.json который можно редактировать.
  5. Если npm модули не устанавливались, то их можно установить в любой момент командойnpm install
  6. Установим в проект ESLint ng add @angular-eslint/schematicsПодробнее можно посмотреть на сайте Переносим Angular проект на ESLint, с Prettier, Husky и lint-staged
  7. Добавим в наш конфиг ESLint правила стайл гайда AirBnBnpm install eslint-plugin-import eslint-config-airbnb-typescript --save-dev
  8. Добавим в наш проект Prettier npm i prettier eslint-config-prettier eslint-plugin-prettier --save-dev
  9. Добавим в наш проект Angular Materuai UI ng add @angular/material
  10. Добавим в наш проект Bootstrapng add @ng-bootstrap/ng-bootstrap

Деплой angular проекта на github

Есть несколько способов выложить готовый проект angular на github

  1. Первый способ. Собрать проект в папке docs, для этого в файле angular.json изменить значение параметра outputPath"outputPath": "docs",
  2. Собрать проект командойng build
  3. Перенести проект на гитхаб
  4. Перейти settings > pages > при выборе источника source выбрать docs вместо root
  5. Второй способ. Установим пакет ghpages глобально, чтобы задеплоить Angular проект на Github npm i -g angular-cli-ghpagesПосле этого можно собрать готовый проект командойng build --prod --base-href="https://frontend200tb.github.io/my-app/"

Структура проекта

После создания нового проекта, в папке проекта появляются начальные файлы и папки. В корне проекта лежат файлы

  • .angular-cli.json отвечает за настройку angular cli
  • .editorconfig настройки редактора чтобы код в проекте был одинаковый для всех
  • .gitignore говорит какие файлы и папки не отслеживать в git
  • karma.conf.js настройки инструмента тестирования karma
  • npm-debug.log ошибки связанные с пакетным менеджером
  • package.json содержит объекты scripts (команды для запуска из консоли), dependencies (зависимости для работы проекта), devDependencies (зависимости для разработки)
  • protractor.conf.js настройка системы protractor предназначенной для e2e (end-to-end) тестирования
  • README.md красивое описание проекта в формате markdown для git
  • tsconfig.json конфикурация typescript (например для компилирования в js с синтаксисом es5)

В папке node_modules лежат все пакеты необходимые для работы нашего приложения

В папке src лежит весь исходный код нашего приложения. Здесь происходит вся разработка. Рассмотрим файлы и папки из папки src

  • favicon.ico фавиконка нашего приложения
  • index.html единственная страница нашего проекта
  • main.ts запускает все наше приложение, в нем есть ссылка на environments, где выставляется режим разработки или продакшн
  • polyfills.ts подключает все библиотеки для работы typescript
  • styles.scss общий файл стилей. Содержит глобальные стили которые применятся ко всему нашему сайту
  • test.js нужен для тестирования
  • tsconfig.app.json отвечает за компиляцию typescript и наследуется от ../tsconfig.json
  • tsconfig.spec.json содержит конфигурацию для тестирования typescript
  • typings.d.js вспомогательный файл для работы с typescript
  • папка environments для хранения режима разработки (development mode или production mode). Мы запускаем приложение из консоли, поэтому в этой папке ничего изменять не надо
  • папка assets для хранения статических файлов
  • папка app хранится весь исходный код нашего приложения

Изначально в папке app лежит только один компонент AppComponent который описывается файлами

  • app.component.html html шаблон
  • app.component.scss scss стили
  • app.component.spec.ts unit тесты
  • app.component.ts код на тайпскрипте
  • app.module.ts файл модуля

Начало работы

  1. Откроем наш проект в VS Code. В папке src в файле index.html находится только один элемент<app-root><app-root>в котором уже будет находиться все наше SPA (одностраничное приложение) работающее без перезагрузки браузера
  2. Соберем проект и запустим сервер одной командойng serve -o(сокращенно от ng serve --open)
    Флаг -o откроет сервер в браузере. Если запустили проект без флага -o, то перейдем в браузере на localhost:4200 и увидим шаблон нашего нового приложения
  3. Создадим компонент comp в папке components. Если такой папки нет, то она создастсяng g c components/comp(сокращенно от ng generate component components/comp)

    В папке components получим новые файлы компонента comp 

    comp.component.html 
comp.component.scss 
comp.component.ts

    Кроме того, компонент прописывается в файле app.module.ts 

    import { CompComponent } from './components/comp.component';
@NgModule({
  declarations: [
    CompComponent,

  4. Создадим директиву dir ng g d dir(сокращенно от ng generate directive dir)
  5. Создадим пайп pipng g p pip(сокращенно от ng generate pipe pip)
  6. Создадим сервис servng g s serv(сокращенно от ng generate service serv)
  7. Создадим интерфейс inter в папке modelsng g i inter(сокращенно от ng generate interface models/inter)

    В папке models получим новый файл интерфейса

    inter.ts
  8. Создадим класс new-class в папке modelsng g cl new-class(сокращенно от ng generate class models/new-class)

    В папке models получим новый файл класса

    new-class.ts
  9. Создадим модуль modng g m mod(сокращенно от ng generate module mod)
  10. Создадим Guard guarng g g guar(сокращенно от ng generate guard guar)
  11. Создадим Enum ennng g e enn(сокращенно от ng generate enum enn)

ng build Сборка проекта

  1. Соберем проект командой. Проект соберется в папке dist
    2. ng build

Ошибки при сборке проекта

  1. error TS2564: Property 'sortWordInput' has no initializer and is not definitely assigned in the constructor. Это ошибка Strict Class Initialization. Когда объявлена переменная и ее тип, но не назначено значение. Чтобы ее решить нужно добавить к типу | undefinedhttps://www.typescriptlang.org/docs/handbook/release-notes/typescript-2-7.html
  2. error TS2322: Type 'string | undefined' is not assignable to type 'string'. Это ошибка Non-null assertion operator. Когда строковая переменная не может быть null или undefined. Чтобы ее решить надо добавить постфиксный оператор "!"" к строковой переменнойhttps://www.typescriptlang.org/docs/handbook/release-notes/typescript-2-0.html#non-null-assertion-operator
  3. error TS2532: Object is possibly 'undefined'. Перед действием с объектом нужно проверить что он есть if (obj)
