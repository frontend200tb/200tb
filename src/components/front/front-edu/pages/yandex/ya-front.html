3. Чем отличается localstorage от sessionstorage?

  • Для localstorage можно выставить время хранения, а sessionstorage сохраняется только в рамках текущей сессии (вкладки)
  • К localstorage можно получить доступ из соседней вкладки на том же домене, а к sessionstorage — нельзя
  • localstorage позволяет хранить произвольный тип данных, а sessionstorage — только строки
  • localstorage позволяет перезаписывать данные, а sessionstorage — не позволяет (применяется для избежания гонки запросов на изменение в соседних вкладках)

4. Чем реляционные базы данных отличаются от нереляционных?

  • Реляционные позволяют легко горизонтально масштабироваться, а нереляционные - вертикально
  • Реляционные имеют множество способов обращения к данным, а нереляционные используют язык структурированных запросов
  • Реляционные позволяют добиться надежности в выполнении транзакций, а нереляционные выбрать тип хранилища под конкретную задачу
  • Реляционные соблюдают CAP-теорему (consistency-availability-partition tolerance), а нереляционные обладают только 2 из 3 пунктов
  • Реляционные базы позволяют храненить данные большого объема данных без фиксированной структуры, а нереляционные позволяют хранить данные по строгим заданным схемам

25. Чем бандл отличается от чанка?

  • Чанки могут разбиваться на отдельные бандлы
  • Уменьшение размера бандла не ведет к уменьшению размера чанков
  • Бандлы могут разбиваться на отдельные чанки
  • Бандлы - это артефакты, которые появляются после сборки приложения, а чанки - это исходный декомпозированный код приложения для последующей сборки бандла
