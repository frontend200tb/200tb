https://nodejs.org/en

Download Node.js(LTS)

Скачается файл

node-v20.17.0-x64

Запустим его. Вместе с node.js поставится и npm

Проверим версии установленных node.js и npm

node -v
v20.17.0
npm -v
10.8.2

(сокращенно от node --version и npm --version)

Создадим папку с названием projectmkdir projectЧтобы удалить пустую папку с названием project наберитеrmdir project

Перейдем в папку с названием project cd project
d://project/ Чтобы выйти из папки наберите cd ..
d://

Инициализируем проектnpm init -y(сокращенно от npm init --yes, то же что и npm init, не отвечая на вопросы)

В итоге в папке с проектом появится файл package.json. После этого можно устанавливать пакеты, которые будут являться зависимостями для нашего приложения.d://project/package.json

Для примера, установим @babel/cli, @babel/core, @babel/preset-env npm i -D @babel/cli @babel/core @babel/preset-env (сокращенно от npm install --save-dev @babel/cli @babel/core @babel/preset-env)

Появится папка node_modules, файл package-lock.json, а в файл package.json добавятся зависимости "devDependencies": {
  "@babel/cli": "^7.18.10",
  "@babel/core": "^7.18.13",
  "@babel/preset-env": "^7.18.10"
}

Чтобы удалить пакеты и удалить зависимости из файла package.json npm uninstall -D @babel/cli @babel/core @babel/preset-env(Флаг -D удаляет devDependencies, флаг -S удаляет Dependencies)

Глобальная и локальная установка npm пакетов

При установке пакетов через npm есть два варианта: глобальная и локальная установка.

При глобальной установке пакеты устанавливаются в определенное место в системе, общее для всех проектов. Это удобно, когда пакет является утилитой командной строки, которую необходимо использовать в разных проектах.

В случае локальной установки пакеты устанавливаются в каталог node_modules внутри проекта. Это подходит для пакетов, которые необходимы только для конкретного проекта.

При локальной установке пакеты располагаются в каталоге node_modules, который находится в корневом каталоге проекта.

Проверка расположения пакетов

Можно посмотреть в какую папку глобально устанавливается каталог node_modules с пакетами

npm prefix -g c:\Users\11001000\AppData\Roaming\npm

Также чтобы получить путь для установки глобальных пакетов, можно выполнить команду

npm config get prefix c:\Users\11001000\AppData\Roaming\npm

Чтобы изменить этот папку на '.node_modules_global', нужно выполнить команду

npm config set prefix=$HOME/.node_modules_global

Можно посмотреть все установленные глобально пакеты и путь к ним

npm list -g

Для проверки расположения установленных локально пакетов можно воспользоваться командой

npm list

Таким образом, установленные через npm пакеты будут находиться либо в каталоге проекта (при локальной установке), либо в определенной директории системы (при глобальной установке).
