Установка Gulp

Настраиваем Gulp так, чтобы другой разработчик вас не избил

Gulp это менеджер задач (task runner). Появился как замена устаревшему grunt.js более сложного в настройке. Раньше была одна версия gulp которую нужно было устанавливать глобально и локально. Локальная версия используется в gulpfile.js глобальная для запуска в консоле. Но начиная с 4 версии gulp нужно устанавливать gulp-cli глобально, а gulp - локально. Разделение произошло из-за проблем совместимости глобальной и локальной версии. gulp-cli (gulp command line interface) - это всего лишь способ запустить gulp-task из консоли.

  1. Проверим версии установленных node.js и npm

    node -v
v16.13.1
npm -v
8.1.2

    если их нет, то поставить с оффициального сайта nodejs

  2. Создадим папку для нашего проекта и перейдем в нее 
    mkdir project1
cd project1
d://project1/

  3. Инициализируем проект

    npm init -y

  4. В итоге в папке с проектом появится файл package.json.

    d://project1/package.json

    После этого можно устанавливать пакеты, которые будут являться зависимостями для нашего приложения.

  5. Установим gulp-cli глобально во всей системе

    npm i -g gulp-cli

    (сокращенно от npm install --global gulp-cli)

  6. Установим gulp в наш проект и занесем плагин gulp в раздел devDependencies (который нужен только для разработки) файла package.json

    npm i -D gulp gulp-cli

    (сокращенно от npm install --save-dev gulp)

  7. В итоге в папке с проектом появится папка node_modules и файл package-lock.json. В node_modules установлен gulp и все необходимые зависимости для того чтобы работал только gulp. Эту папку трогать не нужно

    d://project1/node_modules
d://project1/package-lock.json

  8. Проверим установленную версию gulp

    gulp -v
CLI version: 2.3.0
Local version: 4.0.2

  9. Создадим файл gulpfile.js в который будем прописывать задачи для gulp

    touch gulpfile.js
d://project1/gulpfile.js

gulpfile.js

Поскольку gulp работает на node.js, то в файле gulpfile.js доступны такие команды языка javascript как require, exports и другие.

  1. В файле gulpfile.js создадим переменную в которую поместим плагин gulp

    const gulp = require('gulp');

    или создадим только те переменные, какие мы будем использовать

    const {src, dest, watch, series, parallel} = require('gulp');

  2. Каждая задача пишется в отдельную функцию. Создадим функцию вывода текста в консоль и назовем ее hello. Сделаем export этой функции для выполнения ее из командной строки

    function hello () {
  console.log('Привет, я выполнился');
}
exports.hello = hello;

  3. Чтобы запустить эту задачу выполним команду в командной строке в папке проекта

    gulp hello

Задача копирования файлов

  1. Чтобы скопировать файл напишем функцию copy(), в ней запишем задачу gulp.src (возьми файл), pipe (перейди в следующую задачу), gulp.dist (сохрани файл) и экспортируем эту функцию в node.js

    function copy() {
  return gulp.src('src/css/main.css')
    .pipe(gulp.dest('dist/css/main.css'));
};
exports.copy = copy;

  2. Запустим таск copy

    gulp copy

  3. В итоге появится новый файл

    d://project1/dist/css/main.css

Задача удаления файлов

  1. Первым делом надо очищать папку dist от файлов с предыдущей сборки. Установим плагин gulp-rm набрав в командной строке

    npm i -D gulp-rm

  2. В gulpfile создадим переменную

    const rm = require('gulp-rm')

  3. Чтобы очитить папку напишем функцию clean(), в ней запишем задачу gulp.src (возьми все файлы) с параметром read: false (не читай эти файлы), pipe (перейди в следующую задачу), gulp.rm (удали эти файлы) и экспортируем эту функцию в node.js

    function clean() {
  return gulp.src('dist/**/*', {read: false})
    .pipe(rm());
};
exports.clean = clean

Задачи по умолчанию

  1. В файле gulpfile.js напишем экспорт задачи по умолчанию

    exports.default = gulp.series('clean', 'copy');

  2. Эта задача будет выполняться при запуске gulp в папке проекта

    gulp

gulp watch

  1. Напишем функцию слежения, которая при изменении файлов запускает другую функцию

    function watch () {
  gulp.watch('src/sass/main.sass', style);
}
exports.watch = watch;

  2. В терминале запускаем задачу watch

    gulp watch

Плагины

Плагин gulp-pug

  1. Установим в наш проект плагин gulp-pug для преобразования файлов pug в html и занесем его в раздел зависимости файла package.json

    npm i -D gulp-pug

  2. В файле gulpfile.js создадим переменную pug

    const pug = require('gulp-pug');

  3. В переменную pugTask запишем стрелочную функцию, которая будет брать pug файлы из исходной папки, передавать их плагину gulp-pug, полученный html код делать красивым, записывать его в конечную папку и перезапускать браузер чтобы увидеть примененные изменения

    const pugTask = () => {
return gulp.src(config.src.html)
          .pipe(pug())
          .pipe(pug({
              pretty: true
          }))
          .pipe(gulp.dest(config.dist.html))
}

Плагин gulp-sass

  1. Установим в наш проект плагин gulp-sass для преобразования файлов sass в css и занесем его в раздел зависимости файла package.json

    npm i -D gulp-sass

  2. В файле gulpfile.js создадим переменную sass

    const sass = require('gulp-sass');

  3. Напишем функцию (function) которая вернет (return) выполненную задачу (task). Первая задача берет файл scss и передает второй задаче.Вторая задача преобразует его в css. Третья задача записывает его в папку назначения. Если будут ошибки при компиляции, они запишутся в файл sass.logfile.

    const scssTask = () => {
  return gulp.src(config.src.css)
        .pipe(sass().on('error', sass.logError))
        .pipe(gulp.dest('dist'));
}

Плагин gulp-concat

  1. Установим в наш проект плагин gulp-concat (для соединения файлов в один и переименования его) и занесем его в раздел зависимости файла package.json

    npm i -D gulp-concat

  2. В файле gulpfile.js создадим переменную concat

    const concat = require('gulp-concat');

  3. Напишем функцию (function) которая вернет (return) выполненную задачу (task). Первая задача берет исходные файлы и передает второй задаче.Вторая задача соединяет их в один и дает ему имя. Третья задача записывает его в папку назначения.

    function concat () {
  return gulp.src('src/scss/style.scss')
    .pipe(concat(style.min.css))
    .pipe(gulp.dest('dist/css'));
}

Плагин gulp-autoprefixer

  1. Установим в наш проект плагин gulp-autoprefixer (для создания префикса свойствам css для поддержки в старых браузерах) и занесем его в раздел зависимости файла package.json

    npm i -D gulp-autoprefixer

  2. В файле gulpfile.js создадим переменную autoprefixer

    const autoprefixer = require('gulp-autoprefixer');

  3. Напишем функцию для добавления префиксов в файлах css

    function prefix () {
  gulp.src('src/*.css')
    .pipe(autoprefixer({
      browser: ['last 2 versions'],
      cascade: false
    }))
    .pipe(gulp.dest('dist'))
};
exports.prefix = prefix;

Плагин gulp-clean-css

  1. Установим в наш проект плагин gulp-clean-css (для удаления пробелов и коментариев из css) и занесем его в раздел зависимости файла package.json

    npm i -D gulp-clean-css

  2. В файле gulpfile.js создадим переменную cleanCSS

    const cleanCSS = require('gulp-clean-css');

  3. Напишем функцию для минификации css

    function minify-css () {
  return gulp.src('src/*.css')
    .pipe(cleanCSS({compatibility:'ie8'}))
    .pipe(gulp.dest('dist'));
}

Плагин gulp-plumber

  1. Установим в наш проект плагин gulp-plumber (сборка не остановится не смотря на ошибки, ошибки запломбируются) и занесем его в раздел зависимости файла package.json

    npm i -D gulp-plumber

  2. В файле gulpfile.js создадим переменную plumber

    const plumber = require('gulp-plumber');

  3. Добавим plumber в функцию преобразования pug файлов

    function pugTask () {
  return gulp.src('src/**/*.pug')
        .pipe(plumber())
        .pipe(pug({
            pretty: true
          }))
        .pipe(plumber.stop())
        .pipe(gulp.dest('dist/'))
}

  4. Добавим plumber в функцию преобразования scss файлов

    function scssTask () {
  return gulp.src(config.src.css)
              .pipe(plumber())
              .pipe(sass().on('error', sass.logError))
              .pipe(plumber.stop())
              .pipe(gulp.dest(config.dist.css))
}

Плагин gulp-sourcemaps

  1. Установим в наш проект плагин gulp-sourcemaps (браузер покажет в каком файле источник стилей) и занесем его в раздел зависимости файла package.json

    npm i -D gulp-sourcemaps

  2. В файле gulpfile.js создадим переменную sourcemaps

    const sourcemaps = require('gulp-sourcemaps');

  3. Добавим sourcemaps в функцию минификации css файлов

    function minify-css () {
  return gulp.src('src/css/*.css')
    .pipe(sourcemaps.init())
      .pipe(cleanCSS({compatibility:'ie8'}))
    .pipe(sourcemaps.write())
    .pipe(gulp.dest('dist/css'));
}

  4. Добавим sourcemaps в функцию соединения js файлов

    function scripts () {
  return gulp.src('src/js/*.js')
    .pipe(sourcemaps.init())
      .pipe(concat('main.js'))
    .pipe(sourcemaps.write())
    .pipe(gulp.dest('dist/js'));
}

Плагин browser-sync

  1. Установим плагин browser-sync для запуска локального сервера. Это нужно для автоматического обновления страницы в браузере

    npm i -D browser-sync

  2. В файле gulpfile.js создадим переменную browserSync

    const browserSync = require('browser-sync').create();
  3. Напишем функцию которая запускает сервер 
    function webServer () {
  browserSync.init({
    server: {
      baseDir: './dist/'
    },
    port: 9000,
    host: 'localhost',
    notify: true,
  });
}

Плагин babel

Babel преобразует js код с новым синтаксисом в старый синтаксис, чтобы его понимали старые браузеры. Работает babel при помощи пресетов. Нам будет достаточно одного пресета @babel/preset-env

  1. Установим в наш проект плагин gulp-babel и занесем его в раздел зависимости файла package.json

    npm i -D gulp-babel @babel/core @babel/preset-env

  2. В файле gulpfile.js создадим переменную babel

    const babel = require('gulp-babel');

  3. Добавим babel в функцию соединения js файлов

    function scripts () {
  return gulp.src('src/js/*.js')
    .pipe(sourcemaps.init())
      .pipe(concat('main.js'))
      .pipe(
        babel({
          presets: ['@babel/env']
        })
      )
    .pipe(sourcemaps.write())
    .pipe(gulp.dest('dist/js'));
}

Настройка gulp для pug, scss, js

Установим необходимые пакеты

npm i -D gulp-pug gulp-plumber npm i -D gulp-sass sass npm i -D gulp-concat gulp-sourcemaps browser-sync

В gulpfile.js напишем код

const gulp = require('gulp'),
      pug = require('gulp-pug'),
      plumber = require('gulp-plumber'),
      sass = require('gulp-sass')(require('sass')),
      concat = require('gulp-concat'),
      sourcemaps = require('gulp-sourcemaps'),
      browserSync = require('browser-sync').create();

const src = 'src/',
      dist = 'dist/';

const config = {
  src : {
    copy : src + 'assets/**/*',
    html : src + '**/*.pug',
    nohtml: '!' + src + '**/_*.pug',
    css : src + 'scss/*.scss',
    nocss : '!' + src + '**/_*.scss',
    js : src + '**/*.js',
  },
  dist : {
    copy : dist + 'assets/',
    html : dist,
    css : dist + 'css/',
    js : dist + 'js/',
  },
  watch : {
    copy : src + 'assets/**/*',
    html : src + '**/*.pug',
    css : src + 'scss/*.scss',
    js : src + '**/*.js',
  },
}

const copyTask = () => {
  return gulp.src(config.src.copy)
    .pipe(gulp.dest(config.dist.copy));
}

const pugTask = () => {
  return gulp.src([config.src.html, config.src.nohtml])
              .pipe(plumber())
              //.pipe(sourcemaps.init())
              .pipe(pug({
                pretty: true
              }))
              //.pipe(sourcemaps.write('./'))
              .pipe(plumber.stop())
              .pipe(gulp.dest(config.dist.html))
}

const scssTask = () => {
  return gulp.src(config.src.css)
              .pipe(plumber())
              .pipe(sourcemaps.init())
              .pipe(sass().on('error', sass.logError))
              .pipe(sourcemaps.write('./'))
              .pipe(plumber.stop())
              .pipe(gulp.dest(config.dist.css))
}

const jsTask = () => {
  return gulp.src(config.src.js)
              .pipe(plumber())
              .pipe(sourcemaps.init())
              .pipe(concat('index.js'))
              .pipe(sourcemaps.write('./'))
              .pipe(plumber.stop())
              .pipe(gulp.dest(config.dist.js))
}

const webServer = () => {
  browserSync.init({
    server: {
      baseDir: dist
    },
    port: 9000,
    host: 'localhost',
    notify: true,
  })
}

const watchFiles = () => {
  gulp.watch([config.watch.html], gulp.series(pugTask));
  gulp.watch([config.watch.css], gulp.series(scssTask));
  gulp.watch([config.watch.js], gulp.series(jsTask));
}

const start = gulp.series(pugTask, scssTask, jsTask);

exports.start = start;
exports.copy = copyTask;
exports.pug = pugTask;
exports.scss = scssTask;
exports.js = jsTask;

exports.default = gulp.parallel(start, watchFiles, webServer);

Список доступных задач можно посмотреть по команде

gulp --tasks
