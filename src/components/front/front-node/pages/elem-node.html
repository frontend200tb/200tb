Установка node.js

Установим последнюю версию node.js с официального сайта

https://nodejs.org/en

Нажимаем кнопку

Get Node.js

Скачается файл

node-v22.20.0-x64.msi

Запустим его. Вместе с node.js поставится и npm

Проверим версии установленных node.js и npm

node -v
v22.20.0
npm -v
10.9.3

(сокращенно от node --version и npm --version)

Создание нового проекта

  1. Создадим папку с названием проекта, например 200cf

    mkdir 200cf

    Чтобы удалить пустую папку с названием 200cf наберите

    rmdir 200cf

    Перейдем в папку с названием 200cf

    cd 200cf
d://200cf/

    Чтобы выйти из папки наберите

    cd ..
d://
  2. Откроем VSCode

  3. В VSCode откроем папку 200cf, в которой будет новый проект

    File - Open Folder

  4. Инициализируем проект

    npm init -y

    (сокращенно от npm init --yes, то же что и npm init, не отвечая на вопросы)

  5. В итоге в папке с проектом появится файл package.json. После этого можно устанавливать пакеты, которые будут являться зависимостями для нашего приложения.

    d://200cf/package.json

  6. Для примера, установим webpack, webpack-cli и webpack-dev-server

    npm i -D webpack webpack-cli webpack-dev-server

    (сокращенно от npm install --save-dev webpack webpack-cli webpack-dev-server)

  7. В итоге в папке с проектом появится папка node_modules и файл package-lock.json. В node_modules установлены webpack, webpack-cli, webpack-dev-server и все необходимые зависимости для того чтобы они работали. Эту папку трогать не нужно

    d://200cf/node_modules d://200cf/package-lock.json
  8. В файле package.json появятся записи: 
    "devDependencies": {
  "webpack": "^5.102.0",
  "webpack-cli": "^6.0.1",
  "webpack-dec-server":"^5.2.2"
}

  9. Чтобы удалить пакеты и удалить зависимости из файла package.json

    npm uninstall -D webpack webpack-cli webpack-dev-server

    (Флаг -D удаляет devDependencies, флаг -S удаляет Dependencies)

Глобальная и локальная установка npm пакетов

При установке пакетов через npm есть два варианта: глобальная и локальная установка.

При глобальной установке пакеты устанавливаются в определенное место в системе, общее для всех проектов. Это удобно, когда пакет является утилитой командной строки, которую необходимо использовать в разных проектах.

При локальной установки пакеты устанавливаются в каталог node_modules внутри проекта. Это подходит для пакетов, которые необходимы только для конкретного проекта.

При локальной установке пакеты располагаются в каталоге node_modules, который находится в корневом каталоге проекта.

Проверка расположения пакетов

Можно посмотреть в какую папку глобально устанавливается каталог node_modules с пакетами

npm prefix -g c:\Users\11001000\AppData\Roaming\npm

Также чтобы получить путь для установки глобальных пакетов, можно выполнить команду

npm config get prefix c:\Users\11001000\AppData\Roaming\npm

Чтобы изменить этот папку на '.node_modules_global', нужно выполнить команду

npm config set prefix=$HOME/.node_modules_global

Можно посмотреть все установленные глобально пакеты и путь к ним

npm list -g

Для проверки расположения установленных локально пакетов можно воспользоваться командой

npm list

Таким образом, установленные через npm пакеты будут находиться либо в каталоге проекта (при локальной установке), либо в определенной директории системы (при глобальной установке).

Для примера, установим глобально webpack, webpack-cli и webpack-dev-server

npm i -g webpack webpack-cli webpack-dev-server

(сокращенно от npm install --global webpack webpack-cli webpack-dev-server)

В итоге в папке

c:\Users\11001000\AppData\Roaming\npm\node_modules\

Появятся три папки

webpack webpack-cli webpack-dev-server

Для удаления глобально установленных webpack, webpack-cli и webpack-dev-server

npm uninstall -g webpack webpack-cli webpack-dev-server
