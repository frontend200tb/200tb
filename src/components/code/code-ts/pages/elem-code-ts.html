install

  1. Установка typescript глобально

    npm i -g typescript

    Если нужно установить только в проект

    npm i -D typescript

  2. Проверим установленную версию typescript

    tsc -v Version 5.2.2

  3. Теперь мы можем скомпилировать любой ts файл (например, index.ts) в js файл командой

    tsc index.ts

    При этом создастся файл index.js

  4. Создадим папку для нового проекта

    mkdir ts_project

  5. Перейдем в папку ts_project

    cd ts_project

  6. Инициализируем typescript в новом проекте

    tsc --init

  7. При этом будет создан файл конфигурации

    tsconfig.json

  8. Теперь можно скомпилировать все ts файлы проекта командой

    tsc
  9. Вся конфигурация для компилятора ts файлов записывается в файл tsconfig.json, например 
    {
  "CompilerOptions": {
    "target": "es6",
    "sourceMap": true,
    "outFile": "build/tsc.js"
  },
  "include": [
    "src"
  ],
  "exclude": [
    "node_modules"
  ]
}

Установка webpack для работы с typescript

Далее мы установим пакеты webpack и ts-loader в качестве зависимостей в нашем проекте:

npm i -D webpack webpack-cli ts-loader webpack-dev-server

Давайте добавим файл конфигурации webpack в наш проект. В корневой папке проекта создайте файл webpack.config.js со следующими конфигурациями:

const path = require('path');

module.exports = {
  entry: './src/index.ts',
  module: {
    rules: [
      {
        test: /\.ts?$/,
        use: 'ts-loader',
        exclude: /node_modules/,
      },
    ],
  },
  resolve: {
    extensions: ['.tsx', '.ts', '.js'],
  },
  output: {
    filename: 'bundle.js',
    path: path.resolve(__dirname, 'dist'),
  },
  devServer: {
    static: path.join(__dirname, "dist"),
    compress: true,
    port: 4000,
  },
};

Typescript

Typescript это надстройка над языком Javascript со строгой типизацией, где без указания типа создаваемых данных ничего сделать нельзя. Строгая типизация гарантирует что уменьшится количество ошибок от несовпадения данных которые мы ожидаем на вход нашей функции.

Typescript это компилируемый язык, в отличие от Javascript. Но после компиляции он становится обычным Javascript, который выполняется в браузере.

Плюсы тайпскрипта. Кроссплатформенный язык. Предпочтителен для больших приложений. Улучшает читаемость кода и поддержку готового приложения. Улучшает документацию проекта. На этапе разработке показывает ошибки. Синтаксический сахар.

Минусы тайпскрипта. Больше кода примерно на 30% при разработке, но на размер конечного кода (который будет уже на javascript) это не повлияет. Требует файла конфигурации где описывается его работа. Отдельный пакет для языка.

Типы указываются при объявлении переменной.let isMoodGood: boolean;

Если при объявлении мы записываем в переменную значение то тип указывать не обязательноlet isMoodGood = true;но можно и указыватьlet isMoodGood: boolean = true;

Типы данных: Boolean, String, Number, Array, Tuple, Enum, Any, Void, Null, Undefined, Never.
