1.3 Многошаговые программы (15)

Напишем программу, которая в каждой новой строке выводит последовательно символы #, $, @.

Программа 3 (17)

#include <stdio.h>

main() {
  printf("#\n");
  printf("$\n");
  printf("@\n");
}

Эту же задачу можно решить с помощью одного вызова функции printf().

Программа 4 (17)

#include <stdio.h>

main() {
  printf("#\n$\n@\n");
}

Если после выполнения программы консольное окно сразу закрывается, то можно его задержать так, что программа закончится только после нажатия на какую-либо клавишу. для этого используем вызов функции getch(), которая содержится в библиотеке conio.h

Программа 5 (18)

#include <stdio.h>
#include <conio.h>

main() {
  printf("#\n$\n@\n");
  getch();
}

Очистим экран перед тем как выведем результат работы программы. Для этого используем вызов функции clrscr() из библиотеки conio.h

Программа 6 (18)

#include <stdio.h>
#include <conio.h>

main() {
  clrscr();
  printf("#\n$\n@\n");
  getch();
}

1.5 Оформление программ на Си (23)

Текст программы можно писать с одним пробелом или несколькими, на разных строках или на одной. Для компилятора нет разницы. Лишние пробелы и переводы строк он игнорирует.

Цель программиста не в том, тчобы сделать текст программы компактным, а в том, чтобы сделать его наиболее удобным для чтения и понимания. Сравним программы 14 и 15.

Программа 14 (23)

#include <stdio.h>
#include <conio.h>

main() {
  clrscr();
  printf("***");
  getch();
}

Программа 15 (24)

#include <stdio.h>
#include <conio.h>
main() {clrscr();printf("***");getch();}

Комментарии дают информацию о том что делает программа. Напишем программу с комментариями

Программа 16 (25)

#include <stdio.h>
#include <conio.h>

main() {
  // display one
  clrscr();    // to clear display
  printf("@"); // print @
  getch();     // to stop display

  // display two
  clrscr();    // to clear display
  printf("%"); // print %
  getch();     // to stop display
}

Комментарии помогают во время отладки исключать куски кода. Исключим с помощью коментария вывод второго экрана

Программа 17 (26)

#include <stdio.h>
#include <conio.h>

main() {
  // display one
  clrscr();    // to clear display
  printf("@"); // print @
  getch();     // to stop display
/*
  // display two
  clrscr();    // to clear display
  printf("%"); // print %
  getch();     // to stop display
*/
}

Глава 5. Графика (75)

5.1 Графический режим (75)

У монитора два режима - текстовый и графический.

В текстовом режиме минимальным объектом отображения на экране является символ. Экран монитора может содержать не более 80 символов по горизонтали и 25 символов по вертикали, то есть 2000 визуальных объектов. Имеются ограничения по управлению цветом символов.

В графическом режиме минимальным объектом отображения на экране является пиксель. Пиксель имеет меньшие размеры по сравнению с символом. Его размеры определяются разрешением монитора. Имеется минимально допустимое значение размера пикселя, определяемое техническими параметрами монитора.

Любое изображение формируется из простых геометрических фигур. Это точки, отрезки, окружности и т.д.

Графические координаты задают положение точки на экране дисплея. В качестве координат пикселя используется его номер по горизонтали и вертикали. Начальная точка это верхний левый угол экрана. Значения x отсчитываются слева направо, значения y - сверху вниз.

Библиотека graphics.h содержит функции, отвечающие за запуск графического режима.
