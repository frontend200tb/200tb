about

История создания языка Си

К 1969 году языков программирования было много. Господствовал процедурный стиль программирования. Когда писались функции. Был язык для инженерных расчетов Fortran. Был язык для экономических расчетов Kobol. А языка для программистов не было. Программисты работали под UNIX, писали на ассемблере, а вокруг уже работали на языках высокого уровня. И они стали пытаться сделать язык для себя. Сначала язык A, потом язык B, потом Си и на этом остановились.

1969-1972 - на базе языка B (би) - упрощенного варианта BCPL (Basic Combined Programming Language) в компании Bell Labs программистом Деннисом Ритчи (Dennis Ritchie) разрабатывался язык Си. Второй соавтор языка Си это Кен Томпсон.

Предшественниками языка Си были

1966 язык BCPL (Деннис Ритчи)

1969 язык B (Деннис Ритчи)

1972 язык C (Деннис Ритчи). Создан для разработки ОС UNIX на платформе PDP-7

1973 на язык Си перенесен значительный фрагмент ядра Unix для PDP-11, ранее разработанный на языке ассемблера

1978 Деннис Ритчи вместе с Брайаном Керниганом написал хорошую книгу "Язык программирования Си"

1985 книга Кернигана и Ритчи переведена на русский язык и вышла под названием "Язык программирования Си. Задачи по языку Си"

1988 выходит второе издание книги Кернигана и Ритчи "Язык программирования Си" (первое описание будущего стандарта ANSI C)

1989 - созданный в 1983 году комитет Американского института стандартов (ANSI) ратифицирует стандарт "Programming Language C" (ANSI C, Standard C, C89)

1990 - Международная организация по стандартизации (ISO) принимает стандарт ANSI C как стандарт ISO/IEC 9899:1990(C90)

1992 второе издание книги Кернигана и Ритчи "Язык программирования Си" выходит на русском языке

1999 опубликован стандарт ISO/IEC 9899:1999(C99). В нем добавлены встроенные (inline) функции, новые типы данных (long long int, complex и др.), массивы переменной длины, макроопределения с переменной местностью и однострочные комментарии

2007 начата работа над новым вариантом стандарта C1x

Лексемы языка Си

Единицы текста программы, которые при компиляции воспринимаются как единое целое и по смыслу не могут быть разделены на более мелкие элементы

  1. Идентификаторы

    Идентификатор - любая последовательность букв A-Z, a-z, цифр 0-9, символов подчеркивания _, не начинающаяся с цифры и имеющая длину не более 31 символа

  2. Ключевые слова

    char, double, enum, float, int, long, short, struct, signed, union, unsigned, void, typedef, const, volatile, auto, extern, register, static, break, continue, do, for, goto, if, return, switch, while, default, case, else, sizeof и другие. Ключевые слова не могут быть идентификаторами.

  3. Константы

    Константа - неизменяемое арифметическое значение целого, вещественного, символьного или перечислимого типа, нулевой указатель либо строковый литерал.

    Целые числа записываются в системах счисления по основаниям 10, 8, 16. Целочисленный ноль в любой системе счисления - 0. Десятичные целые не начинаются с нуля. Восьмеричные целые начинаются с нуля. Шестнадцатеричные целые начинаются с 0x или 0X. Вещественные записываются в десятичной системе в следующих форматах:

    {целая часть}.{дробная часть}

    {целая часть}{e|E}{порядок}

    .{дробная часть}{e|E}{порядок}

  5. знаки пунктуации и специальные символы

Типы данных

Целочисленные типы данных

Размеры стандартных типов данных char, short, int, long, long long четко не определены и могут быть разными на разных платформах (компьютерах, операционных системах и компиляторах для них).

  • unsigned char. 8 бит. От 0 ('\x00') до 255 ('\xFF')
  • [signed] char. 8 бит. От -128 до 127
  • unsigned short [int]. 16 бит. от 0 до 65535
  • [signed] short [int]. 16 бит. от -32768 до 32767
  • unsigned [int]. 32 бита. от 0 до 4294967295
  • [signed] int. 32 бита. от -2147483648 до 2147483647
  • unsigned long. 32 бита. от 0 до 4294967295
  • [signed] long. 32 бита. от -2,147,483,648 до 2,147,483,647
  • unsigned long long. 64 бита. от 0 до 18,446,744,073,709,551,615
  • [signed] Long long. 64 бита. от -9,223,372,036,854,775,808 до 9,223,372,036,854,775,807

Платформонезависимые типы данных

Начиная с С99 определены типы данных, в которых можно указать размер для различных целей. Они считаются платформонезависимыми.

  • int32_t и uint32_t - размер ровно 32 бита (знаковый и беззнаковый).
  • int_least32_t и uint_least32_t- ближайший по длине, не менее 32 бит (может и больше).
  • int_fast32_t и uint_fast32_t- самый быстрый длиной не менее 32, для этого типа данных самая быстрая работа на данной платформе и есть 32 бита (или больше). То есть если самые быстрые вычисления будут на 64 битах, то int_fast32_t будет иметь размер 64 бита.

Аналогичные типы не только для 32, но и для 8, 16, 64. int128_t присутствует не везде.

Дробные типы данных

  • float. 32 бита. от 3.4E-38 до 3.4E38
  • [long] double. 64 бита. от 1.7E-308 до 1.7E308

На тип float отводится 32 бита (0..31). В 31 бите хранится знак sign (1 бит), с 23 по 30 бит хранится экспонента exponent (8 бит), с 0 по 22 бит хранится дробная часть fraction (23 бита)

На тип double отводится 64 бита (0..63). В 63 бите хранится знак sign (1 бит), с 52 по 62 бит хранится экспонента exponent (11 бит), с 0 по 51 бит хранится дробная часть fraction (52 бита)

int и float

Даже если int и float имеет одинаковый размер, они будут иметь разный диапазон и разную точность. У float большой диапазон, но низкая точность. У int большая точность, но более узкий диапазон представления чисел. Поэтому при преобразовании из int в float и наоборот может быть потеря точности или переполнение.

Остальные типы данных

  • enum. 32 бита. от -2147483648 до 2147483647

Операции ввода-вывода

В языке Си нет команд работающих с вводом-выводом и с памятью. Все операции ввода-вывода происходят через "поток". "Поток" это абстракция, которая находится между программой и конкретным устройством. Это позволяет программисту не заботиться о взаимодействии программы с конкретными устройствами. При запуске программы автоматически открываются и подключаются к ней 5 стандартных потоков (3 основных и 2 вспомогательных)

  1. stdout - выходной
  2. stdin - входной
  3. stderr - сообщений об ошибках
  4. stdprn - принтера (параллельного порта)
  5. stdaux - последовательного порта

Функция printf()

Функция printf() обеспечивает форматный вывод данных в поток stdout.

Спецификатор формата

В printf спецификатор формата имеет общий вид:

%[флаги][ширина][.точность][размер]тип

Знак % означает - возьми переменную после запятой и распечатай ее значение.

  • Тип - это тип переменной, единственный обязательный параметр спецификатора. Остальные параметры могут быть а могут и не быть. Тип переменной задается после знака % и может быть одним из

    • d, i, u int десятичное целое со знаком
    • o int восьмеричное целое без знака
    • x int шестнадцатеричное целое без знака, использующее abcdef
    • X int шестнадцатеричное целое без знака, использующее ABCDEF
    • f double значение со знаком в формате d.d
    • e double значение со знаком в формате ded
    • E double значение со знаком в формате dEd
    • g double выбирает более компактный формат f или e
    • G double выбирает более компактный формат f или E
    • c int одиночный символ
    • s строка Символы строки печатаются до первого нулевого символа ("\0") или до заданной точности
    • p указатель far на void Печатается адрес, указанный аргументом, в формате xxxx:yyyy, где xxxx - сегмент, yyyy- смещение, а x и y -строчные шестнадцатеричные цифры
  • Размер типа - это l, ll, h, hh
  • ширина - на печать данных отведено ширина символов (или больше).
  • флаги
    • + - указывать явно знак числа, в том числе положительного
    • - - выравнивание влево по указанной ширине
    • 0 - дополнять слева числа до указанной ширины не пробелами, а 0.
  • .точность указывает сколько знаков после точки печатать

Печать строк

Для печати строк используют тип s (string).

Ширина - печатать n или более символов, с выравниванием вправо или влево (отрицательная ширина), дополняя пробелами. Если строка длиннее, то ширина игнорируется.

Точность - если нужно обрезать строку. %.5s - не более 5 символов.

printf("/%5s/\n", "abc"); // / abc/ printf("/%05s/\n", "abc"); // /00abc/ printf("/%-5s/\n", "abc"); // /abc / printf("/%5s/\n", "abcdefgh"); // /abcdefgh/ printf("/%.5s/\n", "abcdefgh"); // /abcde/ printf("/%.5s/\n", "abc"); // /abc/

Функция scanf() обеспечивает форматный ввод данных из потока stdin. Текст внутри будет пропущен. Знак % означает - возьми переменную после запятой и запиши в нее значение определенного размера. Размер значения задается после знака % и может быть одним из

  1. h используется как префикс перед типами d, i, o, x, X для указания того, что тип аргумента short int, или с u для указания того, что тип аргумента short unsigned int
  2. l используется как префикс перед типами d, i, o, x, X для указания того, что тип аргумента long int, или с u для указания того, что тип аргумента long unsigned int. А также перед типами e, E, f, g, G для указания того, что тип аргумента double, а не float

Программа вывода целого числа

Выведем в консоль целое число 100 в десятичной, восьмеричной и шестнадцатеричной системе

#include <stdio.h>

main() {
  printf("%d %o %x", 100, 100, 100);
}
100 144 64

Выведем в консоль целое число в десятичной системе, передав числа в десятичной, восьмеричной и шестнадцатеричной системе

#include <stdio.h>

main() {
  printf("%d %d %d", 100, 0100, 0x100);
}
100 64 256

Программа ввода двух дробных чисел

Введем два числа (через пробел) и запишем их в переменные размером по 4 байта

#include <stdio.h>

main() {
  float a, b;
  scanf_s("%f %f", &a, &b);
  printf("a=%f b=%f", a, b);
}
a=3.000000 b=4.000000

Введем два числа (через пробел) и запишем их в переменные размером по 8 байта

#include <stdio.h>

main() {
  double c, d;
  scanf_s("%lf %lf", &c, &d);
  printf("c=%f d=%f", c, d);
}
c=5.000000 d=6.000000

Введем два числа (через запятую) и запишем их в переменные размером по 8 байта

#include <stdio.h>

main() {
  double c, d;
  scanf_s("%lf,%lf", &c, &d);
  printf("c=%f d=%f", c, d);
}
c=7.000000 d=8.000000

Операция запятая

Операция запятая имеет самый низкий приоритет и выполняется самой последней. Выражения разделенные запятой вычисляются слева направо как части одного выражения. Тип и значение результата совпадают с типом и значением правого выражения.

int c = 3;

Чему равен x?

x = c + 5, 21;

Здесь два выражения разделенные запятой. Первое, в x запишется целое число 8. Второе, 21. Результат второго выражения никуда не записывается. Ответ x = 8

x = (c + 5, 21);

В скобках два выражения разделенные запятой. Первое выражение вернет 8, второе вернет 21. Результат второго выражения запишется в переменную x. Ответ x = 21

Тернарный оператор &:

Тернарный оператор это сокращенная запись конструкции if else.

max = (x > y) ? x: y;
