About

Sass (Syntactically AwesomeStyle Sheets) - это препроцессор CSS. Он помогает придерживаться принципа DRY (Don't Repeat Youself - не повторяйся) и предоставляет компилятор, который позваляет писать в двух разных синтаксисах: SASS и SCSS.

Синтаксис SASS

Это старый синтаксис с отступом, в котором отсутствуют фигурные скобки и точки с запятой. Имеет расширение .sass и напоминает Python.

Синтаксис SCSS

Это более новывй и популярный синтаксис, в котором обязательны фигурные скобки и точки с запятой. Он является подмножеством синтаксиса CSS3. Это позволяет писать обычный CSS с дополнительными функциями.

Установка SASS

Глобальная установка npm пакета sass

npm install -g sass

После установки появится папка

c:\Users\11001000\AppData\Roaming\npm\node_modules\sass\

Проверим установленную версию

sass --version 1.93.2 compiled with dart2js 3.9.3

Ипользование sass

Можно скомпилировать как один файл, таки и все файлы из определенной директории

sass input.scss [output.css] sass input-dir:output-dir

Компиляция файла scss в css

Создадим файл style.scss

nav {
  ul {
    li {
      margin: 0;
    }
  }
}

Преобразуем его в файл style.css

sass style.scss style.css

Компилятор sass создаст новый файл style.css

nav ul li {
  margin: 0;
}

Компиляция файлов scss в css

Создадим папку src, а в ней папку scss. Преобразуем все файлы scss из папки src/scss в папку dist/css. Для этого достаточно указать пути и разделить их двоеточием.

sass src/scss:dist/css

Следить за изменениями файлов sass

Можно изспользовать флаг --watch чтобы следить за изменениями и производить компиляцию после сохранения измененных файлов.

sass –watch style.scss style.css

Можно указать папки для отслеживания изменений и куда сохранять компилированные CSS файлы.

sass --watch src/scss:dist/css

Переменные ($)

В переменных можно хранить цвета, шрифты и другие значения css. При создании и обращении к переменной, ее запись начинается с симовла $.

$font-stack:    Helvetica, sans-serif;
$primary-color: #333;

body {
  font: 100% $font-stack;
  color: $primary-color;
}

Значения переменных $font-stack и $primary-color вставятся там где они встречаются.

body {
  font: 100% Helvetica, sans-serif;
  color: #333;
}

По умолчанию область видимости всех переменных локальная. Добавление переключателя !global делает переменную глобальной, то есть видимой на всех уровнях.

$myColor: red;

h1 {
  $myColor: green !global;
  color: $muColor;
}

p {
  color: $myColor;
}

Заголовок и параграф будут показаны одним цветом - зеленым

h1 {
  color: green;
}
p {
  color: green;
}

Глобальные переменные нужно определять отдельно от локальных переменных, в файле под названием _global.scss. Он включается в основной код с помощью функции @include

Вложенность

В sass можно вкладывать селекторы

nav {
  ul {
    li {
      margin: 0;
    }
  }
}

Получится

nav ul li {
  margin: 0;
}

Родительский селектор (&)

Во вложенном селекторе можно использовать символ & для ссылки на родительский селектор.

a {
  &:hover {
    text-decoration: underline;
  }
}

Получится

a:hover {
  text-decoration: underline;
}

Символ & будет заменен на родительский селектор, каким он компилируется в CSS. Это означает, что если у вас есть глубоко вложенное правило, родительский селектор будет полностью вычислен до символа & и подставится вместо него.

Символ & должен быть указан в начале составного селектора, также он может содержать последующий суффикс, который будет добавлен к родительскому селектору. Например:

#main {
  color: black;

  &-sidebar {
    border: 1px solid;
  }
}

Получится

#main {
  color: black;
}
#main-sidebar {
  border: 1px solid;
}

Шаблоны (@use)

Правило @use используется для подключения шаблонов css кода. Правило @use должно быть написано до любых других правил. При подключения шаблона имя файла пишут без символа подчеркивания и без расширения.

Шаблоны используют чтобы при разработке не писать большие файлы, а разбивать их на несколько маленьких. При этом маленькие файлы не будут компилироваться в отдельные css файлы, а будут включаться внутрь тех файлов, которые подключают эти шаблоны. Для этого нужно в названии файла шаблона первым символом использовать символ подчеркивания. Символ подчеркивания указывает SASS что это файл - частичный шаблон и компилировать отдельно его не нужно.

Файл _reset.scss с кусочком кода

body {
  margin: 0;
  padding: 0;
}

Файл base.scss куда нужно вставить шаблон

@use 'reset';

body {
  font-family: Helvetica;
  background-color: #efefef;
}

Получится один файл base.css

body {
  margin: 0;
  padding: 0;
}

body {
  font-family: Helvetica;
  background-color: #efefef;
}

Миксины (@mixin @include)

Миксины позволяют создавать группы деклараций CSS, которые можно использовать по нескольку раз. Можно передавать переменные в миксины, чтобы сделать их более гибкими. Так же хорошо использовать миксины для вендорных префиксов. Пример для transform:

@mixin transform($property) {
  -webkit-transform: $property;
  -ms-transform: $property;
  transform: $property;
}
.box { @include transform(rotate(30deg)); }

Получится

.box {
  -webkit-transform: rotate(30deg);
  -ms-transform: rotate(30deg);
  transform: rotate(30deg);
}

Наследование (@extend)

Использование @extend позволяет передавать наборы свойств CSS от одного селектора другому.

%message-shared {
  border: 1px solid #ccc;
  padding: 10px;
  color: #333;
}

.message {
  @extend %message-shared;
}

.success {
  @extend %message-shared;
  border-color: green;
}

.error {
  @extend %message-shared;
  border-color: red;
}

Получится

.error, .success, .message {
  border: 1px solid #ccc;
  padding: 10px;
  color: #333;
}

.success {
  border-color: green;
}

.error {
  border-color: red;
}

Математические операторы (+ - * %)

Использовать математику в CSS очень полезно. Sass имеет несколько стандартных математических операторов, таких как +, -, *, %.

div {
  width: 100px + 200px * 2 - 300px;
}

Получится

div {
  width: 200px;
}

Функции (@function)

Встроенные функции позволяют нам выполнять вычисления возвращають значение

@use "sass:math";

@function pxToRem($pxValue) {
  @return math.div($pxValue, 16px) * 1rem;
}

div {
  width: pxToRem(480); // возвращает 30rem
}

Условный оператор (@if @else)

Можно написать условную конструкцию используя операторы @if и @else. Пример миксина с условным оператором

@mixin body-theme($theme) {
  @if $theme == "light" {
    background-color: $light-bg;
  } @else {
    background-color: $dark-bg;
  }
}

Комментарии

Стандартный многострочный CSS комментарий, обозначаемый /* */ сохранится на выходе в файле CSS

Однострочный комментарий, обозначаемый // в выходной файл CSS не попадет

Устаревший синтаксис

К устаревшиму синтаксису относятся:

  1. Математический оператор деления /
  2. Подключение шаблонов с помощью @import

Математический оператор деления

Математический оператор деления / устарел и будет удален в Dart Sass 2.00. Вместо него нужно подключить библиотеку sass:math и использовать функцию math.div.

@use "sass:math";

div {
  width: math.div(100px, 20px) + 100px;
}

Получится

div {
  width: 105px;
}

Импорт

Вместо @use раньше использовался @import, который устарел и будет удален в Dart Sass 3.00.
