info

Всего в HTML 110 элементов - 2 без тегов (доктайп, комментарий), 15 с одиночным тегом, 93 с парными тегами.

Элемент с одиночным тегом называется пустым (empty element). Он не имеет содержимого и закрывающего тега, а состоит только из открывающего тега.

Элемент с парными тегами состоит из открывающего тега (start tag), содержимого (content) и закрывающего тега (end tag).

Семантические теги (15)

<address> <article> <aside> <bdi> <figcaption> <figure> <footer> <header> <main> <mark> <nav> <section> <time> <wbr>

Семантические теги появились в HTML5. До их появления вся разметка веб-страниц строилась при помощи элементов <div>, которым присваивались идентификаторы (id) или классы (class). Они нужны для группировки контента и разметки текстового содержимого. Они четко описывают, какой контент содержат.

Элемент <address> определяет контактную информацию автора/владельца документа или статьи.

Элемент <article> определяет независимый блок, в котором группируются записи (статьи, публикации в блоге, комментарии и т.д).

Элемент <aside> oн определяет раздел, в котором содержится информация, дополняющая основной отдел. Эта информация является факультативной, и если ее удалить, основное содержимое не пострадает. Такой информацией могут быть комментарии, справочная информация, списки терминов, коллекция ссылок и т.д.

Элемент <header> формирует заголовочный блок или “шапку” документа, используется как контейнер для группировки вводных и навигационных элементов (логотип, навигационное меню, форма поиска и т.д.).

Элемент <bdi> определяет фрагмент текста двунаправленного форматирования, который должен быть изолирован от остального текста.

Элемент <figcaption> используется для добавления подписи или пояснений к содержимому тега <figure>.

Элемент <figure> определяет автономный контент, являющийся самостоятельным элементом основного потока. Используется для добавления кратких аннотаций к иллюстрациям, фотографиям, диаграммам, фрагментам кода и т.д.

Элемент <footer> формирует нижний колонтитул секции или корневого элемента, в котором находится. Как правило, в нем размещается информация об авторских правах, контактные данные, ссылки для навигации и т.д.

Элемент <main> формирует основное содержимое документа (содержимое элемента <body>). Содержимое тега должно быть уникальным и не дублировать однотипные блоки, которые повторяются в других документах, таких как шапка сайта (header), нижний колонтитул (footer), меню, форма поиска, информация об авторских правах и т.д.

Элемент <mark> используется для выделения важного содержимого или ключевых слов. По умолчанию выделяет желтым цветом.

Элемент <nav> формирует основной блок навигационных ссылок. Обратите внимание, что не все ссылки в HTML документе могут быть помещены внутрь элемента <nav>, он может включать в себя только крупные навигационные блоки.

Элемент <section> используется для группировки тематического содержимого (оглавление, программа, разделы и т.д.).

Элемент <time> определяет время (24 часа) или дату по григорианскому календарю с возможным указанием времени и смещения часового пояса.

Элемент <wbr> указывает браузеру место возможного разрыва длинной строки в случае необходимости.

Тип документа <!DOCTYPE>

Пишется первой строчкой любого html документа. Для страниц стандарта html5 имеет вид <!DOCTYPE html>

Элемент <!DOCTYPE> предназначен для указания типа текущего документа — DTD (document type definition, описание типа документа). Это необходимо, чтобы браузер понимал, как следует интерпретировать текущую веб-страницу. Поскольку HTML существует в нескольких версиях, существует несколько видов <!DOCTYPE>

Комментарий <!--...-->

Браузер игнорирует все что написано внутри. Таким образом можно оставлять комментарий к коду или экранировать фрагменты кода. Вложенные комментарии (один комментарий в другом) не поддерживаются. <!-- какой-то комментарий к коду -->

Если комментарий расположен внутри тега <title>, он трактуется как текст и будет показан в заголовке страницы. Проще говоря, тег <!-- --> внутри тега <title> не работает.

Базовые элементы (2)

<head> <body>

В корневой элемент <html> последовательно вкладывают базовые элементы <head> и <body> <!DOCTYPE html>
<html>
  <head>

<head>

Содержимое тега <head> не отображается напрямую на веб-странице, за исключением тега <title> устанавливающего заголовок окна веб-страницы. Внутри контейнера <head> находятся метатеги, которые используются для хранения информации предназначенной для браузеров и поисковых систем. Внутри контейнера <head> допускается размещать следующие элементы: <base>, <link>, <meta>, <script>, <style>, <title> (устаревшие <basefont>, <bgsound>).

<body>

В теге <body> находится все содержимое страницы, которое показывается в окне браузера.

Document Metadata (Метаданные) (5)

<meta /> <base /> <link /> <title> <style>

<meta />

Всегда внутри контейнера <head>. Данные в нем не видны пользователю и предназначены для браузеров и поисковых систем.

<base />

Всегда внутри контейнера <head>. Сообщает браузеру полный базовый адрес текущего документа. В атрибут href пишут адрес, который должен использоваться для указания полного пути к файлам. <base /> предназначен для документов, в которых используется относительный адрес и эти документы могут переноситься в другую папку или даже на другой компьютер без потери связи.

Второе применение элемента — задание целевого окна для всех ссылок на текущей странице. В атрибут target пишут имя окна, в котором будут открываться все ссылки.

Если на странице встречается несколько элементов , то берётся значение атрибутов только у первого элемента, остальные игнорируются.

<!DOCTYPE html>
<html>
  <head>
    <base href="https://html5css.ru/images/" target="_blank">
  </head>
  <body>
    <img src="spiderman.gif" width="50" height="100" alt="Spiderman">
    <a href="spiderman.html">Человек-паук</a>
  </body>
</html>

<link />

Всегда внутри контейнера <head>. Устанавливает связь с внешним документом вроде файла со стилями, со шрифтами или favicon.

<!DOCTYPE html>
<html>
  <head>
    <link rel="preload" href="https://htmlacademy.ru/fonts/roboto.woff2" as="font"">
    <link rel="stylesheet" href="style.css">
    <link rel="icon" href="favicon.ico">
  </head>
  <body>
  </body>
</html>

Чтобы сделать предзагрузку шрифта на страницу, нужно в теге <head> разместить тег <link />, в аттрибуте rel прописать значение preload, в аттрибуте href прописать адрес шрифта, в аттрибуте as прописать тип ресурса, в данном случае font.

Headings (Заголовки) (7)

<hgroup> <h1> <h2> - <h6>

HTML предлагает шесть заголовков разного уровня, которые показывают относительную важность секции, расположенной после заголовка. <h1> представляет собой наиболее важный заголовок первого уровня. Считается что он должен быть один на странице. Отображается самым крупным шрифтом жирного начертания, заголовки последующего уровня по размеру меньше. Это блочные элементы. Перед заголовком и после него добавляется пустое пространство.

<hgroup>

Представляет заголовок раздела. Группирует несколько заголовков. Связывает вторичный заголовок или даже теги с основным заголовком без загрязнения структуры документа

<hgroup>
  <h1>Главный заголовок</h1>
  <h2>Вторичный заголовок</h2>
</hgroup> <hgroup id="document-title">
  <h1>HTML: Living Standard</h1>
  <p>Last Updated 12 July 2022</p>
</hgroup>

Гиперссылка <a>

Тег a создает ссылку на другую страницу своего или чужого сайта. Адрес ссылки следует писать в атрибуте href. Ссылки по умолчанию подчеркнуты. Отменить подчеркивание можно с помощью CSS свойства text-decoration.

Атрибуты

href Адрес страницы, на которую ссылается ссылка

target Указывает ссылке в какой вкладке ей открываться

Значения атрибута target (Значение по умолчанию: _self)

_blank Страница откроется в новой вкладке браузера.

_self Страница откроется в текущей вкладке браузера.

_parent Страница откроется во фрейме-родителе.

_top Отменяет все фреймы и загружает страницу в полном окне браузера.

Obsolete (Устаревшие) (26)

Осуждаемыми (устаревние) тегаи удалены из спецификации HTML как морально устаревшие. Вместо них рекомендуется использовать другие теги или стили. Хотя браузеры в большинстве случаев прекрасно понимают осуждаемые теги, код с ними проходит валидацию, только при использовании переходного <!DOCTYPE>.
