1.1 Вид программы C++Builder

Для прохождения курса будем использовать C++Builder 6.

В верхней части окна программы расположена строка заголовка. Она обычно содержит название программы и имя открытого файла. Также на ней находятся кнопки Свернуть, Восстановить, Закрыть окно.

Под заголовком находится строка меню, которая предоставляет доступ ко всем функциям и командам программы.

Под главным меню находятся быстрые кнопки, объединенные в группы по назначению. Они позволяют получить быстрый доступ к наиболее часто используемым командам.

Рядом с быстрыми кнопками расположена палитра компонентов. Она состоит из нескольких закладок, на которых располагаются визуальные компоненты, используемые при создании программ в среде C++Builder. При малом разрешении экрана все эти закладки не умещаются в пределах рабочей области программы и становятся не доступны. Чтобы их увидеть надо воспользоваться кнопками прокрутки, которые расположены в правой верхней части палитры компонентов.

В левой части экрана располагается Object Inspector - инспектор объектов. В этом окне производится настройка основных свойств визуальных компонентов.

Справа от окна инспектора объектов распологается окно дизайнера форм. Это окно содержит строку заголовка, в котором отображается название формы, и кнопки Свернуть, Восстановить, Закрыть окно.

Под ним располагается окно редактора кода, в котором производится набор кода программы.

1.2 Главное меню

Главное меню предоставляет доступ к основным командам программы C++Builder, объединенным в следующие группы:

  1. File. Команды этого меню осуществляют работу с файлами и предоставляют досуп к следующим операциям: создание новых форм и приложений, открытие файлов и проектов, сохранение файлов и проектов, печать текстов программ и добавление заголовков модулей.
  2. Edit. В этом меню собраны команды редактирования, такие как отмена Undo, повторение Redo операций, копирование Copy и удаление Delete. А также команды выравнивания компонентов и настройки редактора кода.
  3. Search. Команды данного меню позволяют осуществить поиск текста в файле, продолжить поиск после первого вхождения, произвести автоматическую замену, а также быстро перейти к строке кода, задав ее номер.
  4. View. Из этого пункта меню вызываются основные диалоговые окна управления проектом и компонентами. Такие как Project Manager менеджер проектов, Component List список компонентов и Window List список окон. Также из этого пункта меню открываются все окна отладки программы Debug Windows, работу с которыми мы рассмотрим позже.
  5. Project. В этом меню собраны команды управления проектом. С иъ помощью можно добавлять и удалять модули, добавить библиотеку компонентов, откомпилировать проект и т.д.
  6. Run. С помощью команд этого меню выполняется запуск программ в различных режимах.
  7. Component. Из этого меню вызываются команды добавления в систему новых компонентов.
  8. Database. Команды для работы с базами данных.
  9. Tools. В этом меню собраны команды как по настройке параметров программы, так и команды вызова различных дополнительных утилит.
  10. Window. С помощью команд данного меню производится управление окнами программы.
  11. Help. Команды, с помощью которых можно получить справку как о языке C++, так и о работе с программой и всеми ее компонентами.

2.3 Быстрые кнопки

Быстрые кнопки расположены под главным меню. С их помощью осуществляется доступ к основным часто используемым командам, разделенным на следующие группы:

  1. Standart. Кнопки создания новых объектов программы. При нажатии на кнопку New появится окно диалого New Items, в котором необходимо выбрать нужный объект. Также кнопки этой группы позволяют сохранить Save и открывать Open как отдельные файлы так и целый проекты. Замыкают группу две кнопки управления проектами, с помощью которых можно добавлять файлы к проекту Add file to project и удалять их из него Remove file from project.
  2. View. В эту группу собраны кнопки, управляющие просмотром форм и модулей, а также кнопка создания новой формы. Для создания новой формы надо нажать кнопку New Form, появится новое окно дизайнера форм, в заголовке которого будет отображаться название новой формы. Для переключения между формами и редактором кода служит кнопка Toggle Form/Unit. Если в вашей программе используется несколько форм, то с помощью кнопки View Forms вы можете выбрать форму для просмотра.
  3. Debug отладка. В этой группе собраны кнопки с командами для отладки программы. Кнопка Run запускает проект на выплнение. Кнопка Pause позволяет приостановить выполнение проекта. С помощью кнопок Trace into и Step over задается запуск программ в пошаговом режиме для детальной отладки программы

Положение кнопок быстрого доступа в среде разработки не фиксированное. Их можно разместить на главной панели по своему усмотрению. Для этого необходимо подвести курсор к левому краю группы. Нажав и удерживая левую кнопку мыши, переместить группу на новое место. Если группа вынесена за пределы главной панели, ей отводится отдельное окно.

1.4 Палитра компонентов

Методика визуального построения приложений реализуется в пакете C++Builder посредством выбора необходимого компонента и помещения его на окно формы. Компоненты являются кирпичиками, из которых выстраивается приложение. Для доступа ко всем компонентам предназначена палитра компонентов, расположенная справа от кнопок быстрого доступа. Палитра компонентов состоит из закладок на которых размещаются компоненты, разделенные на группы по их назначению. При переходе от одной закладке к другой, набор доступных компонентов меняется. На некоторых закладках все компоненты данной группы не умещаются в пределах рабочей области программы. Для того чтобы получить к ним доступ надо нажать кнопку прокрутки в начале или в конце закладки.

Компоненты в палитре изображены в виде кнопок, при наведении курсора на одну из них, она приподнимается и под курсором появляется строка, в которой выводится название компонента. После этого, для помещения компонента на форму надо нажать на кнопку компонента, затем переместить курсор на окно дизайнера форм и щелкнуть мышью.

Палитру компонентов можно как и другие элементы главной панели убрать с экрана. Для этого надо подвести курсор к свободной зоне панели и нажать правую кнопку мыши. Появится выпадающее меню, в котором надо щелкнуть на названии Component Palette. Для перемещения палитры в другое место рабочей области программы надо нажать левую кнопку мыши в левой части палитры. Удерживая кнопку переместить палитру в новое место. Если она при этом выйдет за пределы главной панели, то ей будет отведено отдельное окно.

Если нажать правую кнопку мыши на свободном месте палитры, то появится выпадающее меню, в котором содержится команда вызова окна настроек Properties. В окне настроек Palette Properties палитры компонентов можно настроить каждую из закладок и всю палитру в целом. Например, для того чтобы закладка Dialogs следовала сразу за Standart надо выполнить следующие действия. Вызовите окно свойств палитры компонентов. Щелкните мышью на строке Dialogs. Нажимая кнопку Move Up, переместите строку вверх в нужное место. После этого нажимаем кнопку OK. Теперь закладка Dialogs располагается сразу за закладкой Standart.

1.5 Дизайнер форм

Дизайнер форм предназначен для визуального размещения компонентов на рабочей области вашего приложения и задания размеров самой рабочей области. В верхней части окна дизайнера форм находится строка заголовка, в которой располагается название формы и стандартные кнопки закрытия и масштабирования окна.

Вся рабочая область окна занята сеткой привязки, которая отображается в виде точек. При помещении компонента на форму, его положение автоматически привязывается к сетке. После этого, при перемещении объекта мышью, он сдвигается на целое значение деления сетки.

Для того чтобы задать шаг сетки или вообще отключить ее, надо вызвать окно диалога Tools - Environment Options. В открывшемся окне Environment Options выбрать закладку Preferences. В закладке расположена группа параметров Designer, в которой с помощью установки флажков можно изменить следующие параметры: Display Grid отображать или не отображать сетку привязки. Snap to grid привязка к сетке включена или отключена. В полях Grid size x и y задаются величины шага сетки привязки по осям координат. Если надо переместить компонент на форме на меньшее расстояние чем шаг привязки, то надо выделить компонент, щелкнув левой кнопкой мыши на его изображении, зажав клавишу Ctrl переместить изображение стрелками на клавиатуре.

1.6 Редактор кода

Редактор кода представляет собой надежное и удобное средство для просмотра и редактирования исходного кода программных модулей. Каждый модуль состоит из двух частей: интерфейсной с расширением .h и кодовой с расширением .cpp. Также для каждого модуля предусмотрена страница Diagram, назначение которой мы рассмотрим позже.

Окно редактора кода может содержать несколько именованных закладок. В зависимости от настроек редактора кода, в его составе может отображаться окно просмотра сообщений. Для того чтобы его включить, подведите курсор к верхней части окна, нажмите правую кнопку мыши. На экране отобразится дополнительное меню. В котором недо выбрать Message View.

Основную часть редактора кода занимает окно, в котором происходит редактирование и просмотр модулей. По левому краю окна проходит поле подшивки Gutter, используемое для расстановки закладок и точек останова. Все настройки редактора кода выполняются в окне Editor Properties. Для его вызова надо нажать правую кнопку мыши на рабочей области редактора и выбрать пункт Properties. Окно настроек Editor Properties содержит следующие закладки. General - здесь настраиваются режимы форматирования текста и схемы раскладки клавиатуры. Display - здесь устанавливается форма курсора, ширина полей отступа и подшивки, а также шрифт редактора. Key Mappings - здесь определяются действующие наборы раскладок клавиатуры. Colors содержит параметры, управляющие цветом и типом шрифта для выделения синтаксических конструкций языка C++. Code Insight содержит параметры, управляющие суфлером кода и настройкой шаблонов.

1.7 Инспектор объектов

Инспектор объектов предназначен для редактирования свойств объектов на стадии разработки программы. Тем самым определяя начальное состояние компонента. Окно инспектора разбито на три основные части. В верхней части расположено поле селектора объектов, которое отображает название выбранного компонента. Для выбора другого компонента надо открыть выпадающий список. После этого свойства объекта становятся доступны для редактирования. Под полем селектора объектов есть две закладки.

Закладка свойства Propetries - здесь расположены свойства выбранного компонента. Закладка разбита на два поля. В левой части отображаются названия свойств, а в правой - его значения. Свойство может содержать вложенные свойства. Для их просмотра и редактирования надо нажать на плюс перед названием свойства.

Закладка события Events содержит события на которые может реагировать компонент и позволяет связать его с происходящими событиями в процессе работы программы. Данная закладка разбита на две части. Слева отображаются названия событий на которые умеет реагировать компонент. Справа отображаются названия методов их обработки. Для создания обработчика надо дважды щелкнуть в правом поле напротив выбранного события. При этом появляется окно редактора кода, в котором автоматически создастся функция.

Если нажать на свободном поле инспектора объектов правую кнопку мыши, то появится контекстное меню, в котором можно настроить вид инспектора. Работу с основными свойствами и событиями мы рассмотрим позже.

1.8 Открытие и сохранение проекта.

Перед началом использования среды программирования C++Builder, надо научиться работать с файлами. Для полноценной работы нужно уметь открывать и сохраять файлы. Работая с программой, вы произведете изменения в проекте, которые надо сохранить на жесткий диск. Можно сохранить проект целиком или сохранить только отдельный файл.

Для сохранения всего проекта выберите пункт меню File - Save Project As. Откроется окно Save As, где нужно выбрать место и сохранить проект. Для сохранения одного файла выберете пункт меню File - Save As. Откроется окно Save As, где нужно выбрать место и сохранить файл.

Для открытия проекта выберите пункт меню File - Open Project. Откроется окно Open. Выберите файл проекта и нажмите кнопку Открыть. Для открытия файла выберите пункт меню File - Open. Откроется окно Open. Выберите файл и нажмите кнопку Открыть.

1.9 Команды контекстного меню.

При работе с программой все ее команды и функции доступны из главного меню или из панели быстрых кнопок. Но некоторые команды можно вызвать из контекстного меню, которое появляется при щелчке правой кнопки мыши в рабочей области программы. Набор команд контекстного меню зависит от места его вызова. Рассмотрим основные команды контекстного меню.

Наведите курсор на дизайнер форм и вызовите контекстное меню. В верхней части есть пункт Position, который вызывает дополнительное меню. Команды данного меню позволяют управлять размерами и положением создаваемой формы. При вызове одной из команд появляется диалоговое окно, в котором можно задать все параметры команды.

Команда View as Text позволяет просотреть файл формы в текстовом виде. Для того чтобы переключиться к начальному отображению формы, необходимо вызвать данную команду повторно. При втором вызове она называется View as Form.

Теперь переключитесь на редактор кода и вызовите контекстное меню. Команды в нем разбиты на несколько групп. Верхняя группа команд предназначена для управления файлами проекта. С их помощью можно открывать или сохранять модули. В следующей группе команды для работы с буфером обмена. Следующая группа с выпадающими списками для работы с закладками. Следующая группа меню для работы с отладкой программы. Последнее меню Properties открывает окно для редактирования свойств редактора кода.

1.10 Распечатать код на принтере.

При работе с программой можно распечатать код на принтере. Для этого нужно настроить принтер. Выберите пункт меню File - Print. В открывшемся окне Print Selection можно настроить параметры печати.

На вкладке File to print выбираются файлы для печати. Чтобы выбрать файл, надо установить флажок перед его названием. Если в проекте только один файл, то установка флажка не требуется.

На вкладке Options располагаются дополнительные флажки для параметров печати специальных полей: Line numbers номера строк кода, page number номера страниц. Нажмите кнопку Setup и на экране появится окно Настройка принтера.

На вкладке принтер выбирается принтер для печати. После выбора отобразится состояние устройства. Если нажать Свойства, то откроется окно Свойства, где можно настроить параметры принтера. На вкладке Бумага выбирают размер страныцы, ориентация книжная или альбомная. Когда все параметры уже заданы, нажмите кнопку ОК. Закроется окно Свойства и вернется окно Print Selection. Для начала печати нажмите кнопку ОК.

Урок 2

2.1 Закладка Standart

На закладке Standart располагаются компоненты стандартных интерфейсных элементов имеющихся во всех версиях операционной системы Windows.

  1. Frames кадры - это контейнер для размещения других компонентов. Main menu главное меню создает главное меню программы.
  2. Popup Menu всплывающие меню для создания всплывающих меню формы или другого компонента.
  3. Label метка - отображает на форме текстовую метку, которую нельзя редактировать.
  4. Edit редактируемый ввод - отображает на форме область для ввода одиночной строки.
  5. Memo комментарий - отображает на форме множество редактируемых строк.
  6. Button кнопка - создает на форме прямоугольную кнопку с надписью. Стандартная кнопка неспособна отображать надпись в несколько строк.
  7. CheckBox ячейка состояния - создает прямоугольную ячейку с двумя состояниями и строкой пояснения.
  8. RadioButton радио кнопка - создает круглую кнопку с двумя состояниями и текстом, который поясняет ее назначение.
  9. ListBox простой список - создает прямоугольную область для отображения текстовых строк с возможностью их выбора, добавления или вычеркивания.
  10. ComboBox комбинированный список - комбинация строки ввода и выпадающего списка для выбора.
  11. ScrollBar линейка прокрутки - создает линейку с бегунком и кнопками для прокрутки окна. А также для перемещения в пределах интервала некоторого заанного параметра.
  12. GroupBox группирующая рамка служит для создания области визуально объединяющей несколько интерфейсных элементов.
  13. RadioGroup группа радиокнопок создает на форме контейнер в виде прямоугольной рамки для объединения группы взаимоисключающих радио кнопок.
  14. Panel простая панель создает пустую область, на которой можно разместить другие компоненты. Обычно используется для создания панелей инструментов в программе.
  15. ActionList список действий осуществляет управление взаимодействием между интерфейсными элементами и логикой программы.

2.2 Закладка Editional

Компоненты на закладке Editional осуществляют подключение к программе дополнительных интерфейсных элементов, разработанных для системы Builder. С помощью этих компонентов можно создать более привлекательный интерфейс. На закладке расположены следующие элементы:

  1. BitBtn графическая кнопка создает стандартизованную кнопку с изображением и надписью.
  2. SpeedBtn быстрая кнопка создает кнопку с битовым образцом без надписи. Обычно используется для создания панели инструментов.
  3. Mask Edit форматированный ввод создает прямоугольную область ввода данных в специфическом формате. Корректность вводимого текста проверяется с помощью маски.
  4. StringGrid текстовая таблица создает сетку для отображения текстовых последовательностей по строкам и столбцам.
  5. DrawGrid графическая таблица создает двумерную таблицу для отображения графических данных.
  6. Image изображение создает невидимый контейнер, в который можно поместить один графический файл. Можно вставлять битовые образы, пиктограммы и метафайлы.
  7. Shape фигура рисует простые геометрические фигуры такие как окружность, элипс, квадрат или прямоугольник. Можно менять цвет фигур и их штриховку.
  8. Bevel обрамление создает объемные рамки для объектов. Является визуальным компонентом и не несет программных функций.
  9. ScrollBox контейнер прокрутки создает контейнер автоматически снабжаемый линейками прокрутки, если помещенные в контейнер объекты не умещаются в его рабочей области.
  10. CheckListBox список с ячейками отображает список элементов у каждого из которых есть собственная ячейка состояния.
  11. Splitter разделитель разделяет рабочую область программы на две части изменяемой ширины и дает возможность менять их размеры во время выполнения программы.
  12. Chat диаграмма для создания и вывода на печать многоцветных графиков и диаграмм. Данный компонент полностью заимствован из Delphi.

2.3 Закладка Win32

Компоненты данной закладки обеспечивают подключение интерфейсных элементов, используемых в 32 разрядных версиях операционной системы Windows.
