Мы будем программировать в программе CodeBlocks 20.03. Любая программа на C++ всегда начинается с функции main(). Напишем первую программу - вывод приветствия.

#include 

using namespace std;

int main() {
  cout << "Hello world";
  return 0;
}

Выведем приветствие в две строки

#include 

using namespace std;

int main() {
  cout << "Hello world" << endl;
  cout << "Hello again";
  return 0;
}

Выведем текст, число и математическое выражение

#include 

using namespace std;

int main() {
  cout << "2 + 2" << endl;
  cout << 2 << endl;
  cout << 2 + 2 << endl;
  return 0;
}

В спортивном программировании для перевода строки используется символ '\n', а не endl. Потому что endl кроме перевода строки еще сбрасывает буфер, а это лишние операции, на которые тратится время.

Попробуем подключить русский язык. Сам текст программы, содержащий русский язык, должен быть сохранен в кодировке windows-1251.

#include 

using namespace std;

int main() {
  cout << "Привет" << endl;
  setlocale(LC_ALL, "Russian");
  cout << "Привет" << endl;
  return 0;
}

Типы данных integer (целое число)

Числа со знаком (signed)

В переменных signed можно хранить отрицательные и положительные числа, и ноль. Слово signed перед типом данных можно не писать.

  • short 2 байта. Запишем число 10 в тип данных short

    10 = 00000000 00001010

  • int 2, 4, 8 байт в зависимости от архитектуры процессора

  • long 4 байта. Запишем число 10 в тип данных long

    10 = 00000000 00000000 00000000 00001010

  • long long 8 байт. Запишем число 10 в тип данных long long

    10 = 00000000 00000000 00000000 00000000 00000000 00000000 00000000 00001010

Числа без знака (unsigned)

В переменных unsigned можно хранить тоьлко положительные числа и ноль.

  • unsigned short
  • unsigned int
  • unsigned long
  • unsigned long long

Целые типы данных точного размера

Есть типы данных, которые содержат точное значение байт. У меня эти типы данных работают с библиотекой cstdint.

  • int8_t 1 байт
  • int16_t 2 байта
  • int32_t 4 байта
  • int64_t 8 байт
  • uint8_t 1 байт
  • uint16_t 2 байта
  • uint32_t 4 байта
  • uint64_t 8 байт

int8_t выводится как символ. Поэтому при выводе его нужно приводить к типу данных int.

#include 
#include 

using namespace std;

int main() {
  int8_t num = 70;
  cout << num << '\n';
  cout << int(num) << '\n';
  return 0;
}

Переполнение типа данных int

Для каждого целого типа данных есть минимальное и максимальное число, которое можно записать используя этот тип данных. Если число не попадает в диапазон от минимального до максимального то происходит переполнение.

Для типа данных uint8_t максимальное число 255. Увеличим его и посмотрим что будет записано.

#include 
#include 

using namespace std;

int main() {
  uint8_t num = 255;
  cout << int(num) << '\n';
  num++;
  cout << int(num) << '\n';
  return 0;
}

Символьный тип данных

Символьный тип данных занимает 8 байт и может быть записан в один из следующих типов:

  • char
  • unsigned char
  • int8_t
  • uint8_t

С символьным типом данных можно обращаться так же как и с типом данных int, поскольку в нем записаны числа. Однако при выводе на печать будет выводиться символ в кодировке ASCII.

char от -128 до +127

unsigned char от 0 до +255

#include 

using namespace std;

int main() {
  char x = -128;
  for (int i = 0; i < 256; i++) {
    cout << int(x) << ' ' << x << '\n';
    x++;
  }

  unsigned char y = 0;
  for (int i = 0; i < 256; i++) {
    cout << int(y) << ' ' << y << '\n';
    y++;
  }

  return 0;
}

++i и i++

Постфиксная операция инкремента i++ выполняется на несколько ассемблерных команд длинее префиксной операции инкремента ++i. Но если компилятор видит, что возвращаемое значение не используется, то он оптимизирует и i++ сам заменяет на ++i. Именно поэтому нет никакой разницы в коде, что вы напишите ++i или i++, там где они взаимозаменяемы. Они будут выполняться одинаково быстро.

#include 

using namespace std;

int main() {
  for (int i = 0; i < 10; i++) {
    cout << i << '\n';
  }

  for (int i = 0; i < 10; ++i) {
    cout << i << '\n';
  }

  return 0;
}
