Лекция 1. Команды и первая программа на C++

1. Знакомство с командами

Если когда-то вы представляли работу программиста как что-то непостижимое и таинственное, пришла пора развенчать этот миф! Программирование — это не секретное искусство, а шикарная и очень интересная работа. И сейчас вы в этом убедитесь.

Компьютерные программы пишут при помощи языков программирования — особых правил и слов, понятных компьютеру. Сегодня вы познакомитесь с языком C++: напишете свою первую программу, разберете, что такое команды, и заставите компьютер слушаться вас (насколько он захочет, конечно).

Программа — это набор (список) команд. Сначала выполняется первая команда, затем вторая, третья, и так далее. Когда все команды выполнены, программа завершается.

Какие именно команды могут быть в списке команд, зависит от того, кто их выполняет, а точнее, от того, какие команды знает (и понимает) исполнитель. Собаке можно дать команду «Сидеть», «Голос», кошке — «Брысь», человеку — «Стой!», ну а роботу — «Работай!».

Команда — это простое указание компьютеру: сделай то-то! Самое главное — компьютер выполняет именно то, что вы ему сказали, а не то, что вы хотите. Пока искусственный интеллект не подрастет, конечно.

Набор команд языка C++ довольно обширен. Например, этой командой можно вывести на экран надпись "Робот — друг человека":

std::cout << "Робот — друг человека" << '\n';

Вывод текста в консоль с помощью std::cout

Но мы начнем не с команд, а с пары простых принципов. Знание нескольких принципов заменяет знание многих фактов.

Принцип первый: в языке программирования C++ каждую команду принято писать с новой строки. В конце команды ставится точка с запятой ;.

Допустим вы хотите, чтобы компьютер вывел вашу фразу три раза, дайте ему команду несколько раз.

Программа из трех команд:

std::cout << "Робот — друг человека" << '\n'; std::cout << "Робот — друг человека" << '\n'; std::cout << "Робот — друг человека" << '\n';

Три команды подряд (выполняются сверху вниз)

В результате компьютер выведет на экран фразу 3 раза. Все просто.

Принцип второй: программа не может просто состоять из команд. Команды языка C++ должны находиться внутри функций, а функции — внутри файлов. Это очень интересно - давайте подробнее.

2. Структура типичной программы

Любая программа — это набор команд, но компьютеру нужно знать, с какой команды начать. В C++ точка входа почти всегда одна: функция main. Это как кнопка «Старт» у микроволновки: можно долго выбирать режимы, но пока не нажал — ничего не происходит.

Минимальная программа - это один файл, который содержит одну фукнцию - main() и выглядит вот так:

int main() {
  return 0;
}

Здесь есть три важных элемента: слово int, имя main и блок { ... }. Давайте разберём каждый — спокойно и без магии.

Слово int — это тип результата функции. Пока не пугайтесь слова «тип»: сейчас нам важно только одно — main должен вернуть какое-то число, и это число называют кодом завершения. Мы будет всегда возвращать 0.

Круглые скобки () означают, что это функция. Они обязательно нужны после слова main, иначе C++ не догадается что это та самая функция.

Фигурные скобки { ... } задают блок кода — контейнер для команд, которые выполняются последовательно. Для main блок обязателен: это буквально «тело программы».

Полезная привычка на старте — всегда оформлять main так, чтобы скобки стояли аккуратно и читались без напряга:

int main() {
  команда1;
  команда2;
  команда3;
  return 0;
}

Шаблон: команды внутри тела функции main()

Смысл такой: всё, что внутри {} — выполняется

Лекция 2. Вывод на экран в C++: std::cout и перевод строки

1. Знакомство с командой std::cout

Вы только что написали свою первую программу на C++ и вывели на экран первое сообщение. Поздравляю вас с этим. Давайте же не останавливаться...

Любая программа на C++ состоит из команд (инструкций). Команды бывают разные. В языке C++ есть команды на все случаи жизни. Каждая команда описывает какое-то определенное действие. В конце каждой команды ставится точка с запятой.

Для вывода на экран в C++ обычно используют поток вывода std::cout.

Примеры команд:

std::cout << 1 << '\n'; // Выводит на экран число: 1 std::cout << "Amigo" << '\n'; // Выводит на экран надпись: Amigo std::cout << "Risha & Amigo" << '\n'; // Выводит на экран надпись: Risha & Amigo

Здесь используется объект std::cout и оператор << (читается как «вывести в поток»). Чтобы завершить строку, обычно добавляют перевод строки: '\n'.

Размер букв имеет значение

В C++ размер имеет значение: важно, большими или маленькими буквами написаны имена. std::cout работать будет, а std::Cout — нет. Если вы напишете что-то не теми буквами, компилятор C++ это не распознает, и следовательно выполнять не будет. Так что будьте внимательными.

Двойные кавычки

Если вы хотите вывести на экран текст, его нужно с двух сторон обозначить двойными кавычками.

Одинарная кавычка выглядит вот так ', а двойная — вот так ". Двойная кавычка — это не две одинарных: просьба не путать.

Двойная — это та, которая рядом с кнопкой Enter. На ней еще обычно находится кириллическая буква "Э".

2. Знакомство с выводом без перевода строки

Если вам нужно вывести текст без перехода на новую строку, в C++ просто не добавляйте '\n' в конце.

Команда с '\n' выводит текст и после него добавляет перевод курсора на следующую строку, как бы нажимая Enter. Таким образом следующая команда вывода начнёт писать уже на новой строке.

В отличие от этого, вывод без '\n' не добавляет Enter после вывода текста. Поэтому текст, выведенный несколькими такими командами, будет выводиться в одну строчку.

Пример 1:

std::cout << "Amigo";
std::cout << "The" << '\n';
std::cout << "Best";

Смешанный вывод: часть без перевода строки, часть — с переводом

Выведет на экран:

AmigoThe
Best

Пример 2:

std::cout << "Amigo";
std::cout << "The";
std::cout << "Best";

Вывод без перевода строки — всё «склеится» в одну линию

Выведет на экран:

AmigoTheBest

Если хотите вывести текст на одной строке — не добавляйте '\n', если на нескольких — добавляйте. Все просто.

3. Разбираем вывод через std::cout

Если вы хотя бы немного знаете английский, то изучать программирование вам будет намного проще. Название многих вещей в C++ напрямую связанное с тем, что они делают.

Например cout — это сокращение от console output — «вывод в консоль».

std::cout — это один из самых распространённых способов взаимодействия программы и человека. Когда пользователь пишет программе команды на клавиатуре, а она в ответ выводит ему текст на экран, то программисты говорят, что программа работает с консолью.

Другими словами, работа с консолью — это вывод текста на экран и ввод данных с помощью клавиатуры. Поэтому если вы в будущем услышите, что программа что-то вывела в консоль, то это просто значит, что она вывела текст на экран.

А переход на новую строку в таких примерах мы делаем явно, добавляя '\n' (символ перевода строки).

Мини‑шпаргалка по выводу

Когда вы только начинаете, полезно держать в голове несколько “микро‑правил”, которые постоянно всплывают в задачах. Вместо списка сложим это в маленькую таблицу‑напоминалку.

Вывести текст. строка — в двойных кавычках

std::cout << "Hi";

Вывести число. число — без кавычек

std::cout << 123;

Вывести один символ. символ — в одинарных кавычках

std::cout << 'A';

Перейти на новую строку. '\n' — один символ перевода строки

Вывести “сложную” строку из частей. цепочка << печатает подряд

Лекция 3. Знакомство с переменными в C++

1. Переменные и коробки

Переменные — это такие специальные штуки для хранения данных. Любых данных. Почти все данные в C++ так или иначе оказываются в переменных. Больше всего переменная по смыслу напоминает коробку: самую обычную коробку.

Допустим, вы написали на листке число 13 и положили этот лист в коробку. Теперь мы можем говорить, что «коробка хранит значение 13».

Каждая переменная в C++ имеет три важных свойства: тип, имя и значение.

Имя используется для того, чтобы отличать одну переменную от другой. Это как надпись на коробке.

Тип переменной определяет тип значений/данных, которые в ней можно хранить. В коробке для торта храним торт, в коробке для обуви — туфли, и т.д.

Значение — это некий объект, данные или информация, которая хранится в переменной.

Каждый объект в языке C++ имеет свой тип. Например, могут быть такие типы данных: целое число, дробное число, текст, тип Cat, тип House и т.д.

У каждой переменной (коробки) тоже есть свой тип. Переменная может хранить значения только того же типа, что и она сама. Разные коробки используются для хранения различных вещей: коробка конфет, коробка для десятка яиц и т.п. Все как в жизни.

2. Создание переменной

В языке C++ для того, чтобы создать переменную, используется команда вида:

тип имя;

Где тип — это тип переменной (совпадает с типом значений, которые переменная сможет хранить). А имя — это имя переменной.

Примеры:

int a;

Создаем переменную по имени a типа int.

std::string s;

Создаем переменную по имени s типа std::string.

double c;

Создаем переменную по имени c типа double.

Два самых часто используемых типа — это целые числа (обозначаются словом int) и текст (в C++ обычно используют std::string). Также популярен тип double — это дробные (вещественные) числа.

3. Присваивание

Как уже говорилось выше, у переменной есть имя, тип и значение. Имя и тип мы уже разобрали, а что насчет значения? Как занести значение в переменную?

Чтобы занести значение в переменную, существует специальная операция — операция присваивания. Она копирует значение из одной переменной в другую. Не переносит, а именно копирует. Как файл на диске. Выглядит это присваивание так:

имя = значение;

Где имя — это имя переменной, а значение — это значение, которое будет занесено в переменную. В качестве значения может фигурировать конкретное значение, имя другой переменной или даже какое-нибудь выражение с использованием переменных.

Примеры:

int main() {
  int i;         // Создается переменная i
  int a, b;      // Создаются переменные a, b
  int x;         // Создается переменная x

  i = 3;         // В переменную i заносится значение 3.

  a = 1;         // В переменную a заносится значение 1.
  b = a + 1;     // В переменную b заносится значение 2.

  x = 3;         // В переменную x заносится значение 3.
  x = x + 1;     // В этой строке значение x увеличивается на 1, x теперь равен 4.
}

Символ равно

Для операции присваивания используется символ равно =. Это не сравнение, не равенство. Это именно команда копирования значения справа от знака равно в переменную, которая слева. Для сравнения в языке C++ используется двойное равно ==.

4. Имя переменной: что можно, а что нельзя

Вот какие факты вам полезно будет знать при создании переменных:

  • Имя переменной может быть любым (например, temperature, score, userAge).
  • Но есть ограничения — оно не может начинаться с цифры, и не может совпадать со служебными словами C++ (например, нельзя назвать переменную int или if).
  • Не может содержать специальные символы, кроме _.
  • В имени важна разница между строчными и прописными буквами (Age и age — разные переменные).
  • Обычно используют стиль camelCase: первая буква маленькая, дальше — каждое слово с заглавной (userAge, maxScore).

Примеры разрешенных имен:

6. Краткая история C++

Чтобы понять некоторые вещи в C++, нужно немного вернуться назад — к языку C.

Язык C появился в начале 1970-х годов в Bell Labs. Его создал Деннис Ритчи. Это был компактный, быстрый и очень практичный язык, который позволял писать системные программы и даже операционные системы. Он давал программисту много контроля над памятью и устройством программы. Именно поэтому C стал фундаментом для огромного количества технологий.

В начале 1980-х Бьёрн Страуструп решил расширить C и добавить в него новые возможности, прежде всего поддержку классов. Сначала язык назывался “C with Classes”, а позже получил имя C++. Идея была простой: сохранить эффективность C, но сделать язык удобнее для создания больших и сложных программ.

С тех пор C++ развивался почти 50 лет. Он регулярно обновляется, и каждая новая версия стандарта добавляет новые возможности. Сегодня существуют стандарты C++11, C++14, C++17, C++20, C++23 — и работа над следующими версиями уже идёт. В результате язык стал очень мощным и многогранным.

Однако в C++ нередко можно встретить несколько способов сделать одно и то же. Например, даже инициализировать переменную можно по-разному:

int a = 5;
int b(5);
int c{5};

Все три варианта корректны — просто появились они в разные периоды развития языка. Похожая история и со строками: есть разные формы создания, разные способы объединения, форматирования и преобразования значений. Это следствие долгой эволюции языка.

Важно понимать: C++ не переписывался заново каждые 10 лет. Он аккуратно расширялся, сохраняя совместимость со старым кодом. Именно поэтому в нём накопилось так много возможностей.

Но это не значит, что нам нужно изучать всё подряд. Мы будем использовать самые современные и актуальные стандарты — C++20 и C++23. Мы будем писать код в современном стиле, выбирать более безопасные и понятные конструкции и постепенно разбираться в языке без перегруза историческими деталями.

Лекция 4. Типы переменных в C++: int, double и std::string

1. Объявление переменных

Давайте еще раз вспомним, как создавать переменные. Чтобы создать переменную, нужно написать такую команду: тип имя;.

Примеры:

Создается переменная s типа std::string. Такая переменная может хранить текст.

int x;

Создается переменная x типа int. Такая переменная может хранить в себе целые числа.

int a, b, c; int d;

Создаются переменные a, b, c, d типа int. Такие переменные могут хранить целые числа.

Важно! Нельзя создать две переменные с одинаковыми именами в одном блоке кода. А вот в разных — можно. Это как коробки, стоящие в разных домах. Подробнее о блоках кода — в следующих лекциях.

Еще есть ограничения на имя переменной. С одной стороны, оно может быть любым, с другой — оно не может содержать пробелов, символов +, - и т.д. Лучше всего в имени переменной использовать только латинские буквы и цифры.

И опять напоминаю, что в языке C++ играет роль, какие буквы вы пишете — большие или маленькие. int a — это не то же самое, что и int A.

Кстати, в C++ можно одновременно создавать переменную и присваивать ей значение. Это экономит время и место:

int a = 5;

int c = 7;

std::string s = "I'm Amigo";

Так гораздо компактнее и понятнее.

2. Тип int

В переменной типа int можно хранить целые числа: 5, -10, 100500. Также над типом int можно выполнять различные арифметические операции: сложение, вычитание, умножение, деление и другие.

Примеры:

int main() {
  {
    int x = 1;                  // x будет равен 1
    int y = x * 2;              // y будет равен 2
    int z = 5 * y * y + 2 * y + 3; // z будет равен 20+4+3, будет равен 27
  }

  {
    int a = 5;                  // a будет равно 5
    int b = 1;                  // b будет равно 1
    int c = (a - b) * (a + b);  // c будет равно 4*6, будет равно 24
  }

  {
    int a = 64;                 // a будет равно 64
    int b = a / 8;              // b будет равно 8
    int c = b / 4;              // c будет равно 2
    int d = c * 3;              // d будет равно 6
  }
}

Если вы в школе не прогуливали математику, тот тут вам все должно быть понятно

А если прогуливали, то вот вам список разрешенных операций:

+ — сложение
- — вычитание
* — умножение
/ — деление
% — остаток от деления

3. Тип double

Тип int хранит только целые числа. А что если нам нужны числа с дробной частью? Для этого существует тип double.

Он хранит вещественные числа:

double x = 3.14;
double y = -0.5;
double z = 2.0;

Особенность double

Числа типа double хранятся в памяти приближённо. Иногда могут появляться неожиданные эффекты:

double x = 0.1 + 0.2; // 0.300000004

Вы можете ожидать ровно 0.3, но внутри хранится приближённое значение. Это нормально. Так работает представление дробных чисел в компьютере.

4. Тип std::string

Тип std::string позволяет хранить текстовые строки. А чтобы задать какую-то текстовую строку, в C++ надо написать текст строки, а с обеих сторон поставить двойные кавычки.

std::string s = "Amigo";

s будет содержать текст Amigo

std::string s = "123";

s будет содержать текст 123

std::string s = "Bond 007";

s будет содержать текст Bond 007

Выглядит несложно, правда? Ну если так, тогда вот еще один интересный факт.

Строки в C++ можно склеивать с помощью знака плюс — +. Пример:

#include <string>

int main() {
  std::string s1 = std::string("Amigo") + " the best"; // s будет содержать Amigo the best

  std::string s2 = ""; // s будет содержать пустую строку — вообще без символов.

  int x = 333;
  std::string s3 = std::string("Amigo") + std::to_string(x); // s будет содержать Amigo333
}

Обратите внимание на последний пример: мы сложили строку и число. Тут тоже все просто: число будет преобразовано в строку, затем две строки будут склеены. При сложении строк и чисел всегда получается строка.

5. Вывод переменной на экран

Вроде все так очевидно и элементарно. Ну тогда, может, вы сразу догадаетесь, какой командой можно вывести переменную на экран?

На самом деле все просто. Чтобы вывести что-то на экран, мы используем std::cout, куда передаем то, что мы выводим. Для перевода строки удобно добавить '\n'.

#include <iostream>
#include <string>

int main() {
  std::cout << "Amigo\n";              // Amigo

  std::cout << "Ami" << "go" << '\n';  // Amigo

  std::string s1 = "Amigo";
  std::cout << s1 << '\n';          	// Amigo

  std::string s2 = "Am";
  std::cout << s2 << "igo" << '\n'; 	// Amigo
}

6. Сокращённые операции присваивания (+=, -=, *=, /=)

В C++ есть удобная форма записи операций вида:

x = x + 5;

Её можно записать короче:

x += 5;

Это называется сокращённое присваивание. Примеры:

int x = 10;

x += 5;  // x = 15
x -= 3;  // x = 12
x *= 2;  // x = 24
x /= 4;  // x = 6

То есть:

x += a   →   x = x + a
x -= a   →   x = x - a
x *= a   →   x = x * a
x /= a   →   x = x / a

Это не просто «красота кода» — такие записи короче, понятнее и используются постоянно.

Сокращённые операции работают и со строками:

std::string s = "Hello";
s += " World";   // "Hello World"

Лекция 5. Компилятор и комментарии

1. Знакомство с компилятором

Компьютер не понимает человеческий язык. Компьютер не понимает даже языки программирования. Он понимает только программы, написанные в машинных кодах — состоящие из нулей и единичек. Люди же мыслят словами, идеями, и конечно же, им сложно сочинять целые программы из одних только 0 и 1.

Поэтому и появились языки программирования вроде C++. Они стали своего рода "переводчиками": теперь программист может написать команды «Сделай то-то и то-то» на понятном ему языке, а не в виде длинной цепочки команд процессору.

Но есть одна загвоздка. Компьютер всё равно не понимает C++! Если просто скормить ему файл с кодом, он пожмёт плечами и ничего не сделает. Именно для этой ситуации и существует компилятор.

Что же такое компилятор?

Компилятор — это специальная программа, которая компилирует код: переводит C++-код в код понятный компьютеру. Он берёт текст на C++, разбирает его, проверяет на ошибки и превращает в машинный код (обычно — в виде исполняемого файла).

Можно представить компилятор как строгого учителя, который:

  • внимательно проверяет фразы (а не допустил ли ты ошибку?),
  • заставляет соблюдать правила (например, правильные имена переменных, точки с запятыми, скобки…),
  • и только после этого допускает до собеседования с компьютером!

Компилятор еще умнее

Компиляторы C++ сейчас стали еще умнее — они могут не только переводить, но и оптимизировать код, подсказывать ошибки и даже советовать, как писать лучше (через предупреждения — warnings). И еще много много всего.

А ещё ради удобства и правильной сборки большой программы компиляция обычно происходит не “одним магическим шагом”, а как минимум в несколько этапов:

  • Сначала препроцессор обрабатывает директивы вроде #include.
  • Затем компилятор переводит C++-код в объектный код (частично готовые “кусочки” программы).
  • А потом линкер (компоновщик) склеивает эти кусочки и подключаемые библиотеки в один исполняемый файл.

Если у вас крутой процессор и вы включили оптимизации, компилятор может сгенерировать машинный код с учётом возможностей вашей платформы и выбранных настроек сборки. Но в целом важно запомнить простую идею: C++ обычно компилируется сразу в машинный код, и уже потом этот готовый исполняемый файл запускается.

2. Ошибки при компиляции

Скорость и оптимизация кода нам пока что не сильно важна, а вот что важно, так это то, что компилятор проверяет вашу программу на ошибки. Он проверяет код на корректность и не пустит его дальше, если нашёл даже маленькую ошибку.

Пример ошибки:

Вы пытаетесь присвоить число переменной, которая может хранить только текст.

#include 

int main() {
  std::string userName = 42; // Ой! Нельзя так.
}

Компилятор сразу выдаст ошибку, что типы не соответствуют (формулировка зависит от компилятора), что-то вроде: «нет подходящего преобразования из int в std::string».

Другой пример — опечатка в имени std::cout:

#include 

int main() {
  std::coutt << "Привет!\n"; // Ошибка в названии
}

Компилятор скажет что-то вроде: «coutt не является членом std» — проверяй синтаксис!

Как читать ошибки?

Большинство компиляторов подсказывают, где в коде ошибка: обычно это файл, номер строки и часто даже номер колонки, плюс текст ошибки. Не бойтесь ошибок: каждая из них — это шаг к просветлению и прохождению вашего внутреннего "босса".

4. Комментарии

Итак, представьте: вы пишете сложную программу. Все работает, вы — гений! Но через месяц вы открываете тот самый код и... ничего не понимаете. Пару строчек кода тут, странная переменная там. В голове только: «Это явно писал кто угодно, но не я!». Вот тут-то и спасают комментарии.

Комментарии — это как записки на полях учебника или подписи на холодильнике ("Пиво. Не пить. На утро!"). Компилятор их игнорирует, а люди — наоборот. Комментарии позволяют объяснить смысл кода, помечать важные детали и даже «выключать» куски программ, не удаляя их.

Важно: Компилятор не “понимает смысл” ваших комментариев — для него это просто текст, который нужно пропустить.

Однострочные комментарии

В C++ есть два вида комментариев — однострочные и многострочные. Однострочные начинаются с двойного слэша // и идут до конца строки. Всё, что находится после //, считается комментарием и абсолютно игнорируется компилятором.

Пример:

#include 

int main() {
  std::cout << "Привет, мир!\n"; // Это выводит приветствие на экран
}

Вот тут // Это выводит приветствие на экран — комментарий. Если в коде кажется непонятным, зачем нужна строка, вот так и пишите!

Еще пример:

int main() {
  int x = 42; // Значение возраста программиста
}

Можно писать однострочные комментарии и на отдельных строках:

#include 

int main() {
  // Инициализация переменной с именем пользователя
  std::string userName = "Вася";
}

Почти как реплики в сценарии: комментарий объясняет, зачем и почему.

Многострочные комментарии

Что, если у вас много всего накопилось? Настоящая история? Для этого есть многострочные комментарии, которые начинаются с /* и заканчиваются на */. Всё между этими символами игнорируется компилятором, даже если это несколько строк.

Пример:

#include 

int main() {
  /*
    Здесь мы выводим подсказку пользователю.
    Если имя отсутствует, в других программах
    можно написать "Гость".
  */
  std::cout << "Введите ваше имя:\n";
}

Или так:

int main() {
  int age = 18; /* Начальное значение возраста
                   для нового пользователя */
}

Внимание: нельзя вложить одни многострочные комментарии в другие. Не пытайтесь быть хитрее компилятора — это не сработает.

5. Примеры из жизни

«Отключение» строки кода:

Бывает, что вы временно хотите не выполнять какой-то фрагмент кода. Комментарии тут как раз к месту:

// std::cout << "Текст, который временно не нужен\n";

Строчка осталась, но компилятор её не «видит».

Пометки TODO и FIXME

Программисты — тоже люди, и иногда нужно самому себе напомнить, что код ещё не доделан, или что тут есть баг. Для этого часто пишут:

// TODO: сделать проверку на пустой ввод
// FIXME: эта функция неверно считает сумму

Многие редакторы и IDE даже выделяют такие слова!

Программисты шутят

Ну и, конечно, иногда, в коде попадаются очень интересные комментарии:

// Я не отвечаю за этот код. Меня заставили
// написать его против моего желания.

// Дорогой, я_из_будущего! Пожалуйста, прости меня за этот код.
// Если я еще раз увижу такое, мне придется
// начать носить на работу оружие.
// Если это условие когда-нибудь выполнится, пожалуйста,
// сообщите мне по тел. ххх-ххх-ххх за вознаграждение.

// Дорогой программист:
//
// Когда ты закончишь «оптимизировать» эту подпрограмму
// и поймешь, насколько большой ошибкой было делать это,
// пожалуйста, увеличь счетчик внизу как предупреждение
// для следующего парня:
//
// количество_часов_потраченных_здесь = 42

// Когда я начинал это писать,
// только Бог и я понимали, что я делаю.
// Сейчас остался только Бог

// иногда мне кажется, что компилятор игнорирует все мои комментарии

// Я посвящаю весь свой код, всю работу своей жене Дарлин,
// которой придётся содержать меня, наших троих детей
// и собаку, когда это пойдет в продакшн.

Да, комментарии могут быть очень смешными. Ведь их пишут живые люди.

Лекция 1. Целые числа: int, арифметика, деление и остаток %

1. Переменные int: зачем нужны и как с ними работать

На старте кажется, что числа в программе можно использовать «как в тетрадке»: написал 2 + 2 — и прекрасно. Но реальная программа обычно не живёт одной строкой: она хранит данные, меняет их, использует результат вычислений несколько раз и печатает разные куски в разном месте. Поэтому нам нужен способ сказать: «Вот это число — важное, давай дадим ему имя и будем к нему обращаться».

Тип int — самый базовый инструмент для этого. Он хранит целые числа: 0, 7, -12, 42. Без дробной части. Можно думать о нём как о «коробке для целого числа», на которой мы пишем ярлык (имя переменной), чтобы не путаться.

Объявление, начальное значение и присваивание

Когда вы создаёте переменную, вы делаете две вещи: придумываете ей имя и выбираете тип, то есть «что именно» там будет храниться. А потом обычно сразу даёте начальное значение, чтобы переменная не была «пустой» (в C++ это особенно важно).

Посмотрим на минимальный пример. Здесь мы не делаем ничего умного — просто храним возраст и печатаем его. Но именно так начинается почти любая программа: с данных.

#include 

int main() {
  int age = 18;

  std::cout << "Age = " << age << '\n'; // Age = 18
}

Обратите внимание на три момента. Во‑первых, int age = 18; читается как «создай переменную age типа int и положи туда 18». Во‑вторых, в std::cout можно печатать и текст, и числа, просто «склеивая» их оператором <<. В‑третьих, перевод строки '\n' помогает сделать вывод аккуратным.

Теперь самое важное: переменная — это не константа. Её значение можно менять с помощью присваивания.

#include 

int main() {
  int score = 0;
  std::cout << score << '\n'; // 0

  score = 10;
  std::cout << score << '\n'; // 10
}

Здесь строка score = 10; означает «замени старое значение на новое». И да: знак = в программировании обычно означает не «равно», а «положи/запиши».

2. Арифметика int и скобки в выражениях

Операции +, -, * для целых чисел

С арифметикой в int всё выглядит очень знакомо, и это приятно: сложение, вычитание и умножение работают «как вы ожидаете». Правда, есть важный нюанс: результат тоже будет целым числом, потому что мы работаем в мире int. Пока что это звучит очевидно, но на делении это выстрелит особенно громко — и мы до этого дойдём.

Начнём с простого: посчитаем сумму, разность и произведение.

#include 

int main() {
  int a = 7;
  int b = 2;

  std::cout << (a + b) << '\n'; // 9
  std::cout << (a - b) << '\n'; // 5
  std::cout << (a * b) << '\n'; // 14
}

Скобки тут не обязательны, но новичка они часто спасают: вы явно показываете, что именно хотите посчитать, и меньше шансов превратить выражение в ребус.

Ещё один полезный приём — сохранять результат вычисления в переменную. Это делает код читаемым: сначала вычислили, потом использовали.

#include 

int main() {
  int apples = 3;
  int pears = 4;

  int total = apples + pears;
  std::cout << total << '\n'; // 7
}

Скобки: для смысла и читаемости

Когда выражения становятся чуть длиннее, мозг начинает делать то же, что и компилятор: пытаться понять порядок вычислений. Проблема в том, что мозг устаёт, а компилятор — нет. Поэтому хороший стиль в программировании часто выглядит как «ставим скобки там, где человеку может быть неочевидно». Это не слабость, это забота о будущем себе (и о менторе, который будет проверять ваш код).

Сравните два варианта: формально они могут быть эквивалентны, но читаются по‑разному.

#include 

int main() {
  int x = 10;
  int y = 3;

  int r1 = x + y * 2;
  int r2 = (x + y) * 2;

  std::cout << r1 << '\n'; // 16
  std::cout << r2 << '\n'; // 26
}

Скобки меняют смысл. И это отличный повод держать в голове правило: если формула важна, лучше сделать её очевидной глазами, а не надеяться на память о приоритетах операторов.

3. Деление / и остаток % для int

Деление /: результат всегда целый

Оператор / для int делает целочисленное деление. Это значит: результат тоже int, и дробная часть не округляется «по правилам математики», а просто отбрасывается.

Проверим на классическом примере:

#include 

int main() {
  int a = 7;
  int b = 2;

  std::cout << (a / b) << '\n'; // 3
}

Почему 3, а не 3.5? Потому что int не умеет хранить .5. Он хранит только целые значения. Поэтому программа берёт целую часть и выкидывает остальное.

Это поведение удобно, когда вы считаете «сколько целых предметов помещается». Например, сколько пачек печенья по 6 штук можно собрать из 25 печенек. Но это же поведение опасно, если вы ожидали «настоящее» деление. Пока мы работаем только с int, просто зафиксируйте правило: int / int даёт int.

Отдельно отмечу, что с отрицательными числами результат тоже будет целым. В современном C++ отбрасывание дробной части идёт в сторону нуля, то есть -7 / 2 будет -3.

#include 

int main() {
  int a = -7;
  int b = 2;

  std::cout << (a / b) << '\n'; // -3
}

Если вы сейчас подумали «ммм, а остаток тогда какой?» — вы мыслите как программист. Переходим к %.

Остаток %: что осталось после деления

Оператор % читается как «остаток от деления». Он работает только для целых чисел (в нашем случае — для int). И обычно используется в задачах вида «разложить по коробкам» или «разменять сумму» или «понять, делится ли число нацело».

Самый базовый пример:

#include 

int main() {
  int a = 7;
  int b = 2;

  std::cout << (a % b) << '\n'; // 1
}

То есть 7 апельсин поделить на 2 человек будет 3 апельсина каждому и 1 в остатке.

Полезно помнить связку деления и остатка в виде формулы: a = (a / b) * b + (a % b)

Это не магия и не заклинание, а просто аккуратный способ сказать: «число a состоит из целого количества групп по b плюс остаток».

Давайте это прям проверим кодом:

#include 

int main() {
  int a = 25;
  int b = 6;

  int q = a / b;  // сколько полных групп
  int r = a % b;  // сколько осталось

  std::cout << q << '\n'; // 4
  std::cout << r << '\n'; // 1
}

Пока что это выглядит как «игрушка», но через пару задач вы начнёте видеть % везде: время на часах, чётность/нечётность, разбиение на страницы, перевод секунд в минуты, и так далее.

4. Деление на ноль недопустимо

В математике деление на ноль — не определено. В программировании это превращается в практическую проблему: выражения a / 0 и a % 0 недопустимы. Программа либо аварийно завершится, либо попадёт в совсем непредсказуемое поведение (что ещё хуже, потому что ошибки могут проявиться «где-то потом»).

Здесь важен не страх, а привычка: если у вас есть делитель, который теоретически может стать нулём, вы должны это контролировать. Пока мы ещё не проходили if, поэтому сегодня правило простое: в примерах и первых задачах не делим на ноль и держим в голове, что делитель должен быть ненулевым.

5. Мини‑приложение: «Автомат с монетами»

Сейчас мы напишем маленькую программу, которая выглядит как «часть реального мира»: у нас есть сумма денег и цена товара, и мы хотим узнать две вещи. Во‑первых, сколько товаров можно купить. Во‑вторых, сколько денег останется. Это идеальная задача для пары / и %.

Важно: пока мы не умеем вводить данные, поэтому числа будут «захардкожены» в коде. Это нормально: сегодня наша цель — не ввод, а арифметика.

Версия 1: считаем количество и сдачу

Представим, что у нас 275 центов (то есть $2.75), а печенье стоит 60 центов. Считаем.

#include 

int main() {
  int moneyCents = 275;
  int cookiePrice = 60;

  int cookies = moneyCents / cookiePrice;
  int change = moneyCents % cookiePrice;

  std::cout << cookies << '\n'; // 4
  std::cout << change << '\n';  // 35
}

Тут прекрасно видно, зачем нам переменные: один раз задали цену и сумму, а дальше используем их в вычислениях и можем легко поменять сценарий, не переписывая всю программу.

Версия 2: делаем вывод понятнее

Пока вывод у нас «компьютерный»: просто числа в столбик. Давайте добавим подписи, чтобы стало похоже на нормальную консольную программу. Мы не используем строки как тип (это будет следующая лекция), но печатать текст в кавычках мы уже умеем.

#include 

int main() {
  int moneyCents = 275;
  int cookiePrice = 60;

  int cookies = moneyCents / cookiePrice;
  int change = moneyCents % cookiePrice;

  std::cout << "Cookies: " << cookies << '\n'; // Cookies: 4
  std::cout << "Change: " << change << " cents\n"; // Change: 35 cents
}

Да, это уже похоже на приложение, а не на контрольную по арифметике.

Версия 3: ещё одна покупка

Допустим, после покупки печенек вы ещё хотите купить жвачку за 35 центов. У нас как раз есть сдача. Посчитаем, сколько жвачек можно купить на сдачу (это, кстати, тоже / и %, только с другими числами).

#include 

int main() {
  int moneyCents = 275;
  int cookiePrice = 60;
  int gumPrice = 35;

  int cookies = moneyCents / cookiePrice;
  int changeAfterCookies = moneyCents % cookiePrice;

  int gums = changeAfterCookies / gumPrice;

  std::cout << "Cookies: " << cookies << '\n'; // Cookies: 4
  std::cout << "Gums: " << gums << '\n';       // Gums: 1
}

Здесь особенно видно, что переменные помогают держать «сюжет» программы. Мы не просто считаем числа, мы описываем процесс.

6. Типичные ошибки при работе с int, / и %

Ошибка №1: ожидать, что 7 / 2 даст 3.5.

Это одна из самых частых ловушек у новичков: мозг помнит школьную математику, а программа работает в мире int. В целочисленном делении дробная часть отбрасывается, потому что int не умеет хранить половинки. Если вам нужна дробь — это отдельная тема, но сегодня мы честно живём в мире целых чисел и принимаем его правила.

Ошибка №2: использовать % «для всего подряд».

Остаток — это не «крутая кнопка», а конкретная операция со смыслом «что осталось после деления на равные группы». Если вы не можете объяснить словами, что именно остаётся, скорее всего, % вы применяете наугад. А угадывание в программировании работает примерно так же, как «угадывание пароля»: иногда получается, но лучше не надо.

Ошибка №3: забыть, что a % 0 и a / 0 недопустимы.

Деление на ноль не просто «даёт ошибку». Оно ломает саму логику вычисления. Поэтому правило простое: делитель (то, на что делим) должен быть ненулевым. Позже, когда появятся условия (if), вы научитесь защищать программу от такого ввода. Сейчас же — просто не пишите такого кода и не подставляйте 0 в делитель.

Ошибка №4: писать длинные выражения без скобок и потом не понимать результат.

Формально компилятор разберётся, но человек — не всегда. В итоге вы получаете ситуацию «вроде всё правильно, но почему ответ не тот». Скобки — дешёвый способ сделать смысл выражения очевидным и для вас, и для того, кто будет читать код. В программировании это называется не занудством, а профилактикой багов.

Ошибка №5: путать “создать переменную” и “изменить переменную”.

int x = 10; — это создание переменной с начальным значением. x = 10; — это присваивание уже существующей переменной. Если попытаться «создать» переменную второй раз с тем же именем в том же месте, компилятор справедливо возмутится: «так, минутку, у нас тут уже есть x».

Лекция 2. Строки — std::string, конкатенация, длина

1. Введение

Когда вы только начинаете программировать, легко попасть в иллюзию, что «главное — числа». Ну правда: оценки, деньги, возраст, счётчики — везде цифры. Но как только вы захотите сделать программу хоть немного дружелюбной к человеку, внезапно выясняется: человек общается словами. Имя, фамилия, город, комментарий к заказу, название фильма — всё это текст, то есть строка.

Когда С++ только появился, то типа строка (string) в нем не было - строку воспринимали просто как набор символов (тип char). Однако со временем выяснилось, что это очень неудобно, и было решено создать отдельный тип string и добавить его в стандартнутю (STD) библиотеку С++. Так в C++ появился тип std::string.

Его можно воспринимать как контейнер для последовательности символов: внутри лежат буквы, пробелы, знаки пунктуации. И, что особенно приятно, стандартная библиотека даёт нам готовые операции для строк: склеивать, узнавать длину, брать символ по номеру.

Чтобы покачать работу с std, мы будем развивать одно небольшое приложение по мере курса: «консольная визитка». Пока оно будет просто собирать красивую строку визитки и печатать её. Сегодня мы добавим туда имя и пару текстовых деталей.

2. Первая строка: подключаем <string>

Сейчас будет момент, который сначала кажется мелочью, а потом становится второй натурой: чтобы использовать std::string, обычно нужен заголовок <string>. Да, в некоторых компиляторах «как-то работает и без него», но это плохая привычка: лучше подключать то, что вы используете.

Начнём с простого: создадим строку из строкового литерала "..." и выведем её.

#include 
#include 

int main() {
  std::string course = "C++ for beginners";
  std::cout << course << '\n'; // C++ for beginners
}

Здесь важна идея: std::string — это тип, как int, только для текста. Мы объявили переменную course и положили в неё текст. Дальше она ведёт себя как «значение», которое можно передавать в вывод.

Строка "A" и символ 'A'

В программировании есть шутка: «две кавычки решают судьбу проекта». И, к сожалению, это почти правда. В C++ есть строковые литералы в двойных кавычках "..." и символьные литералы в одинарных '...'. Разница принципиальная: строка — это последовательность символов, а символ — это ровно один символ.

Если вам нужно хранить имя "Ada", это строка. Если вам нужен один символ, например первая буква имени 'A', это символ. std::string хранит строку, а тип для одного символа мы пока воспринимаем просто как «один знак» (формально это char, но об этом позднее).

Сравните:

#include 
#include 

int main() {
  std::string word = "Cat";                   // строка текста
  char firstLetter = 'C';                     // один символ

  std::cout << word << '\n';        // Cat
  std::cout << firstLetter << '\n'; // C
}

На вид похоже, но смысл другой: word может быть любой длины, а firstLetter — строго один символ.

3. Конкатенация: + и +=

Переходим к очень «человеческой» операции: собрать из кусочков одну красивую фразу. В строках это называется конкатенацией. Представьте, что строка — это конструктор LEGO, только вместо кубиков слова и пробелы. А вы собираете предложение, которое потом покажете пользователю.

Есть два основных способа склеивания:

  • a + b создаёт новую строку (как будто вы взяли два листочка и сделали третий, где написано всё вместе).
  • a += b дописывает в существующую строку (как будто вы продолжили писать на том же листочке).

Начнём с +:

#include 
#include 

int main() {
  std::string first = "Ada";
  std::string last = "Lovelace";

  std::string full = first + " " + last;
  std::cout << full << '\n'; // Ada Lovelace
}

Обратите внимание на " ". Пробел — это тоже символ, а часто и смысл. Если забыть пробел, получится AdaLovelace, что звучит как имя класса, но не как имя человека.

Теперь вариант с += — он удобен, когда вы строите строку постепенно:

#include 
#include 

int main() {
  std::string badge = "Name: ";
  badge += "Ada";
  badge += " ";
  badge += "Lovelace";

  std::cout << badge << '\n'; // Name: Ada Lovelace
}

С точки зрения читаемости это похоже на «собираем строку шаг за шагом». В будущем, когда появятся условия и циклы, += будет особенно полезен, но даже сейчас он хорошо показывает идею «дописывания».

Небольшая важная ремарка: строку и число нельзя просто так «плюсовать» как строки. Например, "Age: " + 10 — так не работает, потому что это разные типы. Мы пока не будем углубляться в преобразования; просто фиксируем: строки склеиваются со строками.

4. Длина строки: size()

Когда вы держите в руках строку, часто хочется понимать, сколько в ней символов. Для этого у std::string есть метод size() (иногда также встречается length(), но сегодня достаточно size()). Это как спидометр, который показывает «сколько символов сейчас в строке».

Посмотрим в деле:

#include 
#include 

int main() {
  std::string city = "Boston";
  std::cout << city.size() << '\n'; // 6
}

Важно привыкнуть к мысли: size() — это первое, что вы вспоминаете перед тем, как полезть внутрь. Потому что индексация без понимания длины — это как идти по лестнице в темноте, считая, что ступеньки бесконечные. Спойлер: ступеньки конечные, а колено одно.

Ещё один пример, где длина помогает увидеть, что пробел — тоже символ:

#include 
#include 

int main() {
  std::string s = "Hi Bob";
  std::cout << s.size() << '\n'; // 6  (H i _ B o b)
}

5. Индексирование: s[i]

Как вы уже выяснили - стока это набор символов. А раз это набор символов, то можно получить отдельные символы из стоки по их номеру.

В программировании почти вся нумерация идет с нуля. Нумерация с нуля означает: первый символ — это s[0], второй — s[1], и так далее.

Представим строку как ящик с ячейками:

s = 'c a t' // индексы: 0 1 2

Посмотрим на код:

#include 
#include 

int main() {
  std::string animal = "cat";

  std::cout << animal[0] << '\n'; // c
  std::cout << animal[1] << '\n'; // a
  std::cout << animal[2] << '\n'; // t
}

Про границы

Чтобы меньше путаться, в программировании приятно использовать термин индекс вместо номер. Индекс всегда начинается с нуля.

Индекс должен быть в диапазоне от 0 до s.size() - 1. Если строка длины 3, допустимые индексы — 0, 1, 2. Если попытаться взять animal[3], вы выходите за границы строки и программа падает.

В C++ это опасно тем, что программа может вести себя непредсказуемо: иногда «как будто работает», иногда печатает странные символы, иногда падает. И это как раз тот случай, когда «иногда работает» хуже, чем «сразу падает», потому что баг прячется и ждёт подходящего момента.

Можно ли менять символы в строке?

Да, символ по индексу можно заменить: строка становится изменяемой, и это удобно для простых правок.

#include 
#include 

int main() {
  std::string word = "bat";
  word[0] = 'c';

  std::cout << word << '\n'; // cat
}

Обратите внимание: справа стоит 'c' в одинарных кавычках, потому что мы присваиваем один символ, а не строку "c".

6. Первое знакомство с методами

Вы могли заметить странную запись:

city.size()

Почему мы не пишем просто size(city)? Потому что size() — это метод.

Метод — это функция, которая принадлежит конкретному типу или объекту. В нашем случае std::string — это тип, а city — объект этого типа. И у строк есть свои «встроенные умения»: узнавать длину, дописывать текст, получать символ по индексу и многое другое.

Когда функция принадлежит объекту, к ней обращаются через точку:

объект.метод()

Точка читается почти по-человечески: «Возьми объект city и вызови у него метод size()».

То есть строка как бы «сама знает», как определить свою длину. Это и есть идея методов: поведение связано с данными.

Посмотрите на уже знакомые примеры:

std::string full = first + " " + last;

std::cout << full.size() << '\n';
std::cout << full[0] << '\n';

full.size() — строка сама сообщает свою длину. full[0] — мы обращаемся к её внутреннему символу.

Позже вы увидите, что почти все сложные типы в C++ имеют свои методы. Например, у std::vector есть push_back(), у std::string есть substr(), у потоков есть clear(). Это один из ключевых принципов языка: данные и связанные с ними операции объединены.

Важно привыкнуть к этой записи. Всё, что выглядит как:

что_то.что_то()

почти всегда означает вызов метода объекта.

Именно поэтому мы пишем full.size(), а не просто size(full). Строка сама предоставляет инструменты для работы с собой.

7. Мини-приложение: консольная визитка

Теперь соберём всё в один небольшой цельный фрагмент: сделаем визитку, которая печатает имя, длину полного имени и первую букву. Пока данные возьмём прямо из кода (ввод будет в следующей лекции), чтобы не смешивать новые темы в одну кучу.

В этом примере у нас появится «ощущение приложения»: не просто отдельные операции, а маленький полезный вывод.

#include 
#include 

int main() {
  std::string first = "Ada";
  std::string last = "Lovelace";

  std::string full = first + " " + last;

  std::cout << "Full name: " << full << '\n';          // Full name: Ada Lovelace
  std::cout << "Length: " << full.size() << '\n';      // Length: 12
  std::cout << "First letter: " << full[0] << '\n';    // First letter: A
}

Здесь спрятано сразу несколько важных привычек.

Во-первых, мы явно добавили пробел между именем и фамилией, потому что компьютер не догадается, что «между словами должно быть место». Во-вторых, мы вывели длину строки: это помогает понять, что пробел тоже считается. В-третьих, мы взяли full[0] и достали первый символ, что даёт нам в будущем возможность делать инициалы или простые сокращения.

Если хочется чуть-чуть «поиграться», можно достать последнюю букву фамилии. Формально последняя буква имеет индекс size() - 1. Мы делаем это только на заведомо непустой строке, чтобы не рисковать выходом за границы:

#include 
#include 

int main() {
  std::string last = "Lovelace";
  char lastChar = last[last.size() - 1];

  std::cout << lastChar << '\n'; // e
}

Да, выражение выглядит слегка «математически», но логика простая: если длина 8, индексы идут 0…7, значит последний — 7, то есть size() - 1.

8. Типичные ошибки при работе со std::string

Ошибка №1: забыли #include <string>.

Иногда код «вроде бы» компилируется из‑за косвенных подключений, а иногда внезапно перестаёт. Правильная привычка простая: используете std::string — подключайте <string>. Это не прихоть, а реальная дисциплина: даже в примерах и документах встречаются правки, где добавляют недостающий include, потому что иначе пример некорректный.

Ошибка №2: перепутали "A" и 'A'.

Когда вы пишете word[0] = "b";, компилятор вам справедливо скажет: «эй, слева один символ, а справа строка». Для одного символа используйте одинарные кавычки: 'b'. Для текста — двойные: "b".

Ошибка №3: забыли про пробел при конкатенации.

Классика: first + last даёт “AdaLovelace”, а вы потом долго смотрите на результат и думаете, что программа сломалась. На самом деле она слишком буквальная: не попросили пробел — не добавила. Добавляйте " " явно.

Ошибка №4: индекс вышел за границы строки.

s[100] у строки "Hi" — это не «пустота», не null и не «просто ничего». Это выход за границы, и поведение программы становится непредсказуемым. До появления if мы просто не делаем такие индексы «наугад» и всегда опираемся на size().

Ошибка №5: ожидали, что size() — это «количество букв в любом языке».

На старте курса мы обычно работаем с простыми латинскими примерами, где один символ примерно равен одной «букве». В реальных текстах (особенно с кириллицей в UTF‑8) это может быть сложнее, но пока наша задача — освоить базовую механику строк, а не войну с кодировками.

Лекция 3. Ввод: std::cin >> и его ограничения

1. Поток std::cin и чтение токенов оператором >>

Когда программа только печатает, она похожа на радио: говорит что-то в пустоту и надеется, что кто-то слушает. Но как только появляется ввод, программа становится похожа на диалог: вы даёте данные, она отвечает. В C++ ввод обычно делают через std::cin. Очень полезно представить cin как ленту (поток) символов: там лежит то, что пользователь напечатал, включая пробелы и переводы строк.

Самое важное: оператор >> читает не «всё подряд», а одну порцию данных за раз. И эта «порция» почти всегда ограничена пробельными символами.

std::cin и оператор >>: читаем значение в переменную

Чтобы что-то прочитать, нам нужна переменная, куда это «что-то» положить. Логика такая: объявили переменную → прочитали в неё данные → используем дальше (например, выводим).

Минимальный пример: читаем одно целое число.

#include 

int main() {
  int age = 0;

  std::cin >> age;
  std::cout << "Age = " << age << '\n'; // если ввели 20, будет: Age = 20
}

Здесь важно заметить две вещи. Во‑первых, std::cin >> age; пытается прочитать следующее значение из ввода и записать его в age. Во‑вторых, тип переменной важен: раз age — это int, программа ожидает, что во входе сейчас «лежит» число (например 20, -5, 0).

Что именно читает >>: токен и пробельные разделители

Сейчас будет слово «токен», но не пугайтесь: это всего лишь «кусочек текста без пробелов». Оператор >> работает по идее: «Пропусти пробелы, потом возьми подряд символы до следующего пробела». Пробелом при этом считается не только ' ' (обычный пробел), но и перевод строки '\n', и табуляция '\t'.

Представьте, что пользователь ввёл такую строку (символ ⏎ — это Enter, то есть '\n'):

12 34 ⏎

Между 12 и 34 тут три пробела. Для >> это не важно: он видит «число», потом «пробелы», потом «число».

#include 

int main() {
  int a = 0;
  int b = 0;

  std::cin >> a >> b;
  std::cout << "a=" << a << ", b=" << b << '\n'; // при вводе "12   34" будет: a=12, b=34
}

Обратите внимание: std::cin >> a >> b; — это два чтения подряд, просто записанные в одну строку. Сначала читается a, затем читается b.

Разделители: пробел, Enter и табуляция одинаково «режут» токены

Иногда новичок ожидает, что Enter «сильнее», чем пробел, или что пробел «просто для красоты». Для operator>> всё наоборот: любой пробельный символ — граница токена, и точка.

Давайте представим такой ввод (каждое слово на новой строке):

Ivan⏎
Ivanov⏎

И такой ввод (оба слова на одной строке):

Ivan Ivanov⏎

Для кода ниже результат будет одинаковым, потому что и пробел, и Enter — это разделители токенов.

#include 
#include 

int main() {
    std::string first;
    std::string last;

    std::cin >> first >> last;
    std::cout << first << " | " << last << '\n'; // Ivan | Ivanov
}

Это хорошая новость: можно вводить данные «как удобно» (в одну строку или несколько). Но это же и причина ограничений, о которых мы поговорим дальше.

Мини-схема: как >> «режет» ввод на токены

Чтобы мозг перестал воспринимать ввод как магию, полезно один раз увидеть модель. Пусть пользователь ввёл вот так:

23 Ada Lovelace⏎

Тогда поток символов можно мысленно разрезать на токены:

23 Ada Lovelace

А пробелы и перевод строки — это «разделительная каша», которая просто пропускается между токенами.

Очень практическое следствие: если вы читаете int, а потом std::string, то int «съест» первый токен, а std::string — следующий. Никаких договорённостей «вводите с новой строки» на уровне >> не существует — он всё равно режет по пробелам.

2. Практика: мини‑анкета и строки без пробелов

Практический пример: делаем мини-анкету

С этого момента будем вести один и тот же мини‑проект: консольную анкету. Он простой, зато на нём удобно наращивать навыки: сегодня читаем токены, чуть позже научимся читать целые строки и аккуратно обрабатывать ввод.

Версия 1: читаем возраст и имя (как одно слово).

#include 
#include 

int main() {
  int age = 0;
  std::string name;

  std::cin >> age >> name;

  std::cout << "User: " << name << '\n';
  std::cout << "Age: " << age << '\n';
}

Если ввести:

20 Ivan

то программа выведет:

User: Ivan
Age: 20

Можно ввести и так (через Enter):

20
Ivan

С точки зрения >> это одно и то же: токены всё равно идут последовательно.

Ввод строк через >>: читается слово, а не фраза

Теперь мы подошли к классическому сюрпризу: std::cin >> s для std::string читает только одно слово, то есть токен до пробела/Enter/табуляции. Если вы введёте несколько слов, они не пропадут, они просто останутся «лежать» во входном потоке и будут прочитаны следующими операциями >>.

Демонстрация на нашей анкете: читаем «город» как строку.

#include 
#include 

int main() {
  std::string city;

  std::cin >> city;
  std::cout << "City=[" << city << "]\n";
}

Если пользователь введёт:

New York

то city станет "New", а слово "York" останется во входном потоке как следующий токен. Программа выведет:

City=[New]

Это не баг и не «кривой C++». Это нормальная логика оператора >>: он читает один токен.

Показательный эксперимент: «потерянное» слово на самом деле не потеряно

Слово, которое не поместилось в первую строковую переменную, обычно не исчезает. Оно просто ждёт своей очереди. Покажем это честно: прочитаем два слова в две переменные.
