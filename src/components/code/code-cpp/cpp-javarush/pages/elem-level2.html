Лекция 1. Целые числа: int, арифметика, деление и остаток %

1. Переменные int: зачем нужны и как с ними работать

На старте кажется, что числа в программе можно использовать «как в тетрадке»: написал 2 + 2 — и прекрасно. Но реальная программа обычно не живёт одной строкой: она хранит данные, меняет их, использует результат вычислений несколько раз и печатает разные куски в разном месте. Поэтому нам нужен способ сказать: «Вот это число — важное, давай дадим ему имя и будем к нему обращаться».

Тип int — самый базовый инструмент для этого. Он хранит целые числа: 0, 7, -12, 42. Без дробной части. Можно думать о нём как о «коробке для целого числа», на которой мы пишем ярлык (имя переменной), чтобы не путаться.

Объявление, начальное значение и присваивание

Когда вы создаёте переменную, вы делаете две вещи: придумываете ей имя и выбираете тип, то есть «что именно» там будет храниться. А потом обычно сразу даёте начальное значение, чтобы переменная не была «пустой» (в C++ это особенно важно).

Посмотрим на минимальный пример. Здесь мы не делаем ничего умного — просто храним возраст и печатаем его. Но именно так начинается почти любая программа: с данных.

#include 

int main() {
  int age = 18;

  std::cout << "Age = " << age << '\n'; // Age = 18
}

Обратите внимание на три момента. Во‑первых, int age = 18; читается как «создай переменную age типа int и положи туда 18». Во‑вторых, в std::cout можно печатать и текст, и числа, просто «склеивая» их оператором <<. В‑третьих, перевод строки '\n' помогает сделать вывод аккуратным.

Теперь самое важное: переменная — это не константа. Её значение можно менять с помощью присваивания.

#include 

int main() {
  int score = 0;
  std::cout << score << '\n'; // 0

  score = 10;
  std::cout << score << '\n'; // 10
}

Здесь строка score = 10; означает «замени старое значение на новое». И да: знак = в программировании обычно означает не «равно», а «положи/запиши».

2. Арифметика int и скобки в выражениях

Операции +, -, * для целых чисел

С арифметикой в int всё выглядит очень знакомо, и это приятно: сложение, вычитание и умножение работают «как вы ожидаете». Правда, есть важный нюанс: результат тоже будет целым числом, потому что мы работаем в мире int. Пока что это звучит очевидно, но на делении это выстрелит особенно громко — и мы до этого дойдём.

Начнём с простого: посчитаем сумму, разность и произведение.

#include 

int main() {
  int a = 7;
  int b = 2;

  std::cout << (a + b) << '\n'; // 9
  std::cout << (a - b) << '\n'; // 5
  std::cout << (a * b) << '\n'; // 14
}

Скобки тут не обязательны, но новичка они часто спасают: вы явно показываете, что именно хотите посчитать, и меньше шансов превратить выражение в ребус.

Ещё один полезный приём — сохранять результат вычисления в переменную. Это делает код читаемым: сначала вычислили, потом использовали.

#include 

int main() {
  int apples = 3;
  int pears = 4;

  int total = apples + pears;
  std::cout << total << '\n'; // 7
}

Скобки: для смысла и читаемости

Когда выражения становятся чуть длиннее, мозг начинает делать то же, что и компилятор: пытаться понять порядок вычислений. Проблема в том, что мозг устаёт, а компилятор — нет. Поэтому хороший стиль в программировании часто выглядит как «ставим скобки там, где человеку может быть неочевидно». Это не слабость, это забота о будущем себе (и о менторе, который будет проверять ваш код).

Сравните два варианта: формально они могут быть эквивалентны, но читаются по‑разному.

#include 

int main() {
  int x = 10;
  int y = 3;

  int r1 = x + y * 2;
  int r2 = (x + y) * 2;

  std::cout << r1 << '\n'; // 16
  std::cout << r2 << '\n'; // 26
}

Скобки меняют смысл. И это отличный повод держать в голове правило: если формула важна, лучше сделать её очевидной глазами, а не надеяться на память о приоритетах операторов.

3. Деление / и остаток % для int

Деление /: результат всегда целый

Оператор / для int делает целочисленное деление. Это значит: результат тоже int, и дробная часть не округляется «по правилам математики», а просто отбрасывается.

Проверим на классическом примере:

#include 

int main() {
  int a = 7;
  int b = 2;

  std::cout << (a / b) << '\n'; // 3
}

Почему 3, а не 3.5? Потому что int не умеет хранить .5. Он хранит только целые значения. Поэтому программа берёт целую часть и выкидывает остальное.

Это поведение удобно, когда вы считаете «сколько целых предметов помещается». Например, сколько пачек печенья по 6 штук можно собрать из 25 печенек. Но это же поведение опасно, если вы ожидали «настоящее» деление. Пока мы работаем только с int, просто зафиксируйте правило: int / int даёт int.

Отдельно отмечу, что с отрицательными числами результат тоже будет целым. В современном C++ отбрасывание дробной части идёт в сторону нуля, то есть -7 / 2 будет -3.

#include 

int main() {
  int a = -7;
  int b = 2;

  std::cout << (a / b) << '\n'; // -3
}

Если вы сейчас подумали «ммм, а остаток тогда какой?» — вы мыслите как программист. Переходим к %.

Остаток %: что осталось после деления

Оператор % читается как «остаток от деления». Он работает только для целых чисел (в нашем случае — для int). И обычно используется в задачах вида «разложить по коробкам» или «разменять сумму» или «понять, делится ли число нацело».

Самый базовый пример:

#include 

int main() {
  int a = 7;
  int b = 2;

  std::cout << (a % b) << '\n'; // 1
}

То есть 7 апельсин поделить на 2 человек будет 3 апельсина каждому и 1 в остатке.

Полезно помнить связку деления и остатка в виде формулы: a = (a / b) * b + (a % b)

Это не магия и не заклинание, а просто аккуратный способ сказать: «число a состоит из целого количества групп по b плюс остаток».

Давайте это прям проверим кодом:

#include 

int main() {
  int a = 25;
  int b = 6;

  int q = a / b;  // сколько полных групп
  int r = a % b;  // сколько осталось

  std::cout << q << '\n'; // 4
  std::cout << r << '\n'; // 1
}

Пока что это выглядит как «игрушка», но через пару задач вы начнёте видеть % везде: время на часах, чётность/нечётность, разбиение на страницы, перевод секунд в минуты, и так далее.

4. Деление на ноль недопустимо

В математике деление на ноль — не определено. В программировании это превращается в практическую проблему: выражения a / 0 и a % 0 недопустимы. Программа либо аварийно завершится, либо попадёт в совсем непредсказуемое поведение (что ещё хуже, потому что ошибки могут проявиться «где-то потом»).

Здесь важен не страх, а привычка: если у вас есть делитель, который теоретически может стать нулём, вы должны это контролировать. Пока мы ещё не проходили if, поэтому сегодня правило простое: в примерах и первых задачах не делим на ноль и держим в голове, что делитель должен быть ненулевым.

5. Мини‑приложение: «Автомат с монетами»

Сейчас мы напишем маленькую программу, которая выглядит как «часть реального мира»: у нас есть сумма денег и цена товара, и мы хотим узнать две вещи. Во‑первых, сколько товаров можно купить. Во‑вторых, сколько денег останется. Это идеальная задача для пары / и %.

Важно: пока мы не умеем вводить данные, поэтому числа будут «захардкожены» в коде. Это нормально: сегодня наша цель — не ввод, а арифметика.

Версия 1: считаем количество и сдачу

Представим, что у нас 275 центов (то есть $2.75), а печенье стоит 60 центов. Считаем.

#include 

int main() {
  int moneyCents = 275;
  int cookiePrice = 60;

  int cookies = moneyCents / cookiePrice;
  int change = moneyCents % cookiePrice;

  std::cout << cookies << '\n'; // 4
  std::cout << change << '\n';  // 35
}

Тут прекрасно видно, зачем нам переменные: один раз задали цену и сумму, а дальше используем их в вычислениях и можем легко поменять сценарий, не переписывая всю программу.

Версия 2: делаем вывод понятнее

Пока вывод у нас «компьютерный»: просто числа в столбик. Давайте добавим подписи, чтобы стало похоже на нормальную консольную программу. Мы не используем строки как тип (это будет следующая лекция), но печатать текст в кавычках мы уже умеем.

#include 

int main() {
  int moneyCents = 275;
  int cookiePrice = 60;

  int cookies = moneyCents / cookiePrice;
  int change = moneyCents % cookiePrice;

  std::cout << "Cookies: " << cookies << '\n'; // Cookies: 4
  std::cout << "Change: " << change << " cents\n"; // Change: 35 cents
}

Да, это уже похоже на приложение, а не на контрольную по арифметике.

Версия 3: ещё одна покупка

Допустим, после покупки печенек вы ещё хотите купить жвачку за 35 центов. У нас как раз есть сдача. Посчитаем, сколько жвачек можно купить на сдачу (это, кстати, тоже / и %, только с другими числами).

#include 

int main() {
  int moneyCents = 275;
  int cookiePrice = 60;
  int gumPrice = 35;

  int cookies = moneyCents / cookiePrice;
  int changeAfterCookies = moneyCents % cookiePrice;

  int gums = changeAfterCookies / gumPrice;

  std::cout << "Cookies: " << cookies << '\n'; // Cookies: 4
  std::cout << "Gums: " << gums << '\n';       // Gums: 1
}

Здесь особенно видно, что переменные помогают держать «сюжет» программы. Мы не просто считаем числа, мы описываем процесс.

6. Типичные ошибки при работе с int, / и %

Ошибка №1: ожидать, что 7 / 2 даст 3.5.

Это одна из самых частых ловушек у новичков: мозг помнит школьную математику, а программа работает в мире int. В целочисленном делении дробная часть отбрасывается, потому что int не умеет хранить половинки. Если вам нужна дробь — это отдельная тема, но сегодня мы честно живём в мире целых чисел и принимаем его правила.

Ошибка №2: использовать % «для всего подряд».

Остаток — это не «крутая кнопка», а конкретная операция со смыслом «что осталось после деления на равные группы». Если вы не можете объяснить словами, что именно остаётся, скорее всего, % вы применяете наугад. А угадывание в программировании работает примерно так же, как «угадывание пароля»: иногда получается, но лучше не надо.

Ошибка №3: забыть, что a % 0 и a / 0 недопустимы.

Деление на ноль не просто «даёт ошибку». Оно ломает саму логику вычисления. Поэтому правило простое: делитель (то, на что делим) должен быть ненулевым. Позже, когда появятся условия (if), вы научитесь защищать программу от такого ввода. Сейчас же — просто не пишите такого кода и не подставляйте 0 в делитель.

Ошибка №4: писать длинные выражения без скобок и потом не понимать результат.

Формально компилятор разберётся, но человек — не всегда. В итоге вы получаете ситуацию «вроде всё правильно, но почему ответ не тот». Скобки — дешёвый способ сделать смысл выражения очевидным и для вас, и для того, кто будет читать код. В программировании это называется не занудством, а профилактикой багов.

Ошибка №5: путать “создать переменную” и “изменить переменную”.

int x = 10; — это создание переменной с начальным значением. x = 10; — это присваивание уже существующей переменной. Если попытаться «создать» переменную второй раз с тем же именем в том же месте, компилятор справедливо возмутится: «так, минутку, у нас тут уже есть x».

Лекция 2. Строки — std::string, конкатенация, длина

1. Введение

Когда вы только начинаете программировать, легко попасть в иллюзию, что «главное — числа». Ну правда: оценки, деньги, возраст, счётчики — везде цифры. Но как только вы захотите сделать программу хоть немного дружелюбной к человеку, внезапно выясняется: человек общается словами. Имя, фамилия, город, комментарий к заказу, название фильма — всё это текст, то есть строка.

Когда С++ только появился, то типа строка (string) в нем не было - строку воспринимали просто как набор символов (тип char). Однако со временем выяснилось, что это очень неудобно, и было решено создать отдельный тип string и добавить его в стандартнутю (STD) библиотеку С++. Так в C++ появился тип std::string.

Его можно воспринимать как контейнер для последовательности символов: внутри лежат буквы, пробелы, знаки пунктуации. И, что особенно приятно, стандартная библиотека даёт нам готовые операции для строк: склеивать, узнавать длину, брать символ по номеру.

Чтобы покачать работу с std, мы будем развивать одно небольшое приложение по мере курса: «консольная визитка». Пока оно будет просто собирать красивую строку визитки и печатать её. Сегодня мы добавим туда имя и пару текстовых деталей.

2. Первая строка: подключаем <string>

Сейчас будет момент, который сначала кажется мелочью, а потом становится второй натурой: чтобы использовать std::string, обычно нужен заголовок <string>. Да, в некоторых компиляторах «как-то работает и без него», но это плохая привычка: лучше подключать то, что вы используете.

Начнём с простого: создадим строку из строкового литерала "..." и выведем её.

#include 
#include 

int main() {
  std::string course = "C++ for beginners";
  std::cout << course << '\n'; // C++ for beginners
}

Здесь важна идея: std::string — это тип, как int, только для текста. Мы объявили переменную course и положили в неё текст. Дальше она ведёт себя как «значение», которое можно передавать в вывод.

Строка "A" и символ 'A'

В программировании есть шутка: «две кавычки решают судьбу проекта». И, к сожалению, это почти правда. В C++ есть строковые литералы в двойных кавычках "..." и символьные литералы в одинарных '...'. Разница принципиальная: строка — это последовательность символов, а символ — это ровно один символ.

Если вам нужно хранить имя "Ada", это строка. Если вам нужен один символ, например первая буква имени 'A', это символ. std::string хранит строку, а тип для одного символа мы пока воспринимаем просто как «один знак» (формально это char, но об этом позднее).

Сравните:

#include 
#include 

int main() {
  std::string word = "Cat";                   // строка текста
  char firstLetter = 'C';                     // один символ

  std::cout << word << '\n';        // Cat
  std::cout << firstLetter << '\n'; // C
}

На вид похоже, но смысл другой: word может быть любой длины, а firstLetter — строго один символ.

3. Конкатенация: + и +=

Переходим к очень «человеческой» операции: собрать из кусочков одну красивую фразу. В строках это называется конкатенацией. Представьте, что строка — это конструктор LEGO, только вместо кубиков слова и пробелы. А вы собираете предложение, которое потом покажете пользователю.

Есть два основных способа склеивания:

  • a + b создаёт новую строку (как будто вы взяли два листочка и сделали третий, где написано всё вместе).
  • a += b дописывает в существующую строку (как будто вы продолжили писать на том же листочке).

Начнём с +:

#include 
#include 

int main() {
  std::string first = "Ada";
  std::string last = "Lovelace";

  std::string full = first + " " + last;
  std::cout << full << '\n'; // Ada Lovelace
}

Обратите внимание на " ". Пробел — это тоже символ, а часто и смысл. Если забыть пробел, получится AdaLovelace, что звучит как имя класса, но не как имя человека.

Теперь вариант с += — он удобен, когда вы строите строку постепенно:

#include 
#include 

int main() {
  std::string badge = "Name: ";
  badge += "Ada";
  badge += " ";
  badge += "Lovelace";

  std::cout << badge << '\n'; // Name: Ada Lovelace
}

С точки зрения читаемости это похоже на «собираем строку шаг за шагом». В будущем, когда появятся условия и циклы, += будет особенно полезен, но даже сейчас он хорошо показывает идею «дописывания».

Небольшая важная ремарка: строку и число нельзя просто так «плюсовать» как строки. Например, "Age: " + 10 — так не работает, потому что это разные типы. Мы пока не будем углубляться в преобразования; просто фиксируем: строки склеиваются со строками.

4. Длина строки: size()

Когда вы держите в руках строку, часто хочется понимать, сколько в ней символов. Для этого у std::string есть метод size() (иногда также встречается length(), но сегодня достаточно size()). Это как спидометр, который показывает «сколько символов сейчас в строке».

Посмотрим в деле:

#include 
#include 

int main() {
  std::string city = "Boston";
  std::cout << city.size() << '\n'; // 6
}

Важно привыкнуть к мысли: size() — это первое, что вы вспоминаете перед тем, как полезть внутрь. Потому что индексация без понимания длины — это как идти по лестнице в темноте, считая, что ступеньки бесконечные. Спойлер: ступеньки конечные, а колено одно.

Ещё один пример, где длина помогает увидеть, что пробел — тоже символ:

#include 
#include 

int main() {
  std::string s = "Hi Bob";
  std::cout << s.size() << '\n'; // 6  (H i _ B o b)
}

5. Индексирование: s[i]

Как вы уже выяснили - стока это набор символов. А раз это набор символов, то можно получить отдельные символы из стоки по их номеру.

В программировании почти вся нумерация идет с нуля. Нумерация с нуля означает: первый символ — это s[0], второй — s[1], и так далее.

Представим строку как ящик с ячейками:

s = 'c a t' // индексы: 0 1 2

Посмотрим на код:

#include 
#include 

int main() {
  std::string animal = "cat";

  std::cout << animal[0] << '\n'; // c
  std::cout << animal[1] << '\n'; // a
  std::cout << animal[2] << '\n'; // t
}

Про границы

Чтобы меньше путаться, в программировании приятно использовать термин индекс вместо номер. Индекс всегда начинается с нуля.

Индекс должен быть в диапазоне от 0 до s.size() - 1. Если строка длины 3, допустимые индексы — 0, 1, 2. Если попытаться взять animal[3], вы выходите за границы строки и программа падает.

В C++ это опасно тем, что программа может вести себя непредсказуемо: иногда «как будто работает», иногда печатает странные символы, иногда падает. И это как раз тот случай, когда «иногда работает» хуже, чем «сразу падает», потому что баг прячется и ждёт подходящего момента.

Можно ли менять символы в строке?

Да, символ по индексу можно заменить: строка становится изменяемой, и это удобно для простых правок.

#include 
#include 

int main() {
  std::string word = "bat";
  word[0] = 'c';

  std::cout << word << '\n'; // cat
}

Обратите внимание: справа стоит 'c' в одинарных кавычках, потому что мы присваиваем один символ, а не строку "c".

6. Первое знакомство с методами

Вы могли заметить странную запись:

city.size()

Почему мы не пишем просто size(city)? Потому что size() — это метод.

Метод — это функция, которая принадлежит конкретному типу или объекту. В нашем случае std::string — это тип, а city — объект этого типа. И у строк есть свои «встроенные умения»: узнавать длину, дописывать текст, получать символ по индексу и многое другое.

Когда функция принадлежит объекту, к ней обращаются через точку:

объект.метод()

Точка читается почти по-человечески: «Возьми объект city и вызови у него метод size()».

То есть строка как бы «сама знает», как определить свою длину. Это и есть идея методов: поведение связано с данными.

Посмотрите на уже знакомые примеры:

std::string full = first + " " + last;

std::cout << full.size() << '\n';
std::cout << full[0] << '\n';

full.size() — строка сама сообщает свою длину. full[0] — мы обращаемся к её внутреннему символу.

Позже вы увидите, что почти все сложные типы в C++ имеют свои методы. Например, у std::vector есть push_back(), у std::string есть substr(), у потоков есть clear(). Это один из ключевых принципов языка: данные и связанные с ними операции объединены.

Важно привыкнуть к этой записи. Всё, что выглядит как:

что_то.что_то()

почти всегда означает вызов метода объекта.

Именно поэтому мы пишем full.size(), а не просто size(full). Строка сама предоставляет инструменты для работы с собой.

7. Мини-приложение: консольная визитка

Теперь соберём всё в один небольшой цельный фрагмент: сделаем визитку, которая печатает имя, длину полного имени и первую букву. Пока данные возьмём прямо из кода (ввод будет в следующей лекции), чтобы не смешивать новые темы в одну кучу.

В этом примере у нас появится «ощущение приложения»: не просто отдельные операции, а маленький полезный вывод.

#include 
#include 

int main() {
  std::string first = "Ada";
  std::string last = "Lovelace";

  std::string full = first + " " + last;

  std::cout << "Full name: " << full << '\n';          // Full name: Ada Lovelace
  std::cout << "Length: " << full.size() << '\n';      // Length: 12
  std::cout << "First letter: " << full[0] << '\n';    // First letter: A
}

Здесь спрятано сразу несколько важных привычек.

Во-первых, мы явно добавили пробел между именем и фамилией, потому что компьютер не догадается, что «между словами должно быть место». Во-вторых, мы вывели длину строки: это помогает понять, что пробел тоже считается. В-третьих, мы взяли full[0] и достали первый символ, что даёт нам в будущем возможность делать инициалы или простые сокращения.

Если хочется чуть-чуть «поиграться», можно достать последнюю букву фамилии. Формально последняя буква имеет индекс size() - 1. Мы делаем это только на заведомо непустой строке, чтобы не рисковать выходом за границы:

#include 
#include 

int main() {
  std::string last = "Lovelace";
  char lastChar = last[last.size() - 1];

  std::cout << lastChar << '\n'; // e
}

Да, выражение выглядит слегка «математически», но логика простая: если длина 8, индексы идут 0…7, значит последний — 7, то есть size() - 1.

8. Типичные ошибки при работе со std::string

Ошибка №1: забыли #include <string>.

Иногда код «вроде бы» компилируется из‑за косвенных подключений, а иногда внезапно перестаёт. Правильная привычка простая: используете std::string — подключайте <string>. Это не прихоть, а реальная дисциплина: даже в примерах и документах встречаются правки, где добавляют недостающий include, потому что иначе пример некорректный.

Ошибка №2: перепутали "A" и 'A'.

Когда вы пишете word[0] = "b";, компилятор вам справедливо скажет: «эй, слева один символ, а справа строка». Для одного символа используйте одинарные кавычки: 'b'. Для текста — двойные: "b".

Ошибка №3: забыли про пробел при конкатенации.

Классика: first + last даёт “AdaLovelace”, а вы потом долго смотрите на результат и думаете, что программа сломалась. На самом деле она слишком буквальная: не попросили пробел — не добавила. Добавляйте " " явно.

Ошибка №4: индекс вышел за границы строки.

s[100] у строки "Hi" — это не «пустота», не null и не «просто ничего». Это выход за границы, и поведение программы становится непредсказуемым. До появления if мы просто не делаем такие индексы «наугад» и всегда опираемся на size().

Ошибка №5: ожидали, что size() — это «количество букв в любом языке».

На старте курса мы обычно работаем с простыми латинскими примерами, где один символ примерно равен одной «букве». В реальных текстах (особенно с кириллицей в UTF‑8) это может быть сложнее, но пока наша задача — освоить базовую механику строк, а не войну с кодировками.

Лекция 3. Ввод: std::cin >> и его ограничения

1. Поток std::cin и чтение токенов оператором >>

Когда программа только печатает, она похожа на радио: говорит что-то в пустоту и надеется, что кто-то слушает. Но как только появляется ввод, программа становится похожа на диалог: вы даёте данные, она отвечает. В C++ ввод обычно делают через std::cin. Очень полезно представить cin как ленту (поток) символов: там лежит то, что пользователь напечатал, включая пробелы и переводы строк.

Самое важное: оператор >> читает не «всё подряд», а одну порцию данных за раз. И эта «порция» почти всегда ограничена пробельными символами.

std::cin и оператор >>: читаем значение в переменную

Чтобы что-то прочитать, нам нужна переменная, куда это «что-то» положить. Логика такая: объявили переменную → прочитали в неё данные → используем дальше (например, выводим).

Минимальный пример: читаем одно целое число.

#include 

int main() {
  int age = 0;

  std::cin >> age;
  std::cout << "Age = " << age << '\n'; // если ввели 20, будет: Age = 20
}

Здесь важно заметить две вещи. Во‑первых, std::cin >> age; пытается прочитать следующее значение из ввода и записать его в age. Во‑вторых, тип переменной важен: раз age — это int, программа ожидает, что во входе сейчас «лежит» число (например 20, -5, 0).

Что именно читает >>: токен и пробельные разделители

Сейчас будет слово «токен», но не пугайтесь: это всего лишь «кусочек текста без пробелов». Оператор >> работает по идее: «Пропусти пробелы, потом возьми подряд символы до следующего пробела». Пробелом при этом считается не только ' ' (обычный пробел), но и перевод строки '\n', и табуляция '\t'.

Представьте, что пользователь ввёл такую строку (символ ⏎ — это Enter, то есть '\n'):

12 34 ⏎

Между 12 и 34 тут три пробела. Для >> это не важно: он видит «число», потом «пробелы», потом «число».

#include 

int main() {
  int a = 0;
  int b = 0;

  std::cin >> a >> b;
  std::cout << "a=" << a << ", b=" << b << '\n'; // при вводе "12   34" будет: a=12, b=34
}

Обратите внимание: std::cin >> a >> b; — это два чтения подряд, просто записанные в одну строку. Сначала читается a, затем читается b.

Разделители: пробел, Enter и табуляция одинаково «режут» токены

Иногда новичок ожидает, что Enter «сильнее», чем пробел, или что пробел «просто для красоты». Для operator>> всё наоборот: любой пробельный символ — граница токена, и точка.

Давайте представим такой ввод (каждое слово на новой строке):

Ivan⏎
Ivanov⏎

И такой ввод (оба слова на одной строке):

Ivan Ivanov⏎

Для кода ниже результат будет одинаковым, потому что и пробел, и Enter — это разделители токенов.

#include 
#include 

int main() {
    std::string first;
    std::string last;

    std::cin >> first >> last;
    std::cout << first << " | " << last << '\n'; // Ivan | Ivanov
}

Это хорошая новость: можно вводить данные «как удобно» (в одну строку или несколько). Но это же и причина ограничений, о которых мы поговорим дальше.

Мини-схема: как >> «режет» ввод на токены

Чтобы мозг перестал воспринимать ввод как магию, полезно один раз увидеть модель. Пусть пользователь ввёл вот так:

23 Ada Lovelace⏎

Тогда поток символов можно мысленно разрезать на токены:

23 Ada Lovelace

А пробелы и перевод строки — это «разделительная каша», которая просто пропускается между токенами.

Очень практическое следствие: если вы читаете int, а потом std::string, то int «съест» первый токен, а std::string — следующий. Никаких договорённостей «вводите с новой строки» на уровне >> не существует — он всё равно режет по пробелам.

2. Практика: мини‑анкета и строки без пробелов

Практический пример: делаем мини-анкету

С этого момента будем вести один и тот же мини‑проект: консольную анкету. Он простой, зато на нём удобно наращивать навыки: сегодня читаем токены, чуть позже научимся читать целые строки и аккуратно обрабатывать ввод.

Версия 1: читаем возраст и имя (как одно слово).

#include 
#include 

int main() {
  int age = 0;
  std::string name;

  std::cin >> age >> name;

  std::cout << "User: " << name << '\n';
  std::cout << "Age: " << age << '\n';
}

Если ввести:

20 Ivan

то программа выведет:

User: Ivan
Age: 20

Можно ввести и так (через Enter):

20
Ivan

С точки зрения >> это одно и то же: токены всё равно идут последовательно.

Ввод строк через >>: читается слово, а не фраза

Теперь мы подошли к классическому сюрпризу: std::cin >> s для std::string читает только одно слово, то есть токен до пробела/Enter/табуляции. Если вы введёте несколько слов, они не пропадут, они просто останутся «лежать» во входном потоке и будут прочитаны следующими операциями >>.

Демонстрация на нашей анкете: читаем «город» как строку.

#include 
#include 

int main() {
  std::string city;

  std::cin >> city;
  std::cout << "City=[" << city << "]\n";
}

Если пользователь введёт:

New York

то city станет "New", а слово "York" останется во входном потоке как следующий токен. Программа выведет:

City=[New]

Это не баг и не «кривой C++». Это нормальная логика оператора >>: он читает один токен.

Показательный эксперимент: «потерянное» слово на самом деле не потеряно

Слово, которое не поместилось в первую строковую переменную, обычно не исчезает. Оно просто ждёт своей очереди. Покажем это честно: прочитаем два слова в две переменные.

#include 
#include 

int main() {
  std::string w1;
  std::string w2;

  std::cin >> w1 >> w2;

  std::cout << "w1=[" << w1 << "], w2=[" << w2 << "]\n";
}

Если ввести:

New York

получим:

w1=[New], w2=[York]

То есть >> отлично подходит для форматов «вводятся отдельные слова» (имя, фамилия, команда, логин). Но для полей вроде «адрес» или «название книги» он неудобен, потому что там почти всегда есть пробелы.

Памятка: что считается разделителем для >>

Когда вы начнёте писать небольшие программы, вы постоянно будете ловить себя на мысли «почему оно остановилось здесь?». Обычно ответ в таблице ниже:

пробел ' ' Ivan Ivanov разделитель строка через >> прочитает только Ivan

перевод строки '\n' (Enter) Ivan⏎Ivanov разделитель поведение такое же, как с пробелом

табуляция '\t' Ivan\tIvanov разделитель токены тоже разделяются

Вывод простой: >> читает «до ближайшего пробельного символа».

Как выбрать формат ввода под std::cin >>

Когда вы выбираете std::cin >>, вы по сути выбираете формат: «пользователь вводит значения отдельными токенами». Это идеально подходит для задач, где вход выглядит как «два числа», «три числа», «слово и число», «имя и фамилия», «команда и параметр».

Например, наша анкета хорошо ложится на токены, если мы заранее договорились, что имя и фамилия вводятся отдельно, а не одной строкой. Тогда приложение можно расширить аккуратно: читаем имя, фамилию и возраст.

#include 
#include 

int main() {
  std::string first;
  std::string last;
  int age = 0;

  std::cin >> first >> last >> age;

  std::cout << "Profile: " << first << " " << last << '\n';
  std::cout << "Age: " << age << '\n';
}

Если ввести:

Ada Lovelace 36

то всё отлично читается: три токена → три переменные.

Микро-нюанс: >> не «съедает» пробелы как данные

Когда мы говорим «пробел — разделитель», важно понимать: пробелы не попадают в строку, которую читает >>. Он читает только «значимые символы токена». Поэтому std::string word; std::cin >> word; никогда не прочитает строку "Hello world" целиком: пробел как символ внутри токена невозможен по определению токена.

Для сравнения, если бы вы хотели читать всю строку целиком, включая пробелы, нужен другой инструмент. Но мы здесь сознательно не забегаем вперёд: пока наша задача — уверенно владеть >> и не ждать от него того, чего он не обещал.

3. Ограничения >>: несоответствие типа и безопасное чтение текста

Когда тип важнее ваших желаний: чтение может «сломаться»

Вторая большая группа ограничений — несоответствие типа. Если вы читаете int, а пользователь вводит «слово», программа не может превратить «котик» в число по‑честному.

Покажем сценарий, который выглядит правдоподобно: человек перепутал и ввёл возраст словами.

#include 
#include 

int main() {
  int age = 0;
  std::string name;

  std::cin >> age >> name;

  std::cout << "age=" << age << ", name=" << name << '\n';
}

Если ввести:

twenty Ivan

то программа уже не сможет корректно прочитать age как число. Мы пока не умеем красиво чинить такой ввод (это будет отдельная тема позже в курсе). Поэтому сейчас правило простое и честное: когда вы пишете задачи, где формат ввода задан, предполагается, что пользователь вводит данные в правильном формате.

Почему мы читаем строки в std::string, а не в char[]

Иногда в старых примерах по C/C++ можно встретить что-то вроде char name[20]; std::cin >> name;. Это работает, но опасно: если пользователь введёт слишком длинное слово, можно получить проблемы вплоть до переполнения буфера.

Мы пока не изучали массивы и указатели, поэтому держим практическое правило для новичка: для текста используйте std::string, она значительно безопаснее как «контейнер для букв».

4. Типичные ошибки при работе с std::cin >>

Ошибка №1: ожидать, что std::cin >> s; прочитает целую фразу.

Это одна из самых частых ловушек. Новичок вводит New York, а в переменной оказывается только New. Причина проста: >> читает токен до пробельного разделителя. Если по смыслу вам нужна строка «с пробелами», значит, по выбранному формату ввода >> не подходит.

Ошибка №2: не объявить переменную до чтения.

\

Иногда хочется написать что-то вроде «прочитай возраст» без переменной, но std::cin не умеет читать «в никуда». Сначала объявляется переменная нужного типа (int age = 0;), а только потом выполняется std::cin >> age;. Это не формальность: переменная — это место в памяти, куда попадёт результат.

Ошибка №3: перепутать порядок чтения и порядок данных.

Если программа делает std::cin >> age >> name;, а пользователь вводит Ivan 20, то age попытается стать Ivan (что не число) и чтение пойдёт не так. У >> нет телепатии: он читает токены строго по порядку и пытается засунуть их в переменные указанного типа.

Ошибка №4: думать, что Enter «заканчивает ввод для программы».

Enter всего лишь добавляет символ '\n' во входной поток. Для >> это такой же пробельный разделитель, как обычный пробел. Поэтому «ввод на новой строке» и «ввод через пробел» чаще всего эквивалентны. Это удобно, но иногда ломает ожидания, если вы мысленно строили ввод «строго построчно».

Ошибка №5: вводить не тот тип и удивляться странным результатам.

Если ожидается int, ввод вроде ten не может быть корректно преобразован в число оператором >>. На этом этапе курса мы ещё не чиним такой ввод «красиво», поэтому лучшее решение для учебных задач — внимательно следить за форматом данных и вводить значения в том виде, в каком их ждёт программа.

Лекция 4. Ввод строк — std::getline

1. Знакомство с std::getline

Если вы только что освоили std::cin >>, может показаться, что он умеет всё: вводишь “слово”, вводишь число, всё работает. Но ровно до момента, когда пользователь вводит фразу, ФИО, название книги или адрес — то есть текст с пробелами. И тут >> внезапно ведёт себя как слишком буквальный робот: видит пробел и считает, что “ну всё, слово закончилось, я ухожу”.

Представьте, что std::cin >> — это турникет в метро: он пропускает ровно одного пассажира (токен) и закрывается на первом же “препятствии” вроде пробела или '\n'. А std::getline — это дверь в вагон: вы заходите целиком, со всеми сумками-пробелами, и никто не отрезает вам фамилию на середине.

Ключевая идея: std::cin >> читает “одно слово/число” (токен), а std::getline читает строку целиком до конца строки.

Как работает std::getline

Когда вы хотите прочитать всю строку, включая пробелы, нужен std::getline. Это функция из стандартной библиотеки, обычно используется вместе с std::string. Важно: строка читается до символа перевода строки '\n', но сам '\n' в результат не попадает — он как граница, а не часть текста.

Минимальный шаблон выглядит так:

#include 
#include 

int main() {
  std::string line;
  std::getline(std::cin, line);

  std::cout << "Line: " << line << '\n';
}

Здесь есть несколько очень важных моментов.

Во‑первых, std::getline может прочитать даже пустую строку — это не ошибка и не “сломалось”, это вполне легальный ввод.

Во‑вторых, std::getline не боится пробелов: если пользователь введёт Ivan Ivanov, строка так и останется Ivan Ivanov, не превратится в одно Ivan.

Чтобы увидеть границы более явно (особенно когда строка пустая), удобно печатать строку в квадратных скобках:

#include 
#include 

int main() {
  std::string line;
  std::getline(std::cin, line);

  std::cout << "[" << line << "]\n"; // например: [Hello world]
}

2. Пустые строки и как их заметить

Пустая строка — это ситуация, когда пользователь просто нажал Enter и не ввёл ни одного символа. Новички часто воспринимают это как “getline не работает”, но на самом деле это такая же строка, просто длины 0. В программировании пустота — тоже данные (иногда даже самые честные).

Давайте сделаем пример, который явно покажет длину строки. Так вы точно отличите “пусто” от “не прочиталось”:

#include 
#include 

int main() {
  std::string line;
  std::getline(std::cin, line);

  std::cout << "Text=[" << line << "]\n";
  std::cout << "Size=" << line.size() << '\n'; // Size=0 для пустой строки
}

Обратите внимание: line.size() возвращает количество символов. Если пользователь ввёл Hi, размер будет 2. Если он просто нажал Enter — будет 0.

Пустые строки могут появляться в двух основных сценариях.

  • Первый — пользователь реально ввёл пустую строку (нажал Enter).
  • Второй — более коварный: вы смешали std::cin >> и std::getline, и getline схватил “остаток” перевода строки.

Это как если вы пришли за пиццей, а вам вынесли коробку… но пустую, потому что пиццу уже кто-то съел на предыдущей лекции. Сейчас разберём этот сюжет.
