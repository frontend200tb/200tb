about

location — это объект хранящийся в window, который позволяет получать информацию о текущем адресе страницы и менять его с помощью функций или обновления полей объекта.

Для навигации по сайту мы используем адреса и параметры страницы. window.location содержит набор свойств и методов, чтобы удобно получать адрес и управлять им.

Свойства windows.location

  1. window.location.href

    href – полное представление адреса. Можно сказать, что это зеркало того, что находится в адресной строке браузера в данный момент. Если записать значение в это свойство, то произойдёт обновление адреса и редирект на новый адрес. Остальные свойства — это кусочки location.href. Они нужны, чтобы удобно получать каждый из них отдельно, а не вытаскивать их руками из полной строки адреса.

    window.location.href = 'https://example.com:3000/profile/edit?key=1&key=2#user-info'

  2. window.location.protocol

    protocol содержит текущий протокол по которому открыта страница. Чаще всего там будет https и http.

    window.location.protocol // 'https:'

  3. window.location.host

    host содержит значение хоста из ссылки. Хост включает в себя название текущего домена и порта.

    window.location.host // 'example.com:3000'

  4. window.location.hostname

    hostname — частичка с доменом из свойства host, не включает в себя порт.

    window.location.hostname // 'example.com'

  5. window.location.port

    port — вторая составляющая значения host, содержит только номер порта. Если порт не указан в явном виде, то значением свойства будет пустая строка ''.

    window.location.port // 3000

  6. window.location.origin

    origin включает в себя путь, начиная с protocol и заканчивая port.

    window.location.origin // 'https://example.com:3000'

  7. window.location.pathname

    pathname – репрезентация текущего пути на сайте. Если текущий урл не содержит путь, то значением будет корневой путь "/".

    window.location.pathname // '/profile/edit/'

  8. window.location.search

    search содержит параметры в формате ключ=значение разделённые &. Если параметров нет, то значением будет пустая строка.

    window.location.search // '?key=1&key=2'

  9. window.location.hash

    hash — якорная ссылка включая символ #. Она находится в самом конце пути и отвечает за навигацию между размеченными на странице элементами с помощью установки атрибута id на тегах. Эта часть URL не передаётся на сервер. Если параметров нет, то значением будет пустая строка.

    window.location.hash // '#user-info'

Методы windows.location

  1. window.location.assign(url)

    assign(url) – метод вызывает переход на страницу, переданную в аргументах. После перехода на страницу пользователь может вернуться на страницу, с которой произошёл переход, с помощью браузерной кнопки назад.

  2. window.location.reload()

    reload() перезагружает текущую страницу. Если она есть в кеше, то из кеша. То же самое что и нажать на кнопку "F5" или "обновить" в браузере

    reload(true) перезагружает текущую страницу с сервера. То же самое что и нажать на CTRL + F5

  3. window.location.replace(url)

    replace(url) аналогичен методу assign(), но адрес страницы с которой был вызван этот метод не будет сохранён в истории браузера. При попытке вернуться назад пользователь будет отправлен на страницу предшествующую той, с которой произошёл переход.

  4. window.location.toString()

    toString() приводит адрес страницы к строке. Возвращает то же самое значение, что и location.href.
