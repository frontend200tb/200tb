5.1 Быстрое считывание

Мы научимся более быстрому вводу-выводу. Изучим битовое представление чисел различных типов памяти и битовые операции.

Начнем с ускорения ввода-вывода. Есть два способа это сделать. Функции ввода-вывода cin и cout являются не единственным средством ввода-вывода в языке C++.

Исторически, еще в языке Си, первой появилась библиотека ввода-вывода stdio.h или cstdio, в которой для ввода-вывода были функции printf() и scanf(). Буква f в названии функции расшифровывается как форматный. Первым параметром в каждой функции является строка, задающая формат вводимых-выводимых данных.

Функция printf()

В функции printf() в форматной строке может передаваться обычный текст, который выводится без изменений, и параметры вывода переменных. Каждому выводимому значению должен соответствовать ровно один параметр. Порядок параметров должен соответствовать порядку переменных. Выводимые переменные передаются в функцию printf() начиная со второго параметра.

Вывод строки без параметров

printf("Hello, World\n");

Вывод строки с одним параметром

printf("Answer: %d", x);

Вывод строки с двумя параметрами. Здесь каждое число выводится двумя символами. Если число однозначное то первым символом выводится 0.

printf("Current time: %02d:%02d", m, s);

Вывод строки с тремя параметрами

printf("%d %d %d", val1, a[3], val2);

Как видно из примеров, параметры задаются последовательностью символов, начинающихся со знака %. Параметр обозначает в каком формате выводить переданную переменную.

%d выводит как тип int

%lld выводит как тип long long

%u llu выводит как беззнаковый тип int и long long

%lf выводит как тип double

%llf выводит как тип long double

%.20lf выводит как тип double с точностью 20 знаков после запятой

%c выводит как тип char

%s выводит как тип string (только для строк c-style)

Функция printf() не работает с современными строками. Сначала строку надо привести в Си-шную строку методом s.c_str();

Функция scanf()

Основные параметры устроены так же как и в функции printf(). Однако есть особенности при передачи переменных. Так как нам требуется занести считанные данные в переменные, а в C/C++ переменные передаются по значению, то для того чтобы scanf() мог изменить значение переменных, в функцию scanf() надо передать не значение переменной, а ее адрес.

scanf("%d", &a); scanf("%d", &arr[i]);

Так как массив имеет тип указатель, то можно записать по другому.

scanf("%d", &(arr + i));

Если два параметра идут подряд %d%d то между целыми числами, перед первым числом, а также после второго, принимается любое количество пробельных символов. Так как scanf() воспринимает все параметры как токены - объекты ограниченные с двух сторон пробельными символами или началом и концом входных данных.

scanf("%d%d", &a, &b);

Функция scanf() возвращает число удачно считанных параметров. При первом же неудачно считанном параметре, считывание прекращается. Это можно использовать, например, считывая целые числа до конца файла. Напишем цикл while, который будет считывать одно число, пока считывание возможно.

while (scanf("%d", &a) == 1) {}

Ускорение cin/cout

Операторы ввода-вывода cin/cout устроены таким образом, что вы в программе можете попеременно использовать cin/cout, а также Си-шный scanf/printf. За такую возможность приходится платить скоростью работы cin/cout. Синхронизацию между этими двумя способами ввода-вывода можно отключить используя следующую команду

ios_base::sync_with_stdio(0);

После этой команды вы не можете в программе одновременно использовать Си-шный и Си++-ный способы ввода.

C++ использует буфер, в который пишет куски вывода, прежде чем реально его вывести на экран. Такое поведение может сильно ускорить программу, особенно если в ней много операций вывода. Вывод - тяжелая операция. И выводить мало раз, но большими кусками оптимальнее, чем много раз маленькими. Поэтому сброса буфера лучше всего избегать. Чаще всего буфер сбрасывает вывод перевода строки при помощи endl. Вместо него лучше использовать символ '\n'.

Однако это не единственное место, где буфер может сброситься. Еще одним примером сброса буфера является синхронизация между считыванием и выводом. Пользователю будет гораздо удобнее видеть весь вывод программы, прежде чем он введет очередную переменную. Однако такая синхронизация также замедляет ввод-вывод. Особенно если в вашей программе они чередуются. Ее тоже можно отключить следующими командами

cin.tie(0); cout.tie(0);

Оцениваем производительность

Сравним как быстро работают данные методы ввода-вывода. Напишем 3 программы, которые считывают массив и выводят его в обратном порядке.

5.2 Системы счисления

Позиционная система счисления.

Формально, позиционная система счисления с целым основанием b > 1, это представление каждого целого неотрицательного числа в виде многочлена

x = a0 + a1*b + a2*b2 + ... + an*bn

где ai - целые числа между 0 и b-1 включительно. Причем an > 0.

Существование и единственность представления

Покажем что такое разбиение существует и единственное.

Перепишем

a0 + a1*b + a2*b2 + ... + an*bn

как

a0 + b*(a1 + b*(a2 + ... + an*b)...)

Тогда x = a0 + b*r0, где r0 = a1 + b*(a2 + ... + an*b)...). r0 - целое, a0 - неотрицательно и строго меньше b, то есть a0 - остаток от деления x на b, r0 - частное (а значит определяются однозначно).

Переходя от x к r0, аналогично представляем r0 как a1+b*r1. Получаем a1 как остаток от деления r0 на b, а r1 - как частное. Так как последовательность частных от целочисленного деления на b > 1 строго убывает и частные неотрицательны, то в какой-то момент r0 станет равнго нулю и процесс остановится.

В результате мы получим набор коэффициентов a0, a1, ..., an однозначно задающие наш полином.

Запись чисел в различных системах счисления

Обычно запись по сравнению с полиномиальной переворачивается, то есть представление элементов записывается как an...a1a0.

Одной цифре обычно соответствует один символ.

  • 0-9 хватает для записи B от 2 до 10 включительно
  • 0-9 A-Z хватает для B от 2 до 36 включительно
  • 0-9 A-Z a-z хватает для B от 2 до 62 включительно

Цифры 0-9 соответствуют числам 0-9, цифры A-Z для 10-35, цифры a-z для 36-61.

Например, при b=8 используют цифры от 0 до 7. При b=12 используют цифры от 0 до 9 и буквы A,B.

Оценка длины записи числа

Цифры не превосходят b-1. n+1 > logb(x) >= n

Запись отрицательных чисел

Запись отрицательного числа x < 0 осуществляется так же, как и в десятичной системе: сначала пишется знак -, затем идет представление числа -x (которое уже является положительным).

Реализация: обработка одной цифры

Для работы с представлениями целых чисел, записанных в произвольной системе счисления, достаточно написать 2 функции. Одна функция считывает число записанное в системе счисления с заданным основанием в память. Вторая записывает число в заданной системе счисления. То есть на входе, например число в переменной типа long long, а на выходе просто представление числа.

Два массива:

long long values[256] - содержит значения цифр, обозначаемых символом с данным кодом или 100, если данная цифра не используется.

Например: values['0'] = 0; values['9'] = 9; values['A'] = 10; values['Z'] = 35; values['l'] = 47; values['@'] = 100

char digits[62] - содержит символы, используемые для цифр, то есть digits[i] - символ для цифры со значением i.

Например: digits[3] = '3'; digits[9] = '9''; digits[28] = 'S'; digits[60] = 'y'

Реализация: инициализация таблиц цифр

Сначала решим промежуточную задачу: по значению цифры выдать ее символьное представление и по символьному представлению выдать числовое значение. Для этого напишем функцию инициализации.

long long values[256];
char digits[62];

// вызвать сразу же в начале main()
void initDigits() {
  // числовое значение текущей цифры
  long long curDigit = 0LL;

  for (int i = 0; i < 256; i++) {
    // инициализируем все символы значением Impossible
    values[i] = 100LL;
  }

  for (char a = '0'; a <= '9'; a++) {
    // для всех цифр от 0 до 9
    // заполняем индексы символа и значения
    // и увеличиваем значение
    values[a] = curDigit;
    digits[curDigit++] = a;
  }

  for (char a = 'A'; a <= 'Z'; a++) {
    // затем то же самое для всех заглавных букв
    // цифры 10-35
    values[a] = curDigit;
    digits[curDigit++] = a;
  }

  for (char a = 'a'; a <= 'z'; a++) {
    // затем то же самое для всех строчных букв
    // цифры 36-61
    values[a] = curDigit;
    digits[curDigit++] = a;
  }

  return;
}

Для того чтобы получить i-ю по порядку цифру, надо взять элемент digits[i].

Для того чтобы получить число соответствующее цифре с кодом i. надо взять элемент values[i]. Если значение >= b или равно 100, то это значение не используется в системе счисления с основанием b и надо остановиться.

Таким образом, при поиске цифры мы берем элемент массива. Функция инициализации вызывается один раз в самом начале выполнения программы.

Реализация: чтение b-ичного числа

Уже умея решать промежуточную задачу, напишем функцию, которая по строке основания системы счисления b, выдает значение этой строки как b-ичного числа. Считается что число помещается в тип long long. Будем рассматривать цифры справа налево.

long long readBaseB(string a, int base) {
  long long res = 0LL, mult = 1LL;
  int i = 0;

  if (a[i] == '-') {
    // обработка минуса
    mult = -1LL;
    i++;
  }

  for (; i < a.length(); i++) {
    if (values[a[i]] < base) {
      // добавляем цифры слева направо
      res = res * (long long)base + values[a[i]];
    } else {
      // цифры не могут быть меньше base,
      // так что тут остановка считывания
      break;
    }
  }

  return res * mult;
}

Реализация: запись b-ичного числа

Осталось написать последнюю функцию, которая обратна данной и по числу основания системы счисления b, наполняют заданную строку b-ичным представлением числа. Вспомним построение записи числа в b-ичной системе и будем действовать аналогичным образом.

Пока числол x не равно 0, очередная цифра равна остатку от деления x на b. А само x заменяется на частное от целочисленного деления x на b. В итоге получим набор цифр числа записанного в обратном порядке. Поэтому полученную строку надо перевернуть.

string writeBaseB(long long x, int base) {
  string theSign = "", res = "";
  if (x < 0LL) {
    x = -x;
    theSign = "-";
  }
  for (; x != 0LL; x /= base) {
    res.push_back(digits[x%base]);
  }
  int len = res.length();

  // переворачиваем
  for (int i = 0; len-1-i > i; i++) {
    swap(res[i], res[len-1-i]);
  }

  // если строка не пустая, выдаем результат, иначе выдаем 0
  return len ? theSign+res : "0";
}

Мы научились преобразовывать из любой системы в любую систему. Но не все олимпиадные задачи преобразования между системами счисления можно решить с помощью этих функций.

В некоторых задачах требуется специально выполнить преобразование очень длинного числа, которое не влезет в long long и другие стандартные типы данных. Обычно в таких задачах одно основание оказывается степенью другого. Например, надо перевести из 8-чной системы в 4-ную число размером 1000 байт. Понятно, что оно ни в какой стандартный тип не влезет. Городить для этого длинную арифметику смысла нет. Можно сделать преобразование между промежуточными степенями.

Перевод между системами с основаниями b и bk

Пример: между двоичной и шестнадцатеричной

0000=0 0001=1 0010=2 ... 1111=F

Переведем, например, A55B26 в двоичную

A=1010 5=0101 5=0101 B=1011 2=0010 6= 0110

A55B26 = 1010 0101 0101 1011 0010 0110

Переведем, например, 100010111000001001 в шестнадцатеричную:

0010=2 0010=2 1110=E 0000=0 1001=9

// Функция перевода шестнадцатеричной цифры в двоичное представление
string hexToBin(char hex) {
  switch (toupper(hex)) {
    case '0': return "0000";
    case '1': return "0001";
    case '2': return "0010";
    case '3': return "0011";
    case '4': return "0100";
    case '5': return "0101";
    case '6': return "0110";
    case '7': return "0111";
    case '8': return "1000";
    case '9': return "1001";
    case 'A': return "1010";
    case 'B': return "1011";
    case 'C': return "1100";
    case 'D': return "1101";
    case 'E': return "1110";
    case 'F': return "1111";
    default: return ""; // Неверный символ
  }
}

// Функция перевода из двоичного в восьмеричное представление
string binToOct(const string& bin) {
  string result = "";
  int n = bin.size();
  int r = n % 3; // остаток

  // Если количество битов не кратно 3, добавляем ведущие нули
  if (r != 0) {
    string prefix(3 - r, '0');
    result += prefix + bin;
  } else {
      result = bin;
  }

  string oct = "";
  for (size_t i = 0; i < result.size(); i += 3) {
    string chunk = result.substr(i, 3);
    int value = (chunk[0] - '0') * 4 + (chunk[1] - '0') * 2 + (chunk[2] - '0');
    oct += (value + '0');
  }

  // Удаляем ведущие нули
  while (oct.size() > 1 && oct[0] == '0') {
    oct.erase(oct.begin());
  }

  return oct;
}

int main() {
  string hex;
  cin >> hex;

  // Шестнадцатеричное число переводим в двоичное
  string bin = "";
  for (char c : hex) {
    bin += hexToBin(c);
  }

  // Перевод двоичного числа в восьмеричное
  string oct = binToOct(bin);

  // Вывод результата
  cout << oct;
}

5.3 Хранение чисел в компьютере

Размер целочисленных типов

  • char 1 байт
  • short 2 байта
  • int 4 байта
  • long 4 байта
  • long long 8 байт

Беззнаковые типы: диапазон

  • unsigned char 0-255
  • unsigned short 0-65535
  • unsigned int 0-4'294'967'295
  • unsigned long 0-4'294'967'295
  • unsigned long long 0-18'446'744'073'709'551'615

Переполнение

Если к самому максимальному числу прибавить 1, то произойдет переполнение. Посмотрим как это происходит с 8 разрядным числом

1111 1111 + 1 = 1 0000 0000

Результат, а это 8 нулей, запишется в 8 разрядную перменную, а 1 в старшем разряде будет потеряна.

Переход к знаковым переменным

У всех отрицательных чисел старший бит равен 1. Переполнение при сложении положительных чисел сожет дать в результате отрицательное число.

Предотвращение переполнения

  1. Выбирать правильный тип данных
  2. Учитывать не только результат, но и максимальное промежуточное значение, например (a*b)%c
  3. Правильный порядок действий. Пример 1, вместо (a*b)/d выполнить (a/d)*b. Пример 2, вместо (a+b)/2 выполнить a+(b-a)/2.

Устройство вещественных чисел

Хранятся в виде двоичной дроби в экспонениальной форме: x * 2y, где 1 ≤ x < 2 - вещественное, y - целое.

Целая часть x тождественно равна 1 => достаточно хранить дробную.

Тип double хранится в 64-битной переменной

Биты 0-51 дробная часть x

Биты 52-62 показатель степени двойки по формуле y-1023 где y образованное этими битами двоичное число.

Бит 63 знак числа.

Double: диапазон и особые случаи

52-62 биты равны 0 только для числа 0

52-62 биты равны 1 это переполнение или бесконечность

Миимальное значение 2.225073858072014 * 10-308

Максимальное значение 1.7976931348623157 * 10308

Потеря точности

Результаты вычислений почти всегда бесконечная дробь. При делении дробь бесконечная если делим не на степень 2. При отбрасывании хвоста теряется точность.

Поэтому нельзя сравнивать вещественные числа на равенство напрямую. Можно потерять случай когда равные числа стали различные из-за ошибок округления.

Правильное сравнение вещественных чисел

Выбирается минимальное пороговое значение с которым сравнивается разница чисел

#define EPS 0.000000001
bool isEaual(double a, double b) {
  return (fabs(a-b) <= EPS);
}

Всегда необходимо стараться вычислять в целых числах, пока это возможно, не уходя в вещественные числа.
