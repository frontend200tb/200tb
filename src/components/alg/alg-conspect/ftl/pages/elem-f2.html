Занятие 2. Установка CodeBlocks

Занятие 2. Установка CodeBlocks (2023-09-20)

Скачаем установочный файл программы CodeBlocks и MinGW.

Компилятор переводит программу в машинный код, после этого машинный код запускается и выполняется. Интерпретатор в реальном времени переводит программу инструкция за инструкцией а в это время уже переведенные инстукции выполняются.

В CodeBlocks можно настроить стандарт компилятора для нашей программы.

Можно русифицировать CodeBlock, но это не рекомендуется делать. Лучше сразу привыкать пользоваться английской версией программы. Для русификации надо найти

locale CodeBlocks

и скачать архив с русификацией. Распаковываем архив и папочку ru_RU копируем в папку

d:/Programs/CodeBlocks/Share/CodeBlocks/locale/

Если папки locale еще нет, то надо ее создать.

Открываем CodeBlock и выбираем пункт меню

Settings - Environment - View

Ставим галочку около Internationalization и в его окошке выбираем Russian. Нажимаем OK.

Редактор CodeBlocks встречается на олимпиадах, поэтому будем использовать его. Для начала работы в нем можно создать проект, а можно создать файл. При создании проекта создается папочка с проектом. И этот проект можно отлаживать в режиме debug. Мы не будем создавать проект, всегда будем создавать файл. Для этого выберем пункт меню

File - New - Empty File Ctrl-Shift-N

У нас создастся новый файл с названием untitled. Среда предлагает разработку на языках Си, C++ и Fortran. При создании нового файла среда еще не знает на каком языке мы будем разрабатывать программу. Чтобы файл из обычного текстового файла превратился в файл с кодом на языке C++ и подсветкой синтаксиса, нужно файл сохранить с расширением .cpp.

Библиотека bits/stdc++.h

В начале программы мы прописываем все библиотеки которые нам нужны и их может быть много. Некоторые разработчики вместо них подключают всего одну библиотеку bits/stdc++.h. Но нам это не подходит, потому что если вместо строчки

#include <iostream>

на олимпиаде написать строчку

#include <bits/stdc++.h>

то работать ничего не будет. bits это не документированная библиотека и может где-то работать, а где-то нет. Везде работать может только то, что задокументировано в языке. Поэтому никогда не используйте библиотеку bits/stdc++.h.
