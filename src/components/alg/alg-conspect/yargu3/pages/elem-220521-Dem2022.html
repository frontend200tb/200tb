Задача A. Лесник

Ограничение по времени 1с
Ограничение по памяти 256мб

Лесник накопил множество историй за время жизни в лесу (целых n штук), каждую из которых он хочет рассказать своему новому гостю, ведь у него так давно не было гостей.

Наш Лесник давно не общался с людьми, но помнит все правила этикета. По правилам, прежде чем начать рассказывать историю гостю, нужно налить ему выпивки, тогда гость будет слушать внимательнее. Кроме того, каждой новой истории должна предшествовать и новая кружка напитка.

Задача B. Дайте людям рому

Ограничение по времени 1с
Ограничение по памяти 256мб

У трактирщика большие проблемы — в его погребе закончился весь ром. Поэтому он отправил своего помощника Игоря в город за бочкой рома.

В силу ужасных погодных условий прямую дорогу из города в село, где находится трактир, занесло снегом. Теперь перед Игорем стоит сложная задача — вернуться обратно в трактир как можно быстрее, иначе трактирщика будет ждать участь пианиста и скрипача.

Благо у Игоря есть карта. Все населенные пункты на ней пронумерованы, и Игорь в городе с номером s. Трактир же находится в населенном пункте с номером f. Кроме того на карте указаны все дороги. Каждая дорога соединяет два различных населенных пункта. Игорь знает, сколько минут потребуется проехать по каждой из дорог.

Решение

Решение такое же как у задачи C. Поездка в город. Нужно только немножко изменить пару строк под условие.

#include <iostream>
#include <iomanip>
#include <set>
#include <vector>

using namespace std;

int main() {
  // ускорение ввода-вывода
  ios::sync_with_stdio(0);
  cin.tie(0);

  // ввод данных
  int n, m, s, f;
  cin >> n >> m >> s >> f;
  vector<vector<pair<long long, int> > > G(n + 1);
  vector<int> Mark(n + 1);
  vector<long long> Dist(n + 1);
  long long a, b, w;
  pair<long long, int> tmp1, tmp2;

  // соберем список смежности
  for (int i = 0; i < m; i++) {
    cin >> a >> b >> w;
    tmp1.first = w;
    tmp1.second = a;
    G[b].push_back(tmp1);
    tmp1.second = b;
    G[a].push_back(tmp1);
  }

  // инициализация
  set<pair<long long, int> > st;
  tmp1.first = 0;
  tmp1.second = s;
  st.insert(tmp1);

  // алгоритм Дейкстры
  while (st.size() > 0) {
    tmp1 = *st.begin();
    st.erase(st.begin());
    if (Mark[tmp1.second] == 0) {
      Mark[tmp1.second] = 1;
      Dist[tmp1.second] = tmp1.first;
      for (int i = 0; i < G[tmp1.second].size(); i++) {
        tmp2 = G[tmp1.second][i];
        tmp2.first += tmp1.first;
        st.insert(tmp2);
      }
    }
  }

  // если нет пути из s в f
  if (Mark[f] == 0) {
    cout << -1;
    return 0;
  }

  cout << Dist[f];
}

Задача E. Злобный гений

Ограничение по времени 1с
Ограничение по памяти 256мб

Злобный гений опять занимается исследованиями. В своих каверзных планах он сделает n экспериментов с числами. Каждый эксперимент состоит из набора подряд идущих натуральных чисел от lᵢ до rᵢ включительно. Злобный гений хочет определить количество простых чисел в каждом из наборов.

Входные данные

В первой строке задано целое число n (1 ≤ n ≤ 10⁵) — количество экспериментов.

Далее каждый эксперимент задан двумя целыми числами lᵢ и rᵢ (1 ≤ lᵢ ≤ rᵢ ≤ 10⁷) - границы набора целых чисел.

Выходные данные

Выведите для каждого эксперимента в отдельной строке количество простых чисел.

Примеры

Пример 1

3
2 10
5 5
13 1013

4
1
165

Пример 2

1
1 10000000

664579
Решение

Соберем решето Эратосфена от 1 до 10⁷. Создадим префикс-функцию, в которой для каждого числа i в этом диапазоне будет храниться количество простых чисел от 1 до i включительно.

#include <iostream>
#include <vector>

using namespace std;

int main() {
  ios_base::sync_with_stdio(false);
  cin.tie(nullptr);

  int n;
  cin >> n;

  int m = 10000000;
  vector<int> A(m + 1, 0);
  //vector<int> B;
  vector<int> prefB(m + 1, 0);
  A[0] = 1;
  A[1] = 1;

  for (long long i = 2; i <= m; i++) {
    if (A[i] == 0) {
      //B.push_back(i);
      prefB[i] = prefB[i - 1] + 1;
      for (long long j = i * i; j <= m; j += i) {
        A[j] = 1;
      }
    } else {
      prefB[i] = prefB[i - 1];
    }
  }

  // обработка запросов
  while (n--) {
    int li, ri;
    cin >> li >> ri;
    cout << prefB[ri] - prefB[li - 1] << '\n';
  }
}

Задача G. Некромант

Ограничение по времени 1с
Ограничение по памяти 256мб

В селе переполох, мертвецы вышли из своих могил. Игорь отправился на кладбище, чтобы разобраться в причине сего несчастья. Проходя по кладбищу, он наткнулся на колбу со странной жидкостью, сумку, забитую причудливыми ингредиентами, а также дневник с записями.

Увидев ингредиенты, Игорь сразу понял, что причина переполоха в черной магии и зельеварении. Поэтому, чтобы усмирить мертвецов, необходимо сварить антизелье, а для этого узнать состав каждого ингредиента.

В найденном дневнике некроманта написано, что каждый ингридиент имеет свой номер: 1 — росток гороха, 2 — куриная лапа, 3 — ночной гриб, 4 — цветок василька, 5 — корень мандрагоры, 6 — хвост скорпиона, 7 — лягушка и 8 — воронье перо. Также ингредиенты могут быть представлены в определенном виде, например, ингредиенты a, b, c, d, e, f, g, h:

  • a: «+ big red + big blue + big green»,
  • b: «- big red - big blue - big green»,
  • c: «- small red - big blue + small green»,
  • d: «+ small red + big blue - small green»,
  • e: «+ small red - small blue - big green»,
  • f: «- small red + small blue + big green»,
  • g: «- big red + small blue - small green»,
  • h: «+ big red - small blue + small green»,

Из общения с колдунами в трактире Игорь знает, что каждый матерый колдун смешивает ингредиенты в котле. Если компоненты ингредиентов взаимодействуют между собой, то появляется пар определенного цвета (красный, синий или зеленый), наполненный символами плюс, либо символами минус. Если ингредиенты не взаимодействуют, то есть нейтральны друг другу, то пар белый. Взаимодействие между ингредиентами зависит от составляющих их компонентов, пар наполненный символами появится только в том случае, если компоненты соответствующего цвета будут разного размера и иметь одинаковый знак, в противном случае ингредиенты нейтральны, и пар будет белым.

Вам необходимо определить точный состав каждого ингридиента, либо вывести «No solution» (без кавычек), если информации из дневника недостаточно, чтобы однозначно определить составы ингредиентов.

Задача I. Арбузы

Ограничение по времени 5с
Ограничение по памяти 256мб

Это интерактивная задача.

Дядюшке Кадырбечу нужно купить ровно k арбузов на рынке у продавца Гарика. У дядюшки Кадырбеча большая семья, поэтому ему надо купить k самых больших арбуза, которые есть у Гарика.

У Гарика на прилавке в ряд выложено n арбузов, все они различного веса, и он не знает какие арбузы самые большие. Дядюшка Кадырбеч может попросить Гарика взвесить два арбуза, которые стоят на позициях pos1 и pos2, чтобы узнать, какой из них тяжелее. В конце концов дядюшка Кадырбеч должен выбрать k самых тяжелых арбуза для покупки. Помогите дядюшке Кадырбечу со стратегией взвешиваний, чтобы определить какие k арбузов самые тяжелые, но сделать это нужно за не более чем 2n + 113 взвешиваний, потому что позади него выстраивается длинная и яростная очередь.

Обратите внимание на ограничения k.

Протокол взаимодействия

Сначала на вход вашей программе подается два целых числа n и k (2 ≤ n ≤ 1.5⋅10⁵, k ≤ min (n, min(max(10,n/80), 800))) — количество арбузов, которые лежат на прилавке у Гарика, и количество арбузов, сколько стоит выбрать. Все арбузы различны по весу.

После этого ваша программа может делать запросы двух типов к тестирующей системе:

  1. ? pos₁ pos₂ — попросить Гарика взвесить арбузы на позиции pos₁ и pos₂. В ответ тестирующая система в поток ввода пользователя выдаст '<', если арбуз на позиции pos₁ меньше чем на позиции pos₂, иначе выводит '>'. При этом если pos₁ = pos₂, вы получите вердикт «Wrong Answer»
  2. ! PositionFirstMax, PositionSecondMax, ... ,PositionKMAX — означает, что дядюшка Кадырбеч хочет купить арбузы на этих позициях, причем PositionFirstMax — это позиция, на которой лежит самый тяжелый арбуз, PositionSecondMax — на которой лежит второй по тяжести арбуз и т. д. . После этого взаимодействие с тестирующей системой прекращается, и, если выбраны верные арбузы, то вы получаете вердикт «OK», иначе «Wrong Answer».

Количество запросов должно быть не более чем 2n + 113

Если ваша программа сделает слишком много запросов или задаст некорректный запрос, то вы получите любой вердикт отличный от «OK».

После каждого действия вашей программы выводите символ перевода строки. Если вы используете «writeln» в Паскале, «cout << ... << endl» в C++, «System.out.println» в Java или «print» в Python, сброс потока вывода у вас происходит автоматически, дополнительно делать «flush» не oбязательно. Если вы используете другой способ вывода, рекомендуется делать «flush», но все равно обязательно требуется выводить символ перевода строки.

Пример 

2 2

<

? 1 2

! 2 1

В первом тесте арбузы весов {1,2}, и показано возможное взаимодействие между решением и тестирующей системой.

Задача J. Паровой каток

Ограничение по времени 1с
Ограничение по памяти 256мб

Улицы города Иннограда на схеме образуют сетку из N идущих вертикально и M идущих горизонтально. Расстояния между соседними узлами сетки 100 м. Паровой каток должен проехать по всем улицам по только что уложенному асфальту. Каток начинает своё движение в левом нижнем углу. Там же он и должен закончить работу. Напишите программу, рассчитывающую длину минимального маршрута катка в метрах, обеспечивающего выполнение задания.

Входные данные

В единственной строке два целых числа N и M, разделенные пробелом (2 ≤ N ≤ 1000, 2 ≤ M ≤ 1000).

Выходные данные

В единственной строке выведите ответ на задачу.

Пример 4 4 2800

Задача K. Выбор ВУЗа

Ограничение по времени 3с
Ограничение по памяти 256мб

Перед Витей стоит сложный жизненный выбор — пора поступать в ВУЗ. Чтобы упростить себе задачу, он для каждого из N ВУЗов определил K параметров. Для объективной оценки ВУЗа Вася ввёл функцию F, которая для каждого ВУЗа вычисляется как сумма попарных произведений параметров ВУЗа. Теперь ему надо выбрать единственный ВУЗ — тот, в котором он будет учиться ближайшие несколько лет. Для этого Вите необходимо отсортировать ВУЗы по возрастанию значения данной функции. Он просит вас помочь ему в столь нелёгком занятии.

Входные данные

В первой строке вам дано два целых числа — N и K (1 ≤ N, K ≤ 2500) — количество ВУЗов и параметров соответственно. В следующих N строках содержатся K натуральных чисел — параметры ВУЗов, каждый параметр не превосходит 10⁵.

Выходные данные

В единственной строке выведите N чисел через пробел — номера ВУЗов, отсортированных по возрастанию значения F. В случае равенства функции F ВУЗы должны следовать в том порядке, в котором они давались во входных данных.
