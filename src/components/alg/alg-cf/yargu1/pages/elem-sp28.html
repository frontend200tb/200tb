Геометрия #1. База

Проблемы вещественных чисел

Основные проблемы работы с вещественными числами:

  1. Вывод вещественного числа с нужной точностью
  2. Потеря точности с операциями вещественных чисел.

Для решения первой проблемы, необходимо понимать, как работает тестирующая система при проверке ответа участника и члена жюри. Если в условии задачи указано выведите ответ с точностью 4 знака после запятой. Тестирующая система проверяет по формуле |resjury - res| < 0.0001. Банальное хорошее решение проблемы выводить ответ с заведомо больших количеством цифр после запятой.

#include <iomanip> cout << fixed << setprecision(10) << a;

Вторая проблема потеря точности связана с представлением мантиссы у вещественных чисел. К сожалению, мантисса конечна при любой операции вещественных чисел существует ошибка в конце мантиссы. Если операций с вещественными числами большое количество ошибка накапливается и может стать существенной при сборе ответа.

Самый банальный наглядный пример:

При сложении 1000 вещественных чисел в разном порядке можно получить разные ответы. Они будут близки, но могут не совпадать. Чем больше будет чисел в сложении чем больше накопиться ошибка в ответе суммы.

Решение проблемы - стараться избегать вещественные числа и пользоваться по возможности целыми.

Структуры

Шаблон структур C++:

struct имя_структуры {
  компоненты_структуры
};

Доступ к членам структуры. Для доступа к членам структуры используется оператор точки (.).

Членами структуры считаются:

Поля - элементы, которые хранят информацию.

Методы - внутренние функции структуры.

Точка/вектор

Точка — это геометрическая фигура, из которой состоят все остальные геометрические фигуры.

Вектор — это направленный отрезок, то есть отрезок, имеющий длину и определённое направление.

Точка и вектор задаются в пространстве Rⁿ при помощи n координат. Например, на плоскости (x, y).

Структура точки/вектор с конструктором.

struct point {
  double x, y;

  point(double curx = 0, double cury = 0) {
    x = curx;
    y = cury;
  }
};

Конструктор представляют специальную функцию, которая имеет то же имя, что и класс, которая не возвращает никакого значения и которая позволяет инициализировать объект класса во время его создания и таким образом гарантировать, что поля класса будут иметь определенные значения.

Перегрузка операторов позволяет определить для объектов классов встроенные операторы, такие как +, -, * и т.д. Для определения оператора для объектов своего класса, необходимо определить функцию, название которой содержит слово operator и символ перегружаемого оператора.

point operator+ (point TMP) {
  point res;
  res.x = x + TMP.x;
  res.y = y + TMP.y;
  return res;
}

Пример работы:

point A, B(3, 5);
point C = B;
A = B + C;

Остальные полезные перегрузки для работы с векторами:

point operator- (point TMP) {
  point res;
  res.x = x - TMP.x;
  res.y = y - TMP.y;
  return res;
}

point operator* (point k) {
  point res;
  res.x = x * k;
  res.y = y * k;
  return res;
}

Полезное с векторами:

  1. Нахождение длины вектора: |a| = sqrt(a.x * a.x + a.y * a.y)
  2. Нахождение скалярного произведения: (a, b) = |a| * |b|cos(Θ) = a.x * b.x + a.y * b.y
  3. Нахождение векторного произведения: [a, b] = |a| * |b|sin(Θ) = a.x * b.y - a.y * b.x

Полезные функции:

// квадрат длины вектора
double dist2(point A) {
  return A.x * A.x + A.y * A.y;
}

double dot(point A, point B) {
  return A.x * B.x + A.y * B.y;
}

double cross(point A, point B) {
  return A.x * B.y - A.y * B.x;
}

Функцию cross можно посчитать и как определитель матрицы 2 x 2

|  A.x  B.x  |
|  A.y  B.y  |

определитель = A.x * B.y - A.y * B.x

Задача

Определить, являются ли вектора перпендикулярны друг другу. Для этого нужно проверить, что их скалярное произведение равно 0.

Полярные координаты

В евклидовом пространстве есть ось x и ось y. Для опрделения координаты точки нужно опустить перпендикуляры к этим осям.

Полярная координа имеет два параметра - расстояние до начала координат и угол поворота от оси x.

Все что связано с вращением, удобно описывать с помощью полярных координат.

Нормирование

Операция нормирование вектора - это изменение длины вектора до длины равной 1 (его уменьшение или сжатие на нужную величину) new_a = a / |a| .

point N = A * (1 / sqrt(dist2(A)) );

Поворот вектора

Операция поворот вектора на угол Θ против часовой стрелки:

new_a = (cos(Θ) -sin(Θ)) (x)
        (sin(Θ)  cos(Θ)) (y)

В этой формуле матрица поворота умножается на вектор. Перемножаем строчку на столбец:

new_ax = cos(Θ)*x - sin(Θ)*y
new_ay = sin(Θ)*x + cos(Θ)*y

Операция поворот вектора на угол Θ по часовой стрелке:

new_a = ( cos(Θ) sin(Θ)) (x)
        (-sin(Θ) cos(Θ)) (y)

Если нужно повернуть вектор против по часовой стрелки, подставим -Θ:

new_a = ( cos(-Θ) sin(-Θ)) (x)
        (-sin(-Θ) cos(-Θ)) (y)

Треугольник

В задачах по спортивному программированию сложные фигуры в том числе и треугольники будут задаваться при помощи набора точек. Какими способами можно найти площадь треугольника?

Площадь треугольника:

  1. Формула Герона: p = (a + b + c) / 2, S = sqrt(p * (p - a) * (p - b) * (p - c)).

    Никогда не используйте формулу Герона! Слишком большая потеря точности и возможное переполнение типов данных!

  2. Формула сторона на высоту: S = (1 / 2) * h * a.

    К сожалению, в задаче вы не будете знать высоту.

  3. Формула две стороны и угол между ними: S = (1 / 2) * a * b * sin(Θ) = |cross(a, b)| / 2.

    Площадь через синус угла очень похожа на векторное произведение (cross).

Медиана треугольника — это отрезок, соединяющий вершину треугольника с серединой противоположной стороны.

Пусть есть треугольник ABC. Пусть точка D это середина стороны AC. Тогда длина отрезка BD это и есть медиана.

Биссектриса треугольника — отрезок биссектрисы угла, проведённый из вершины треугольника к противолежащей стороне и делящий этот угол пополам.

Высота треугольника — это перпендикуляр, проведённый из вершины треугольника к прямой, содержащей противоположную сторону.

Пусть есть треугольник ABC. s - это его площадь. AC - это длина стороны к которой проведена высота. Тогда саму высоту можно найти по формуле

h = (s / AC) * 2;

Прямая

Школьное уравнение прямой нам не подходит, потому что оно описывает только те прямые, что пересекают ось Y.

y = k*x + b;

Общее уравнение прямой линии на плоскости в декартовых координатах:

Ax + By + C = 0

Так же это уравнение называют нормальным уравнением прямой, так как по коэффициентам A, B определяется вектор нормали (A,B). Вектор нормали - это вектор, который перпендикулярен прямой.

В задачах чаще всего прямые задаются по двум точкам.

Школьное (каноническое) уравнение прямой по двум точкам нам также не подходит, потому что есть операция деления

(x - x1) / (x2 - x1) = (y - y1) / (y2 - y1).

При решении задач с прямыми всегда необходимо переводить из канонического вида в нормальный вид прямой. Для этого умножаем обе части уравнения на знаменатели, раскрываем скобки и приходим к нормальному уравнению прямой.

(y2-y1)*(x-x1) = (x2-x1)*(y-y1) (y2-y1)*(x-x1) - (x2-x1)*(y-y1) = 0 (y2-y1)*x - (y2-y1)*x1 - (x2-x1)*y + (x2-x1)*y1 = 0 A = y2-y1, B = x1-x2, C = y1*x2-y2*x1 Ax + By + C = 0

Проверка принадлежности точки прямой

Первая формула верна математически, но она совсем не подходит для программирования. В программировании используют вторую формулу, где определяют с некоторой погрешностью ε.

A * px + B * py + C = 0. |A * px + B * py + C| < ε

Проверка что две точки лежат в одной полуплоскости относительно заданной прямой

Если подставить точки в уравнение прямой, они должны быть с одним знаком. Что означает, что прямая должна сдвигаться в одном направлении, чтобы пройти через эти точки. Если бы прямая сдвигалась в разных направлениях, чтобы пройти через эти точки, то точки лежат в разных полуплоскостях относительно этой прямой.

Вектор нормали

Вектор нормали прямой это перпендикуляр, опущенный на прямую. Зная нормальное уравнение прямой, можно найти вектор нормали прямой. Координаты вектора нормали это коэффициенты A и B из уравнения прямой. Чтобы найти вектор нормали в другом направлении, нужно изменить знак коэффициентов A и B.

Проверка что прямые параллельны

Есть две прямые, представленные нормальными уравнениями

A1*x + B1*y + C1= 0 A2*x + B2*y + C2= 0

Две прямые параллельны если их векторы нормали колинеарны, то есть коэффициенты A и B пропорциональны.

A1 / A2 == B1 / B2

Первая формула не подходит из-за возможного деления на ноль. Надо использовать вторую формулу.

A1 * B2 == B1 * A2

Точка пересечения двух прямых. Чтобы найти точку с координатами (x, y) пересечения двух прямых, нужно решить систему из двух уравнений.

A1x + B1y + C1 = 0 A2x + B2y + C2 = 0

Расстояние от точки до прямой.

Дана прямая, заданная уравнением A*x + B*y + C = 0. Дана точка p, заданная координатами p.x и p.y

Есть несколько вариантов решения этой задачи.

  1. Найти формулу для нахождения расстояния или вывести самостоятельно. abs (A*p.x + B*p.y + C) / sqrt(A*A + B*B)
  2. Построить перпендикулярную прямую, проходящую через точку A.
  3. Найти две точки на прямой a, построить треугольник и в нем найти высоту, опущенную на прямую.

Площадь фигуры

Задан многоугольник, как определить его площадь?

Во-первых, можно разбить многоугольник на треугольники и посчитать сумму их площадей. Но это очень трудная задача, потому что трудно определить по трем точкам, что треугольник внутри многоугольника, а не снаружи.

Воспользуемся метод трапеций (интегрирование). Площадь трапеции это полу сумма оснований на высоту. За высоту возьмем разность координат x соседних вершин основание значение y. При проходе по списку вершин учитываем направление по оси x, от нее зависит знак площади трапеции.

Плоскость

Нормальное уравнение плоскости отличается от нормального уравнения прямой, только еще одной координатой

Ax + By + Cz + D = 0

Вектор нормали

Вектор нормали плоскости это перпендикуляр, опущенный на плоскость. Зная нормальное уравнение плоскости, можно найти вектор нормали плоскости. Координаты вектора нормали это коэффициенты A, B и C из уравнения плоскости.

Задача A. Полярный угол точки

Ограничение по времени 1с
Ограничение по памяти 64мб

Входной файл angle1.in

Выходной файл angle1.out

Входные данные

Два числа - координаты точки, не совпадающей с началом координат.

Выходные данные

Одно число - величина ее полярного угла в радианах из интервала [0, 2pi]

Пример 2 3 0.98279
Решение

Если провести прямую через точку и через начало координат, то угол между этой прямой и осью X называется полярный угол точки.

Возьмем первый вектор длиной 1, выходящий из начала координат и лежащий на оси X, с координатами (0, 0), (1, 0). Возьмем второй вектор, выходящий из начала координат и заканчивающийся в заданной точке, с координатами (0, 0), (x, y). Надо найти угол между этими двумя векторами.

Есть короткое и красивое решение данной задачи при помощи функции atan2() из библиотеки cmath. Функция atan2 возвращает число в диапазоне от -пи до +пи, нам нужен ответ в диапазоне от 0 до 2пи. Поэтому если получится отрицательное число, то надо к нему прибавить 2пи.

#include <iostream>
#include <vector>
#include <iomanip>

using namespace std;

int main() {
  // ускорение ввода
  ios::sync_with_stdio(false);
  cin.tie(nullptr);

  // работа с файлами
  freopen("angle1.in", "rt", stdin);
  freopen("angle1.out", "wt", stdout);

  // ввод данных
  double x, y;
  cin >> x, y;

  // решение
  double res = atan2(y, x);

  if (res < 0) {
    res += 2 * 3.14159265;
  }

  // вывод результата
  cout << fixed << setprecision(10) << res;
}

Задача B. Угол между векторами

Ограничение по времени 1с
Ограничение по памяти 64мб

Входной файл angle1.in

Выходной файл angle1.out

Входные данные

Четыре числа - координаты двух ненулевых векторов.

Выходные данные

Одно число - величина неориентированного угла между ними с точностью до пятого знака после запятой из интервала [0, pi].

Пример 2 1 3 5 0.56673
Решение

Первый вариант. Определяем в какой четверти находится точка и выразить угол через арксинус или арккосинус.

Второй вариант. Более простой. Использовать арктангенс. Функция atan2 библиотеки cmath возвращает угол от -pi до pi [-pi, pi). При этом не нужно рассматривать каждую четверть по отдельности. Для решения нужно вычислить значение выражения

atan2(cross(A, point(1,0)), dot(A, point(1,0))

Если ответ получисля отрицательным, то к нему нужно прибавить pi.

Задача C. Площадь многоугольника

Ограничение по времени 1с
Ограничение по памяти 64мб

Входной файл area.in

Выходной файл area.out

Входные данные

В первой строке одно число N (3 ≤ N ≤ 100000). Далее в N строках по паре чисел - координаты очередной вершины простого многоугольника в порядке обхода по или против часовой стрелки.

Все координаты - целые числа, по модулю не превосходящие 10⁴

Выходные данные

Одно число - величина площади приведенного многоугольника.

Пример 
3
1 0
0 1
1 1
0.5
Решение

Будем собирать сумму многоугольника через сумму трапеций. Каждая трапеция формируется одной стороной многоугольника. Проходим в цикле по всем соседним вершинам и находим площадь как высота умноженная на полусумму основания трапеции.

#include <iostream>
#include <vector>
#include <iomanip>

using namespace std;

int main() {
  // ускорение ввода
  ios::sync_with_stdio(false);
  cin.tie(nullptr);

  // работа с файлами
  freopen("area.in", "rt", stdin);
  freopen("area.out", "wt", stdout);

  // ввод данных
  int n;
  cin >> n;

  // вектор точек многоугольника
  vector<pair<double, double> > A(n);
  for (int i = 0; i < n; i++) {
    cin >> A[i].first >> A[i].second;
  }

  // решение
  double result = 0;
  for (int i = 0; i < n - 1; i++) {
    result += (A[i].first - A[i + 1].first) * (A[i].second + A[i + 1].second) / 2;
  }
  result += (A[n - 1].first - A[0].first) * (A[n - 1].second + A[0].second) / 2;
  result = abs(result);

  // вывод результата
  cout << fixed << setprecision(8) << result;
}

Ограничение по памяти 64мб

Входной файл distance1.in

Выходной файл distance1.out

Найдите расстояние от заданной точки до заданной прямой.

Входные данные

Пять целых чисел - координаты точки и коэффициенты A, B и C нормального уравнения прямой.

Одно число - расстояние от точки до прямой с точностью не менее 10⁻⁶

Пример 1 1 1 1 -1 0.70711

Задача T. Расстояние между лучами

Ограничение по времени 1с
Ограничение по памяти 64мб

Входной файл raydist.in

Выходной файл raydist.out

На плоскости имеются два луча, заданные парами точек. Необходимо определить расстояние между лучами.

Входные данные

Во входном файле даны четыре пары целых чисел - описания первого и второго луча. Каждый луч описывается координатами своего начала и координатами некоторой точки на луче.

Выходные данные

Вывести одно вещественное число с точностью до 5-го знака после запятой - искомое расстояние.

Пример 
2 1 1 3
0 1 4 -1
0.89443

Задача U. Треугольник и точка

Ограничение по времени 1с
Ограничение по памяти 16мб

В декартовой системе координат на плоскости заданы координаты вершин треугольника и еще одной точки. Требуется написать программу, определяющую, принадлежит ли эта точка треугольнику.

Входные данные

В четырех строках находятся пары целых чисел - координаты точек. Числа в первых трех строках — это координаты вершин треугольника (x1,y1), (x2,y2), (x3,y3), в четвертой строке — координаты тестируемой точки (x4,y4). Все координаты не превышают 10000 по абсолютной величине.

Выходные данные

Необходимо вывести слово «In», если точка находится внутри треугольника и «Out» в противном случае.

Примеры

Пример 1

0 0
100 0
0 100
100 100
Out

Пример 2

0 0
100 0
0 100
10 10
In

Пример 3

0 0
100 0
0 100
50 50
In

Пример 4

0 0
100 0
0 100
0 0
In
